# 문자 발송 내역 (/admin/marketing/history) — 투어 시나리오

## 페이지 목적
SMS/문자 메시지 발송 이력을 조회하고, 각 발송의 상태(성공/실패/대기)와 상세 내용을 확인하는 페이지

## 시나리오 (8 스텝)

### Step 1: 발송 내역 페이지 진입
- **title**: 문자 발송 내역 페이지 개요
- **text**: 이 페이지는 시스템에서 발송된 모든 SMS/문자 메시지의 이력을 관리하는 곳입니다. 상단에는 검색바, 날짜 범위 필터, 조회/초기화 버튼이 있는 도구 모음이 있습니다. 아래로는 발송 내역 테이블이 표시되며, 각 행을 클릭하면 상세 내용 모달이 열립니다.
- **highlight**: `.max-w-7xl.mx-auto`
- **trigger**: confirm
- **tips**:
  - 발송 내역은 생성일 역순(최신순)으로 정렬됨
  - 페이지당 20개씩 표시
  - 총 발송 건수와 현재 페이지 정보가 하단에 표시됨

### Step 2: 검색 입력창
- **title**: 발송 내역 검색하기
- **text**: 상단 검색 입력창(이름, 연락처로 검색...)을 사용하여 특정 수신자나 메시지 내용으로 발송 내역을 검색할 수 있습니다. 입력 시 실시간으로 결과가 필터링되지는 않으며, 입력 후 '조회' 버튼을 클릭하면 검색 결과가 표시됩니다. 검색어를 비우고 초기화 버튼을 클릭하면 모든 발송 내역이 다시 표시됩니다.
- **highlight**: `input[name="q"]`
- **trigger**: input
- **tips**:
  - 검색은 이름, 연락처, 메시지 내용 필드를 포함
  - 다른 필터(날짜 범위)와 함께 사용 가능
  - URL 매개변수로 저장되어 페이지 새로고침 후에도 유지됨

### Step 3: 날짜 범위 필터
- **title**: 발송 기간으로 필터링
- **text**: 도구 모음의 날짜 입력 필드(시작일, 종료일)를 사용하여 특정 기간의 발송 내역을 조회할 수 있습니다. 시작일과 종료일을 입력하고 '조회' 버튼을 클릭하면 해당 기간의 발송 내역만 표시됩니다. 종료일은 자동으로 해당 날짜의 자정 이후를 포함하도록 처리됩니다.
- **highlight**: `input[name="start_date"], input[name="end_date"]`
- **trigger**: input
- **tips**:
  - 날짜 형식: YYYY-MM-DD
  - 시작일만 입력하면 그 이후 발송 내역 표시
  - 종료일만 입력하면 그 이전 발송 내역 표시
  - 검색 조건과 함께 사용 가능

### Step 4: 조회 및 초기화 버튼
- **title**: 필터 적용 및 초기화
- **text**: 도구 모음의 '조회' 버튼을 클릭하면 입력한 검색어와 날짜 범위에 따라 발송 내역이 필터링됩니다. 검색 조건이 있을 때만 '초기화' 버튼이 표시되며, 이를 클릭하면 모든 검색 조건이 해제되고 전체 발송 내역이 다시 표시됩니다.
- **highlight**: `button[type="submit"], a:has-text("초기화")`
- **trigger**: click
- **buttons**:
  - selector: `button[type="submit"]`
  - label: "조회"
  - desc: 검색 조건 적용
  - selector: `a:has-text("초기화")`
  - label: "초기화"
  - desc: 모든 검색 조건 제거
- **tips**:
  - 초기화 버튼은 검색 조건이 없을 시 표시되지 않음
  - 초기화 후 페이지가 새로고침됨

### Step 5: 발송 내역 테이블 구성
- **title**: 발송 내역 조회
- **text**: 발송 내역 테이블에는 다음 칼럼들이 표시됩니다: 일시(발송 일시), 구분(캠페인/상담답변/알림/SMS/개별문자), 수신자(이름과 연락처), 내용(메시지 본문), 상태(성공/실패/대기), 발신자(관리자 또는 '시스템'). 각 행을 클릭하면 상세 내용 모달이 열립니다.
- **highlight**: `table.w-full`
- **trigger**: confirm
- **tips**:
  - 발송 일시는 YY-MM-DD HH:MM 형식으로 표시
  - 메시지 내용이 길 경우 자동으로 말줄임 처리됨
  - 행을 클릭하면 상세 모달이 열림

### Step 6: 발송 상태 배지
- **title**: 발송 상태 확인
- **text**: 테이블의 '상태' 칼럼에는 발송 결과가 컬러 배지로 표시됩니다: 초록색 '성공'(발송 완료), 빨간색 '실패'(발송 오류), 노란색 '대기'(발송 대기 중). 발송 실패 시 오류 메시지가 아래 작은 글씨로 '!오류'로 표시되며, 마우스를 올리면 상세 오류 메시지를 볼 수 있습니다.
- **highlight**: `td:has(.bg-green-100), td:has(.bg-red-100), td:has(.bg-yellow-100)`
- **trigger**: confirm
- **tips**:
  - 성공: 수신자가 메시지를 수신함
  - 실패: 오류로 인해 발송 실패
  - 대기: 아직 발송 대기 중인 상태
  - 오류 메시지는 title 속성에 저장되어 호버 시 표시됨

### Step 7: 메시지 구분 배지
- **title**: 발송 유형 확인
- **text**: 테이블의 '구분' 칼럼에는 발송 메시지의 유형이 배지로 표시됩니다: 캠페인(파란색), 상담 답변(회색), 알림(보라색), SMS(회색), 개별문자(회색). 각 배지의 색상으로 발송 유형을 빠르게 구분할 수 있습니다.
- **highlight**: `span[class*="bg-"][class*="text-"]`
- **trigger**: confirm
- **tips**:
  - 각 유형별로 색상이 일정함
  - 캠페인은 파란색으로 강조 표시
  - 유형별 발송 통계를 추출할 수 있음

### Step 8: 발송 상세 모달
- **title**: 발송 내역 상세 조회
- **text**: 테이블의 행을 클릭하면 '발송 내역 상세' 모달이 열립니다. 모달 상단에는 수신자, 발송 일시, 메시지 유형, 발송 상태 정보가 카드 형태로 표시됩니다. 중간에는 '메시지 내용' 섹션에 메시지 전문이 표시되며, 발송 실패 시 하단의 '에러 메시지' 섹션에 오류 상세가 표시됩니다. '닫기' 버튼으로 모달을 종료합니다.
- **highlight**: `#log-detail-modal`
- **trigger**: click
- **buttons**:
  - selector: `button:has-text("닫기")`
  - label: "닫기"
  - desc: 상세 모달 종료
- **tips**:
  - 모달은 행 클릭 시 자동으로 해당 정보 로드
  - 모달 외부 또는 닫기 버튼으로 종료 가능
  - 메시지 내용은 최대 높이 400px까지 스크롤 가능
  - 오류 메시지는 오류가 있을 때만 표시됨

### Step 9: 모달 정보 구성
- **title**: 모달 상세 정보 확인
- **text**: 상세 모달의 상단 카드에는 수신자(이름, 연락처), 발송 일시, 발송 유형 배지, 발송 상태가 표시됩니다. 수신자 정보는 이름과 연락처(하이픈 형식 010-1234-5678)로 구성되며, 발송 일시는 한국 시간대(Asia/Seoul)로 표시됩니다. 발송 유형과 상태는 색상 배지로 구분됩니다.
- **highlight**: `.grid.grid-cols-2.gap-4.bg-slate-50`
- **trigger**: confirm
- **tips**:
  - 모든 정보는 HTML 이스케이프 처리되어 안전함
  - 연락처는 자동으로 포맷됨 (010-1234-5678)
  - 날짜는 로컬라이제이션되어 한글 형식으로 표시

### Step 10: 페이지네이션
- **title**: 발송 내역 페이지 이동
- **text**: 테이블 하단의 페이지네이션 영역에서 이전/다음 페이지를 이동할 수 있습니다. 현재 페이지와 전체 페이지 수가 표시되며, 페이지 번호 버튼(1, 2, 3... 또는 생략...)을 클릭하여 특정 페이지로 이동 가능합니다. 페이지 전환 시 현재 검색 조건(검색어, 날짜 범위)이 URL 매개변수로 유지됩니다.
- **highlight**: `.flex.gap-1`
- **trigger**: click
- **tips**:
  - 페이지당 20개 발송 내역 표시
  - 현재 페이지 번호는 검정 배경으로 강조
  - 첫 페이지에서는 '이전' 버튼 비활성화
  - 마지막 페이지에서는 '다음' 버튼 비활성화
  - 페이지 이동 시 검색 조건이 자동으로 URL에 포함됨
