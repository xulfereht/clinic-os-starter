# 지출 관리 (/admin/expenses) — 투어 시나리오

## 페이지 목적
병원의 외부 거래처 입금 요청(지출)을 기록하고 지급 상태를 관리합니다.

## 시나리오 (7 스텝)

### Step 1: 현재 월 지출 현황 확인
- **title**: 지출 관리 시작하기
- **text**: 이 페이지는 병원이 거래처나 공급사에 지급해야 할 금액을 관리하는 중심지입니다. 페이지 상단의 월 네비게이션으로 과거 달이나 미래 달의 지출 현황을 확인할 수 있습니다. 현재 년월이 제목에 표시되며, 양쪽 화살표 버튼으로 월을 이동할 수 있습니다. '이번 달' 버튼을 누르면 현재 월로 즉시 돌아갑니다.
- **highlight**: `.flex.flex-col.md:flex-row.gap-4` (월 네비게이션)
- **trigger**: click
- **tips**:
  - 월을 이동해도 해당 월의 지출 현황으로 자동 갱신됩니다
  - 과거 달의 지출 기록도 모두 조회할 수 있습니다
  - 미래에 예정된 지급도 미리 등록할 수 있습니다
- **buttons**:
  - selector: `button[onclick="changeMonth(-1)"]`
  - label: 이전 달
  - desc: 지난 달 지출 현황을 봅니다
  - selector: `button[onclick="changeMonth(1)"]`
  - label: 다음 달
  - desc: 다음 달 지출 현황을 봅니다

### Step 2: 상태별 필터링
- **title**: 지급 상태로 필터링하기
- **text**: 페이지 중간의 필터 버튼들을 이용해 지출을 상태별로 분류할 수 있습니다. '전체'는 모든 지출을 보고, '미지급'(빨간 점 🔴)은 아직 돈을 보내지 않은 항목만, '지급완료'(초록 점 🟢)는 이미 입금한 항목만 표시합니다. 이를 통해 얼마나 더 지급해야 할 액수를 빠르게 파악할 수 있습니다.
- **highlight**: `.flex.flex-wrap.gap-2.items-center` (필터 버튼)
- **trigger**: click
- **tips**:
  - 활성 필터는 배경색으로 구분됩니다
  - 필터를 변경해도 데이터는 실시간으로 업데이트됩니다
  - 미지급 항목의 합계는 별도로 표시될 수 있습니다
- **buttons**:
  - selector: `.filter-btn[data-status="pending"]`
  - label: 미지급
  - desc: 아직 지급되지 않은 지출만 표시합니다

### Step 3: 지출 정보 확인
- **title**: 각 지출 항목 상세 정보
- **text**: 테이블의 각 행은 하나의 지출을 나타냅니다. 요청일(지출 발생일), 내용(품명이나 설명), 금액, 거래처 계좌 정보, 작성자, 지급 예정일, 지급 상태(아이콘) 등이 표시됩니다. 금액은 우측 정렬로 표시되어 시각적으로 크게 보입니다. 계좌 정보는 클릭해도 복사되지 않으므로 오른쪽 액션 메뉴에서 복사 아이콘을 사용해야 합니다.
- **highlight**: `tbody tr` (지출 행)
- **trigger**: hover
- **tips**:
  - 금액은 원화(₩) 기준이며 천 단위 콤마로 표시됩니다
  - 지급 상태는 시각적 아이콘(원형)으로 한눈에 파악할 수 있습니다
  - 같은 날에 여러 지출이 있을 수 있습니다
- **buttons**: 없음 (정보 열람만)

### Step 4: 통계 모달 보기
- **title**: 월별 지출 통계 확인
- **text**: 우측 상단의 '📊 통계' 버튼을 클릭하면 현재 월의 지출 통계 모달이 열립니다. 총 지출액, 미지급액, 지급완료액, 카테고리별 지출 분석 등을 한눈에 볼 수 있습니다. 이 통계를 통해 병원의 재정 상황을 신속하게 파악할 수 있습니다.
- **highlight**: `button[onclick="openStatsModal()"]`
- **trigger**: click
- **tips**:
  - 통계는 현재 선택된 월 기준입니다
  - 통계 모달에서 원본 데이터로 돌아갈 수 있습니다
  - 미지급액이 높으면 곧 대금 지급을 준비해야 합니다
- **buttons**:
  - selector: `button[onclick="openStatsModal()"]`
  - label: 통계
  - desc: 월별 지출 통계를 확인합니다

### Step 5: 새 지출 요청 등록
- **title**: 새로운 지출 항목 추가하기
- **text**: 우측 상단의 파란 '+ 새 요청' 버튼을 클릭하면 새 지출 요청 모달이 열립니다. 요청일(필수), 내용(필수), 금액(필수), 계좌 정보(은행명, 계좌번호, 예금주), 지급 예정일, 메모 등을 입력합니다. 자주 사용하는 거래처는 템플릿으로 저장해두면 다음에 빠르게 입력할 수 있습니다.
- **highlight**: `button[onclick="openCreateModal()"]`
- **trigger**: click
- **tips**:
  - 요청일과 금액은 필수 입력입니다
  - 계좌 정보를 정확히 입력하면 지급 오류를 줄일 수 있습니다
  - 자주 사용하는 거래처는 템플릿으로 저장하세요
  - 메모에 특이 사항이나 발주 내역을 추가할 수 있습니다
- **buttons**:
  - selector: `button[onclick="openCreateModal()"]`
  - label: 새 요청
  - desc: 새로운 지출 항목을 등록합니다

### Step 6: 지출 수정 및 상태 변경
- **title**: 기존 지출 항목 관리하기
- **text**: 테이블의 각 행 오른쪽 액션 메뉴(세로 점 3개)를 클릭하면 수정, 삭제, 상태 변경 등의 옵션이 나타납니다. 지출 내용이나 계좌 정보를 수정할 수 있고, 지급이 완료되면 상태를 '지급완료'로 변경할 수 있습니다. 이렇게 상태를 변경하면 필터링에도 반영되고 통계에도 즉시 반영됩니다.
- **highlight**: `button[class*="action"]` (액션 메뉴)
- **trigger**: click
- **tips**:
  - 지급이 확인되면 상태를 즉시 변경하세요
  - 상태 변경은 되돌릴 수 있습니다
  - 삭제 전에 반드시 확인하세요
- **buttons**:
  - selector: `button[onclick*="editExpense"]`
  - label: 수정
  - desc: 지출 항목을 수정합니다
  - selector: `button[onclick*="markPaid"]`
  - label: 지급완료 표시
  - desc: 지출 상태를 지급완료로 변경합니다

### Step 7: 템플릿 저장 및 재사용
- **title**: 자주 사용하는 거래처를 템플릿으로 저장하기
- **text**: 새 요청 모달의 하단에 '이 내용을 즐겨찾기에 추가' 체크박스가 있습니다. 이를 체크하고 저장하면 은행명, 계좌번호, 예금주 정보가 템플릿으로 저장됩니다. 다음번에 같은 거래처에 지급할 때는 템플릿을 선택하면 계좌 정보가 자동으로 입력되어 시간을 절약할 수 있습니다.
- **highlight**: `#saveTemplateContainer`
- **trigger**: click
- **tips**:
  - 자주 거래하는 업체(임대료, 월급, 통신료 등)는 템플릿으로 저장하세요
  - 템플릿은 언제든 삭제할 수 있습니다
  - 템플릿을 사용해도 각 항목의 금액이나 날짜는 다시 입력해야 합니다
- **buttons**:
  - selector: `#saveTemplate`
  - label: 템플릿 저장
  - desc: 현재 거래처 정보를 템플릿으로 저장합니다
