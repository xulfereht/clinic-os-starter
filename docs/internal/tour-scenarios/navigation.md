# Navigation (/admin/settings/navigation) — 투어 시나리오

## 페이지 목적
상단 헤더와 모바일 메뉴의 구조를 드래그 앤 드롭으로 편집하고, 각 메뉴 항목의 유형(링크/드롭다운/동적)과 표시 여부를 관리합니다.

## 시나리오 (7 스텝)

### Step 1: 메뉴 관리 페이지 개요
- **title**: 메뉴 구조 관리
- **text**: 이 페이지에서는 웹사이트의 주요 메뉴를 시각적으로 편집합니다. 왼쪽에는 메뉴 계층 구조가, 오른쪽에는 선택한 메뉴의 상세 설정이 표시됩니다.
- **highlight**: `.grid.grid-cols-1.lg\\:grid-cols-3`
- **trigger**: none
- **tips**:
  - 메뉴는 트리 구조로 표시됨 (최대 2-3 레벨)
  - 각 메뉴는 아이콘으로 유형 표시 (🔗=링크, 📁=드롭다운, 🏥=동적)
  - 드래그 핸들(≡)을 사용해 순서 변경
- **buttons**: []

### Step 2: 새 메뉴 추가
- **title**: 새 메뉴 항목 추가
- **text**: 왼쪽 상단의 "+ 새 메뉴 추가" 버튼을 클릭하면 새 메뉴 항목이 목록에 추가되고, 오른쪽 패널에서 수정할 수 있습니다.
- **highlight**: `#add-item-btn`
- **trigger**: click
- **tips**:
  - 새 메뉴는 목록 맨 아래에 추가됨
  - 자동으로 선택되어 편집 패널이 열림
  - 기본값: "새 메뉴", 링크 유형, "/"
- **buttons**:
  - `#add-item-btn / 새 메뉴 추가 / 새 메뉴 항목을 목록에 추가`

### Step 3: 메뉴 항목 선택 및 수정
- **title**: 메뉴 항목 선택
- **text**: 메뉴 항목의 텍스트 부분을 클릭하면 오른쪽 편집 패널이 활성화됩니다. 선택된 항목은 파란색 배경으로 강조됩니다.
- **highlight**: `[data-edit-id]`
- **trigger**: click
- **tips**:
  - 한 번에 하나의 메뉴만 선택 가능
  - 선택된 항목만 편집 패널이 표시됨
  - 메뉴명 아래 경로와 자식 항목 개수 표시
- **buttons**:
  - `.tag-item / 메뉴 클릭 / 메뉴 항목 선택 및 편집 패널 열기`

### Step 4: 메뉴 정보 수정 (라벨, 유형, 경로)
- **title**: 메뉴 라벨과 유형 설정
- **text**: 오른쪽 패널에서 메뉴명(라벨)과 유형을 설정합니다. 링크면 경로를 입력하고, 드롭다운이면 하위 메뉴를 지정하며, 동적 프로그램이면 포함할 프로그램을 선택합니다.
- **highlight**: `#edit-panel`
- **trigger**: none
- **tips**:
  - 메뉴명은 다국어 키(nav.key) 또는 직접 입력 가능
  - 링크 유형: 경로 필드 필수
  - 드롭다운: 자동으로 폴더 아이콘(📁) 표시
  - 동적: 진료 프로그램 목록 자동 표시
  - 유형 변경하면 관련 필드가 자동 표시/숨김
- **buttons**:
  - `#edit-label / 메뉴명 / 다국어 키 또는 직접 입력`
  - `#edit-type / 유형 / 링크, 드롭다운, 동적 중 선택`
  - `#edit-path / 링크 경로 / /doctors, /programs 등`
  - `#page-selector / 페이지 빠른 선택 / 드롭다운에서 인기 페이지 선택`

### Step 5: 동적 프로그램 메뉴 설정
- **title**: 진료 프로그램 메뉴 설정
- **text**: 메뉴 유형을 "동적 프로그램"으로 선택하면, 표시할 프로그램을 선택할 수 있습니다. 선택한 프로그램만 메뉴 드롭다운에 나타나며, "'전체 보기' 링크 포함" 체크박스로 전체 보기 버튼 표시 여부를 결정합니다.
- **highlight**: `#field-programs`
- **trigger**: none
- **tips**:
  - 프로그램 체크박스는 스크롤 가능한 박스로 표시
  - 미선택 시 모든 프로그램이 메뉴에 표시됨
  - "전체 보기" 링크는 /programs로 연결
- **buttons**:
  - `#program-checkboxes / 프로그램 선택 / 메뉴에 표시할 프로그램 선택`
  - `#edit-show-view-all / 전체 보기 링크 / 포함 여부 체크`

### Step 6: 메뉴 순서 변경 (드래그, 버튼)
- **title**: 메뉴 순서 및 위치 변경
- **text**: 메뉴 항목 왼쪽의 드래그 핸들(≡)을 드래그하여 순서를 변경하거나, 오른쪽의 화살표 버튼으로 조정합니다. ⬆️/⬇️로 위/아래로 이동, ➡️로 하위 메뉴로 만들기, ⬅️로 상위 메뉴로 꺼내기가 가능합니다.
- **highlight**: `.drag-handle`
- **trigger**: click/drag
- **tips**:
  - 드래그: 부드러운 애니메이션으로 실시간 재배열
  - 버튼: 더 정밀한 제어 가능
  - ➡️: 이전 항목을 부모로 하여 하위 메뉴로 변환
  - ⬅️: 부모 메뉴의 바로 다음으로 이동 (루트 레벨이면 비활성)
  - 변경 후 미리보기는 자동 업데이트
- **buttons**:
  - `.drag-handle / 드래그 / 메뉴 순서 변경`
  - `[data-action="up"] / 위로 / 메뉴를 한 칸 위로 이동`
  - `[data-action="down"] / 아래로 / 메뉴를 한 칸 아래로 이동`
  - `[data-action="in"] / 하위 메뉴로 / 이전 항목의 자식으로 만들기`
  - `[data-action="out"] / 상위 메뉴로 / 부모 메뉴에서 꺼내기`

### Step 7: 메뉴 표시 여부 및 저장
- **title**: 메뉴 표시 여부 설정 및 저장
- **text**: 오른쪽 패널에서 "메뉴에 표시" 체크박스로 메뉴 표시 여부를 제어합니다. 비활성 메뉴는 웹사이트에 나타나지 않습니다. 모든 수정이 끝나면 왼쪽 상단의 "변경사항 저장" 버튼을 클릭하여 저장합니다.
- **highlight**: `#save-btn`
- **trigger**: click
- **tips**:
  - 표시 여부만 변경해도 저장 필요
  - 저장 전 변경 사항은 메모리에만 존재
  - 브라우저 새로고침 시 저장되지 않은 변경사항 손실
  - 저장 후 확인 대화창이 표시됨
  - 저장에 실패하면 에러 메시지 표시
- **buttons**:
  - `#edit-visible / 메뉴에 표시 / 웹사이트에 표시 여부 체크`
  - `#save-btn / 변경사항 저장 / 모든 메뉴 설정을 서버에 저장`
  - `#delete-item-btn / 삭제 / 선택한 메뉴 항목 제거`
  - `#close-panel / 패널 닫기 / 편집 패널 숨기기`
