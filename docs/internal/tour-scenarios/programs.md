# 프로그램 관리 (/admin/programs) — 투어 시나리오

## 페이지 목적
진료 프로그램의 기본 정보, 진료 과목, 연동된 의료진을 관리하고, 홈페이지에 표시되는 순서를 드래그로 조정합니다. 각 프로그램의 섹션(콘텐츠 블록)은 별도 편집기로 관리합니다.

---

## 시나리오 (18 스텝)

### Step 1: 페이지 입장 및 목적 이해
- **title**: 프로그램 관리 페이지 소개
- **text**: 진료 프로그램 목록이 표 형식으로 표시됩니다. 각 프로그램은 아이콘, 이름, 설명, 공개 여부, 연동 상태(의료진/진단/섹션)를 보여줍니다. 순서는 위에서 아래로 홈페이지에 표시됩니다.
- **highlight**: `.max-w-5xl` 컨테이너
- **trigger**: confirm
- **tips**:
  - 프로그램 = 한의원이 제공하는 치료 서비스 단위 (예: 비만 치료, 피부 질환, 통증 관리)
  - 각 프로그램은 고유 ID를 가지며, 같은 ID의 프로그램은 중복 생성 불가

### Step 2: 상단 헤더 영역
- **title**: 페이지 헤더 및 매뉴얼 링크
- **text**: 페이지 제목 "프로그램 관리"와 설명이 표시되고, "매뉴얼 바로가기" 링크(HQ 가이드)를 누르면 상세 가이드를 볼 수 있습니다.
- **highlight**: `PageHeader` 컴포넌트 영역
- **trigger**: confirm
- **tips**:
  - 매뉴얼은 외부 HQ 페이지이므로 새로운 탭에서 열립니다
  - 전체 기능에 대한 설명은 매뉴얼을 참고하세요

### Step 3: 새 프로그램 추가 버튼
- **title**: 새로운 프로그램 생성
- **text**: 우상단의 파란 버튼 "+ 새 프로그램"을 클릭하면 새 프로그램 생성 페이지로 이동합니다. 여기서 프로그램명, 설명, 아이콘 등을 입력합니다.
- **highlight**: `a[href="/admin/programs/new"]`
- **trigger**: click
- **buttons**:
  - selector: `a[href="/admin/programs/new"]`
  - label: "+ 새 프로그램"
  - desc: "새 프로그램을 추가합니다"
- **tips**:
  - 프로그램명은 한의원의 주요 치료 프로그램 이름입니다 (예: "다이어트 전문 프로그램")
  - 한 번에 하나의 프로그램만 생성합니다

### Step 4: 프로그램 테이블 헤더 열 이해
- **title**: 테이블 헤더 구조
- **text**: 테이블 헤더에는 6개 열이 있습니다: 순서 조정 핸들, 공개 여부, 프로그램명, 연동 상태, 진료 과목, 관리. 각 열의 역할을 이해하면 프로그램 관리가 수월합니다.
- **highlight**: `thead tr`
- **trigger**: confirm
- **tips**:
  - 순서 조정 = 드래그 가능 (마우스로 잡아서 위아래로 이동)
  - 공개 여부 = 토글 스위치 (홈페이지 표시 여부)
  - 연동 상태 = 의료진, 진단, 섹션이 제대로 설정됐는지 한눈에 확인

### Step 5: 드래그 핸들 (순서 조정)
- **title**: 프로그램 순서를 마우스로 조정
- **text**: 첫 번째 열의 ≡ 아이콘(드래그 핸들)을 마우스로 잡고 위아래로 드래그하면 프로그램 순서가 바뀝니다. 드래그를 놓으면 자동으로 DB에 저장됩니다.
- **highlight**: `.drag-handle`
- **trigger**: click
- **tips**:
  - 마우스 커서가 "move"로 변하면 드래그 가능 상태입니다
  - 드래그 중 행이 투명해지는 것은 정상입니다
  - 순서 변경은 즉시 반영되므로 새로고침할 필요 없습니다

### Step 6: 공개 여부 토글 스위치
- **title**: 홈페이지 공개 여부 설정
- **text**: 두 번째 열의 토글 스위치를 클릭하면 프로그램의 공개 여부(is_visible)가 즉시 변경됩니다. ON = 홈페이지에 표시, OFF = 숨김.
- **highlight**: `.visibility-toggle`
- **trigger**: click
- **buttons**:
  - selector: `.visibility-toggle`
  - label: "공개 여부 토글"
  - desc: "프로그램을 홈페이지에 표시하거나 숨깁니다"
- **tips**:
  - 청색 = 공개 (ON), 회색 = 비공개 (OFF)
  - 토글 변경 후 "상태 변경이 저장되었습니다" 토스트 메시지 확인
  - 실패하면 "상태 변경에 실패했습니다" 오류 표시

### Step 7: 프로그램 아이콘 및 정보
- **title**: 프로그램 이름, 설명, ID 확인
- **text**: 세 번째 열에 프로그램 아이콘(이모지), 굵은 프로그램명, 설명 한 줄(회색 작은 글씨), 프로그램 ID(회색 더 작은 글씨, 고정값)가 표시됩니다.
- **highlight**: `.program-row td:nth-child(3)`
- **trigger**: confirm
- **tips**:
  - ID는 시스템이 자동 생성하며, 나중에 변경할 수 없습니다
  - 설명은 한 줄로 자동 생략됩니다 (말줄임표)
  - 각 프로그램마다 고유의 이모지 아이콘이 할당됩니다

### Step 8: 연동 상태 배지 해석
- **title**: 의료진, 진단, 섹션 연동 상태 확인
- **text**: 네 번째 열에 3개 배지가 표시됩니다: "의료진 O/X" (의료진 연동 여부), "진단 O/X" (MiniDiagnosis 섹션 포함 여부), "섹션 N" (섹션 개수). 녹색/보라색/파란색 = 정상, 회색 = 미설정.
- **highlight**: `.program-row td:nth-child(4)`
- **trigger**: confirm
- **tips**:
  - 의료진 X = 이 프로그램을 담당할 의사/대체의학 전문가가 미지정
  - 진단 X = 간단한 진단 도구(MiniDiagnosis)가 없음 (선택 사항)
  - 섹션 0 = 프로그램 상세 페이지의 콘텐츠 블록이 없음 (비추천)

### Step 9: 진료 과목 카운트
- **title**: 프로그램이 다루는 진료 과목 개수
- **text**: 다섯 번째 열에 "N개 항목" 또는 "-" (미설정)이 표시됩니다. 이는 이 프로그램이 다루는 질환/치료 항목의 개수입니다 (예: "5개 항목" = 비만, 산후비만, 소아비만 등 5가지).
- **highlight**: `.program-row td:nth-child(5)`
- **trigger**: confirm
- **tips**:
  - 진료 과목은 프로그램 상세 정보 모달에서 쉼표로 구분된 텍스트로 관리됩니다
  - 화면에는 개수만 표시되며, 구체적 목록은 모달에서 확인

### Step 10: 프로그램 행을 클릭하여 모달 열기
- **title**: 프로그램 정보 수정 모달 오픈
- **text**: 프로그램 행의 아무 곳이나 클릭하면 (드래그 핸들, 토글, 섹션 관리 버튼 제외) "프로그램 정보 수정" 모달이 팝업으로 열립니다. 이 모달에서 프로그램의 상세 정보를 편집합니다.
- **highlight**: `.program-row`
- **trigger**: click
- **buttons**:
  - selector: `.program-row`
  - label: "프로그램 행 클릭"
  - desc: "정보 수정 모달을 엽니다"
- **tips**:
  - 모달이 열리면 배경이 어두워지고 중앙에 창이 표시됩니다
  - Escape 키를 누르거나 배경을 클릭하면 모달이 닫힙니다

### Step 11: 모달 내 프로그램 ID 필드 (읽기 전용)
- **title**: 프로그램 ID 확인 (수정 불가)
- **text**: 모달 상단의 첫 번째 필드는 "ID"로, 회색 배경에 프로그램의 고유 ID가 표시됩니다. 이 필드는 읽기 전용이며 수정할 수 없습니다.
- **highlight**: `#modal-id-input`
- **trigger**: confirm
- **tips**:
  - ID는 프로그램 생성 시 자동 결정되며, 변경 불가능합니다
  - 예: "diet", "pain", "skin", "woman", "mental", "digestion"

### Step 12: 모달 내 프로그램명 편집
- **title**: 프로그램 이름 수정
- **text**: "프로그램명" 입력 필드에 프로그램의 한글 이름을 입력합니다 (예: "비만 치료 전문 프로그램"). 이 이름이 홈페이지에 표시됩니다.
- **highlight**: `#modal-title-input`
- **trigger**: input
- **buttons**:
  - selector: `#modal-title-input`
  - label: "프로그램명"
  - desc: "한글 프로그램 이름을 입력하거나 수정합니다"
- **tips**:
  - 필수 필드입니다 (빈 값으로 저장 불가)
  - 한글, 영문, 숫자 모두 가능합니다
  - 약 50자 이내로 작성하는 것이 좋습니다

### Step 13: 모달 내 설명 편집
- **title**: 프로그램 설명 작성
- **text**: "설명" 텍스트 영역에는 프로그램의 한 줄 설명을 입력합니다. 이 설명은 프로그램 목록 페이지의 작은 글씨로 표시됩니다.
- **highlight**: `#modal-desc-input`
- **trigger**: input
- **buttons**:
  - selector: `#modal-desc-input`
  - label: "설명"
  - desc: "프로그램을 간단히 설명하는 한 줄 텍스트"
- **tips**:
  - 선택 사항입니다 (빈 값 허용)
  - 예: "현대인의 스트레스로 인한 통증을 근본적으로 치료합니다"
  - 약 100자 이내가 읽기 좋습니다

### Step 14: 모달 내 순서(Order) 편집
- **title**: 프로그램 표시 순서 설정
- **text**: "순서 (낮은 숫자일수록 먼저 표시)" 필드에 숫자를 입력합니다. 이 값이 작을수록 홈페이지에서 위쪽에 표시됩니다 (0, 1, 2, 3...). 드래그로도 변경 가능하지만 여기서 정확한 값을 입력할 수도 있습니다.
- **highlight**: `#modal-order-input`
- **trigger**: input
- **buttons**:
  - selector: `#modal-order-input`
  - label: "순서"
  - desc: "정수값을 입력합니다 (낮을수록 먼저)"
- **tips**:
  - 숫자만 입력 가능합니다 (음수도 가능)
  - 같은 값을 여러 프로그램에 할당하면 DB의 제목 순(title ASC)으로 정렬됩니다
  - 드래그로 변경한 후 저장하면 자동으로 0부터 순차 재정렬됩니다

### Step 15: 모달 내 진료 과목 편집
- **title**: 프로그램이 다루는 질환/치료 항목 입력
- **text**: "진료 과목 (쉼표로 구분)" 텍스트 영역에는 이 프로그램이 다루는 질환이나 증상을 쉼표로 구분하여 입력합니다. 예: "비만, 산후비만, 소아비만".
- **highlight**: `#modal-conditions-input`
- **trigger**: input
- **buttons**:
  - selector: `#modal-conditions-input`
  - label: "진료 과목"
  - desc: "쉼표로 구분된 질환/치료 항목 목록"
- **tips**:
  - 선택 사항입니다 (빈 값 허용)
  - 각 항목 앞뒤의 공백은 자동으로 제거됩니다
  - 예: "비만, 산후비만, 소아비만" 또는 "비만,산후비만,소아비만" 둘 다 가능
  - 이 데이터는 홈페이지 프로그램 목록에 "N개 항목"으로 표시됩니다

### Step 16: 모달 내 의료진 선택 (체크박스)
- **title**: 프로그램을 담당할 의료진 지정
- **text**: "연동된 의료진" 섹션에 병원 시스템에 등록된 모든 의료진(의사, 한의사, 간호사 등)의 체크박스가 2열 그리드로 표시됩니다. 이 프로그램을 담당할 의료진을 선택하세요.
- **highlight**: `#modal-doctors-list`
- **trigger**: click
- **buttons**:
  - selector: `#modal-doctors-list input[type="checkbox"]`
  - label: "의료진 선택"
  - desc: "프로그램을 담당할 의료진을 선택합니다"
- **tips**:
  - 여러 명을 선택할 수 있습니다
  - 선택된 체크박스는 파란 체크 표시가 됩니다
  - 의료진이 없으면 모달의 테이블 헤더에서 "의료진 X" 배지가 표시됩니다

### Step 17: 모달 내 저장, 섹션 편집, 삭제 버튼
- **title**: 모달의 하단 액션 버튼
- **text**: 모달 하단에 3개 버튼이 있습니다: 파란색 "저장하기" (정보 저장), 흰색 "섹션 편집기로 이동" (프로그램 상세 페이지 콘텐츠 편집), 빨간색 "삭제" (프로그램 완전 삭제).
- **highlight**: `#programEditForm .bg-slate-50`
- **trigger**: confirm
- **buttons**:
  - selector: `#saveBtn`
  - label: "저장하기"
  - desc: "현재 정보를 DB에 저장합니다"
  - selector: `#modal-edit-link`
  - label: "섹션 편집기로 이동"
  - desc: "프로그램 상세 페이지의 섹션(콘텐츠 블록)을 편집하는 페이지로 이동합니다"
  - selector: `#deleteBtn`
  - label: "삭제"
  - desc: "이 프로그램을 완전히 삭제합니다 (확인 필요)"
- **tips**:
  - "저장하기" 클릭 시 "저장 중..." 표시 후 완료하면 페이지 새로고침
  - "섹션 편집기로 이동" = /admin/programs/{id} 페이지로 이동
  - "삭제"는 확인 창이 뜬 후 "복구하려면 관리자에게 문의" 메시지 표시

### Step 18: 모달 저장 후 재입장 또는 닫기
- **title**: 모달 닫기 및 목록으로 돌아가기
- **text**: 저장 완료 후 페이지가 자동으로 새로고침되고 모달이 닫혀 목록으로 돌아옵니다. 변경사항은 즉시 테이블에 반영됩니다. X 버튼이나 배경 클릭으로도 모달을 닫을 수 있습니다.
- **highlight**: `#closeInfoModalBtn`, `#infoModalBackdrop`
- **trigger**: click
- **buttons**:
  - selector: `#closeInfoModalBtn`
  - label: "닫기 (X)"
  - desc: "모달을 닫고 목록으로 돌아갑니다"
- **tips**:
  - 저장하지 않고 닫으면 변경사항이 반영되지 않습니다
  - 배경 클릭으로도 닫을 수 있습니다
  - Escape 키로도 닫힙니다

---

## 통합 워크플로우 예시

1. **신규 프로그램 추가**: "+ 새 프로그램" → 기본정보 입력 → 저장 → 목록으로 돌아옴
2. **프로그램 순서 조정**: 드래그 핸들 마우스로 잡기 → 위아래 이동 → 놓으면 자동 저장
3. **공개/비공개 전환**: 토글 스위치 클릭 → 즉시 저장됨
4. **프로그램 상세 편집**: 행 클릭 → 모달 열림 → 각 필드 수정 → "저장하기" 클릭 → 저장됨
5. **프로그램 삭제**: 행 클릭 → 모달 열림 → "삭제" 클릭 → 확인 창 → 삭제 완료

| 상황 | 해결 방법 |
