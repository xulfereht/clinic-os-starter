# 검사 라이브러리 (/admin/surveys) — 투어 시나리오

## 페이지 목적
설문/검사도구를 효율적으로 관리하고 환자에게 배정할 수 있도록 하는 검사 라이브러리입니다.

## 시나리오 (6 스텝)

### Step 1: 페이지 진입 및 검사 목록 확인
- **title**: 검사 라이브러리 소개
- **text**: 이 페이지에서는 한의원에서 사용할 수 있는 검사 도구들을 한눈에 볼 수 있습니다. 시스템 기본 검사, 설치된 플러그인 검사도구, 그리고 직접 만든 커스텀 검사까지 모두 관리합니다. 표에서 각 검사의 유형, 태그, 연동 프로그램을 확인할 수 있습니다.
- **highlight**: `.bg-white.rounded-xl.border.border-slate-200.shadow-sm` (표 전체)
- **trigger**: click
- **tips**:
  - 각 행의 색상 배지로 검사 출처를 구분합니다
  - 패널(패널), 시스템(기본), 플러그인(스토어), 코어 등 4가지 유형이 있습니다
- **buttons**: 
  - selector: `.filter-btn`
  - label: 프로그램 필터 버튼
  - desc: 프로그램별 검사를 필터링합니다

### Step 2: 프로그램별 필터 적용
- **title**: 필터로 검사 찾기
- **text**: 상단의 필터 버튼들을 이용해 특정 프로그램에 연동된 검사만 볼 수 있습니다. 예를 들어 '소화기' 버튼을 누르면 소화기 프로그램에서 사용하는 검사들만 표시됩니다. '전체' 버튼은 모든 검사를 다시 보여줍니다. 검사도구 버튼을 누르면 특수 검사 도구들만 필터링됩니다.
- **highlight**: `.filter-btn.px-4.py-2.rounded-lg.text-sm.font-bold` (필터 버튼 영역)
- **trigger**: click
- **tips**:
  - 활성 필터는 어두운 배경색으로 표시됩니다
  - 필터를 적용해도 테이블의 행은 숨겨지기만 하고 삭제되지 않습니다
  - 여러 프로그램을 한 번에 필터링할 수는 없습니다
- **buttons**:
  - selector: `[data-program="/programs/digestive"]`
  - label: 소화기
  - desc: 소화기 프로그램 검사만 표시

### Step 3: 검사 상세 정보 열람
- **title**: 검사 정보 확인
- **text**: 테이블의 각 행은 한 개의 검사를 나타냅니다. 가장 왼쪽 열에는 검사명과 설명이, 두 번째 열에는 연동된 프로그램이, 세 번째 열에는 공개 여부(마케팅 또는 원내용)가 표시됩니다. 마지막에는 관련 태그들을 볼 수 있습니다.
- **highlight**: `tbody tr:first-child`
- **trigger**: hover
- **tips**:
  - 공개(마케팅) 검사는 환자가 웹에서 자가진단으로 이용할 수 있습니다
  - 원내용 검사는 의료진만 사용합니다
  - 태그는 검사의 카테고리나 특성을 나타냅니다
  - 태그가 3개 이상이면 "+N" 형태로 축약됩니다
- **buttons**:
  - selector: `.px-2.py-1.bg-slate-100.text-slate-600.text-xs.font-bold.rounded.border`
  - label: 연동 프로그램 배지
  - desc: 어떤 프로그램과 연결되었는지 표시

### Step 4: 공개 검사 QR 코드 생성
- **title**: 환자용 QR 코드 만들기
- **text**: 공개(마케팅) 검사가 있는 행에 마우스를 올리면 오른쪽에 QR 아이콘이 나타납니다. 이를 클릭하면 환자들이 모바일에서 스캔할 수 있는 QR 코드를 생성합니다. QR 코드는 환자가 익명으로 접근할 수 있는 검사 페이지로 연결됩니다. 생성된 코드는 인쇄하거나 다운로드해서 대기실에 부착할 수 있습니다.
- **highlight**: `button[onclick*="openQR"]`
- **trigger**: click
- **tips**:
  - 원내용 검사는 QR 생성 버튼이 보이지 않습니다
  - QR 코드는 clinic-os-hq.pages.dev 도메인으로 생성됩니다
  - 모바일과 PC 모두에서 스캔 가능합니다
- **buttons**:
  - selector: `#qrDownload`
  - label: 다운로드
  - desc: PNG 형식으로 QR 코드를 다운로드합니다

### Step 5: 커스텀 검사 수정
- **title**: 자신의 검사 수정하기
- **text**: 직접 만든 검사(데이터베이스 출처)는 수정할 수 있습니다. 수정하려는 행에 마우스를 올리면 오른쪽에 '수정' 버튼이 나타납니다. 이를 클릭하면 모달 창이 열려 검사의 이름, 설명, 연동 프로그램, 태그, 공개 여부 등을 변경할 수 있습니다. 시스템 기본 검사나 플러그인 검사는 수정 불가능하며 "수정불가" 텍스트만 표시됩니다.
- **highlight**: `button[onclick*="openEditModal"]`
- **trigger**: click
- **tips**:
  - 수정 모달에서 검사 정보를 모두 수정할 수 있습니다
  - URL 패턴은 시스템이 검사에 접근하기 위해 사용하므로 신중하게 수정해야 합니다
  - 태그는 쉼표로 구분해 여러 개를 입력할 수 있습니다
- **buttons**:
  - selector: `#editForm button[type="submit"]`
  - label: 저장하기
  - desc: 수정한 검사 정보를 저장합니다

### Step 6: 새 검사 등록
- **title**: 새로운 커스텀 검사 추가하기
- **text**: 우측 상단의 파란 '새 검사 등록' 버튼을 클릭하면 새 검사를 등록할 수 있습니다. 검사명, 설명, URL 패턴, 연동 프로그램, 태그, 공개 여부 등을 입력해 한의원만의 특화된 검사를 만들 수 있습니다. 이 검사는 자동으로 "커스텀" 태그가 붙고 데이터베이스에 저장되어 관리 페이지에서 언제든 수정할 수 있습니다.
- **highlight**: `a[href="/admin/surveys/new"]`
- **trigger**: click
- **tips**:
  - 새 검사 등록은 별도 페이지에서 진행됩니다
  - URL 패턴은 시스템이 검사 결과를 저장할 때 사용됩니다
  - 처음부터 공개로 설정하면 바로 환자들이 이용할 수 있습니다
  - 테스트 후 공개 여부를 결정하는 것이 좋습니다
- **buttons**:
  - selector: `button[type="submit"]`
  - label: 등록하기
  - desc: 새 검사를 시스템에 추가합니다
