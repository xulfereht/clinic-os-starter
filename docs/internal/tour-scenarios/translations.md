# 다국어 번역 관리 (/admin/translations) — 투어 시나리오

## 페이지 목적
사이트의 콘텐츠(페이지, 프로그램, FAQ, 블로그)를 다국어로 번역하고, 언어별 번역 진행도를 관리하는 페이지

## 시나리오 (7 스텝)

### Step 1: 번역 관리 페이지 진입
- **title**: 다국어 번역 관리 개요
- **text**: 이 페이지는 한국어 콘텐츠를 다국어로 번역하고 관리하는 곳입니다. 상단에는 페이지 제목과 설명이 있으며, 아래로 빠른 통계 카드들이 표시됩니다: 지원 언어 수, UI 번역 관리, 콘텐츠별 번역 진행도(페이지, 프로그램, FAQ, 블로그). 각 섹션별로 번역 현황과 진행도를 시각적으로 확인할 수 있습니다.
- **highlight**: `.max-w-7xl.mx-auto`
- **trigger**: confirm
- **tips**:
  - 기본 언어는 한국어(ko)
  - 지원 언어: 영어(en), 일본어(ja), 중국어(zh-hans), 베트남어(vi) 등
  - 각 콘텐츠 유형별로 독립적으로 번역 관리

### Step 2: 빠른 통계 카드
- **title**: 번역 진행도 통계 확인
- **text**: 페이지 상단의 통계 카드 섹션(그리드 레이아웃)에는 다음 정보들이 표시됩니다: 1)🌍 지원 언어 (총 개수), 2)🔤 UI 번역 (링크), 3)📄 페이지 번역 (진행도), 4)📚 프로그램 번역 (진행도), 5)❓ FAQ 번역 (진행도), 6)📰 블로그 번역 (진행도). 각 카드는 색상으로 진행도를 시각화합니다.
- **highlight**: `.grid.grid-cols-2.md\\:grid-cols-6.gap-4`
- **trigger**: confirm
- **tips**:
  - 지원 언어는 활성화된 언어만 포함
  - UI 번역은 별도 페이지에서 관리
  - 진행도는 퍼센트로 표시됨
  - 링크를 클릭하면 해당 항목의 번역 행렬로 이동

### Step 3: UI 번역 링크
- **title**: UI 번역 관리 페이지 접근
- **text**: '🔤 UI 번역' 카드를 클릭하면 UI 요소(버튼, 메뉴, 메시지 등)의 다국어 번역을 관리하는 별도 페이지로 이동합니다. 이 페이지에서는 각 UI 문자열을 언어별로 번역할 수 있습니다.
- **highlight**: `a[href="/admin/translations/ui"]`
- **trigger**: click
- **tips**:
  - UI 번역은 콘텐츠 번역과 별도로 관리됨
  - 우선순위가 높음 (사용자가 먼저 접하는 요소)
  - 번역 메모리 (TM) 지원 가능

### Step 4: 콘텐츠 번역 행렬 - 페이지
- **title**: 페이지 번역 행렬 확인
- **text**: 통계 카드 아래로는 '📄 페이지', '📚 프로그램', '❓ FAQ', '📰 블로그' 섹션들이 표시됩니다. 각 섹션은 번역 행렬 형태로 구성되며, 세로 축은 콘텐츠(페이지/프로그램/FAQ/블로그), 가로 축은 언어(en, ja, zh-hans, vi 등)입니다. 각 셀에는 번역 상태(✅ Published, 📝 Draft, ⚪ Missing)가 색상 배지로 표시됩니다.
- **highlight**: `.grid.gap-6 > div`
- **trigger**: confirm
- **tips**:
  - 페이지 행렬: 발행된 페이지와 번역 상태 매칭
  - 프로그램 행렬: 모든 프로그램과 번역 상태
  - FAQ 행렬: 발행된 FAQ 항목과 번역 상태
  - 블로그 행렬: 발행된 블로그 포스트와 번역 상태

### Step 5: 번역 행렬 셀 상태
- **title**: 번역 상태 확인 및 작업
- **text**: 행렬의 각 셀은 다음 상태 중 하나를 표시합니다: 1)✅ Published (초록색) - 완성된 번역, 2)📝 Draft (노랑색) - 번역 진행 중, 3)⚪ Missing (회색) - 아직 시작하지 않음. 셀을 클릭하면 해당 콘텐츠의 번역 편집 페이지로 이동합니다. 각 언어 칼럼의 상단에는 평균 진행도가 퍼센트로 표시됩니다.
- **highlight**: `[class*="bg-"][class*="text-"]`
- **trigger**: click
- **tips**:
  - 초록색(✅): 완성된 번역이므로 검수만 필요
  - 노란색(📝): 번역 진행 중이므로 완성 필요
  - 회색(⚪): 번역 미시작이므로 우선순위 높음
  - 클릭하면 해당 번역 편집 페이지로 이동

### Step 6: 언어별 진행도 바
- **title**: 각 언어별 번역 진행도
- **text**: 각 콘텐츠 섹션(페이지, 프로그램, FAQ, 블로그)의 상단에는 언어별 평균 진행도가 표시됩니다. 진행도는 프로그레스 바와 퍼센트로 표시되며, 각 언어별로 완료된 번역 개수와 전체 필요 개수가 표시됩니다(예: 8/12). 이를 통해 각 언어별 번역 작업의 우선순위를 결정할 수 있습니다.
- **highlight**: `.grid.grid-cols-4.gap-4`
- **trigger**: confirm
- **tips**:
  - 진행도는 Published/Draft 모두 포함
  - 100% 도달 시 해당 언어로 완전 지원
  - 각 언어의 진행도를 비교하여 작업 우선순위 결정
  - 최우선 언어(예: 영어)부터 100% 달성 권장

### Step 7: 대량 번역 및 발행
- **title**: 번역 작업 흐름
- **text**: 번역 작업 흐름은 다음과 같습니다: 1)Missing(⚪) 상태의 콘텐츠 선택, 2)각 언어별로 번역 편집 페이지로 이동, 3)번역 입력 및 Draft(📝) 상태로 저장, 4)검수 후 Published(✅) 상태로 발행. 대량 발행을 위해서는 번역 관리자가 여러 항목을 일괄 처리할 수 있는 도구를 사용합니다.
- **highlight**: `.grid.gap-6`
- **trigger**: confirm
- **tips**:
  - 우선 높은 콘텐츠(FAQ, 주요 페이지)부터 번역 시작
  - Draft 상태로 저장하여 검수 기간 확보
  - 검수 후 Published로 발행하여 공개
  - 번역 메모리 활용으로 중복 번역 시간 감소
