# 휴지통 (/admin/trash) — 투어 시나리오

## 페이지 목적
삭제된 환자, 리드(문의), 예약, 접수를 조회하고 복구하는 페이지

## 시나리오 (8 스텝)

### Step 1: 휴지통 페이지 진입 및 헤더
- **title**: 휴지통 페이지 개요
- **text**: 이 페이지는 실수로 삭제한 환자, 문의, 예약, 접수 정보를 조회하고 복구할 수 있는 곳입니다. 상단 왼쪽에는 페이지 제목과 삭제된 항목의 총 개수가 표시되며, 우측 상단에는 대시보드로 돌아가는 링크가 있습니다.
- **highlight**: `.max-w-7xl.mx-auto`
- **trigger**: confirm
- **tips**:
  - 삭제 항목 개수는 실시간으로 업데이트됨
  - 4가지 카테고리별로 분류되어 있음

### Step 2: 검색 입력창
- **title**: 삭제된 항목 검색하기
- **text**: 상단 검색창(#searchInput)을 사용하여 이름이나 연락처로 삭제된 항목을 검색할 수 있습니다. 입력 시 자동으로 각 카테고리별 항목이 필터링되며, 검색 결과의 개수도 실시간으로 업데이트됩니다.
- **highlight**: `#searchInput`
- **trigger**: input
- **tips**:
  - 환자 이름, 연락처로 검색 가능
  - 리드의 이름, 연락처로 검색 가능
  - 검색은 300ms 디바운싱이 적용됨
  - 검색 결과는 각 탭별로 개수 표시

### Step 3: 카테고리 탭 전환
- **title**: 탭을 이용한 카테고리 전환
- **text**: 페이지 중단의 탭 버튼들(환자, 문의, 예약, 접수)을 클릭하여 카테고리를 전환할 수 있습니다. 각 탭 옆에는 해당 카테고리의 삭제된 항목 개수가 괄호 안에 표시됩니다. 탭을 클릭하면 해당 카테고리의 테이블이 표시됩니다.
- **highlight**: `.tab-btn`
- **trigger**: click
- **buttons**:
  - selector: `.tab-btn[data-tab="patients"]`
  - label: "환자"
  - desc: 삭제된 환자 목록 표시
  - selector: `.tab-btn[data-tab="leads"]`
  - label: "문의"
  - desc: 삭제된 문의 목록 표시
  - selector: `.tab-btn[data-tab="reservations"]`
  - label: "예약"
  - desc: 삭제된 예약 목록 표시
  - selector: `.tab-btn[data-tab="intakes"]`
  - label: "접수"
  - desc: 삭제된 접수 목록 표시
- **tips**:
  - 각 탭은 색상 변화로 활성 상태를 표시함
  - 탭 전환 시 검색 필터가 유지됨
  - 페이지네이션도 탭별로 독립적으로 작동

### Step 4: 환자 탭 - 테이블 구성
- **title**: 환자 삭제 이력 조회
- **text**: 환자 탭의 테이블에는 삭제일, 이름, 연락처, 생년월일, 복구 버튼이 표시됩니다. 각 행의 배경은 호버 시 색상이 변합니다. 삭제일은 빨간색으로 강조 표시되어 최근 삭제 항목을 쉽게 파악할 수 있습니다.
- **highlight**: `#patients-tbody`
- **trigger**: confirm
- **tips**:
  - 테이블은 삭제일 기준 역순(최신순)으로 정렬됨
  - 페이지당 10개씩 표시됨
  - 각 행을 클릭하면 복구 버튼이 강조됨

### Step 5: 문의/리드 탭 - 테이블 구성
- **title**: 삭제된 문의 조회
- **text**: 문의 탭의 테이블에는 삭제일, 이름(익명 표시 가능), 연락처, 채널(전화/카카오톡 등), 상태, 복구 버튼이 표시됩니다. 문의의 경우 채널 정보(예: 전화, 문자)와 상태(신규, 진행중, 완료)를 함께 확인할 수 있습니다.
- **highlight**: `#leads-tbody`
- **trigger**: confirm
- **tips**:
  - 이름이 없는 경우 '익명'으로 표시됨
  - 채널별로 문의의 출처를 구분 가능
  - 상태를 통해 처리 진행도를 확인할 수 있음

### Step 6: 예약 탭 - 테이블 구성
- **title**: 삭제된 예약 조회
- **text**: 예약 탭의 테이블에는 삭제일, 예약 날짜/시간, 환자 정보(이름과 연락처), 예약 상태, 복구 버튼이 표시됩니다. 환자 정보는 두 줄로 나뉘어 표시되며(이름/연락처), 예약 상태(예정/완료/취소)를 함께 확인할 수 있습니다.
- **highlight**: `#reservations-tbody`
- **trigger**: confirm
- **tips**:
  - 예약 날짜는 형식화되어 읽기 쉽게 표시됨
  - 환자 정보가 없으면 '알 수 없음'으로 표시됨
  - 예약 상태로 진행도를 파악할 수 있음

### Step 7: 접수 탭 - 테이블 구성
- **title**: 삭제된 접수 조회
- **text**: 접수 탭의 테이블에는 삭제일, 접수 날짜, 접수 ID(축약), 요약 제목, 복구 버튼이 표시됩니다. 접수 ID는 짧은 형태로 표시되며(UUID 처음 8자), 요약 제목을 통해 접수 내용을 빠르게 파악할 수 있습니다.
- **highlight**: `#intakes-tbody`
- **trigger**: confirm
- **tips**:
  - 접수 요약은 JSON 데이터에서 headline 필드를 추출함
  - 요약이 없으면 '-'으로 표시됨
  - 접수 ID는 전체를 확인하려면 요소에 호버하면 전체 ID가 보임

### Step 8: 복구 버튼 - 항목 복구하기
- **title**: 삭제된 항목 복구
- **text**: 각 행의 우측에 있는 초록색 '복구' 버튼을 클릭하면 해당 항목 복구 여부를 묻는 확인 대화창이 나타납니다. 확인하면 항목이 복구되고 휴지통에서 제거되며, 페이지가 자동으로 새로고침됩니다. 복구 실패 시 오류 메시지가 토스트 알림으로 표시됩니다.
- **highlight**: `.restore-patient-btn, .restore-lead-btn, .restore-reservation-btn, .restore-intake-btn`
- **trigger**: click
- **buttons**:
  - selector: `.restore-patient-btn`
  - label: "복구"
  - desc: 환자 복구하기
  - selector: `.restore-lead-btn`
  - label: "복구"
  - desc: 문의 복구하기
  - selector: `.restore-reservation-btn`
  - label: "복구"
  - desc: 예약 복구하기
  - selector: `.restore-intake-btn`
  - label: "복구"
  - desc: 접수 복구하기
- **tips**:
  - 복구 확인 대화창이 표시되며 취소 가능
  - 복구 후 자동으로 목록에서 제거됨
  - 연락처 중복 등의 이유로 복구 실패할 수도 있음
  - 페이지네이션은 복구 후 유지됨
