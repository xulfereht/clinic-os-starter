# 고객 문의 관리 (/admin/leads) — 투어 시나리오

## 페이지 목적
다양한 채널의 고객 문의를 상태별로 관리하고 영업 기회(리드)를 추적합니다.

## 시나리오 (28 스텝)

### Step 1: 페이지 진입 및 통계 카드
- **title**: 문의 현황 대시보드
- **text**: 이 페이지는 고객 문의 관리 시스템입니다. 상단의 3개 카드는 오늘의 중요한 지표를 보여줍니다. "오늘 신규 문의"(접수 건), "답변 대기중"(미처리 건), "오늘 처리 완료"(완료 건)입니다. 각 카드를 클릭하면 상세 목록이 모달로 열립니다.
- **highlight**: .grid-cols-1.md-grid-cols-3
- **trigger**: confirm
- **tips**:
  - 모든 시간은 KST(한국 표준시) 기준입니다
  - "답변 대기중": 상태가 'new' 또는 'pending'인 문의
  - 각 카드는 클릭 가능합니다 (상세 모달 열기)

### Step 2: 통계 카드 - 오늘 신규 문의 클릭
- **title**: 오늘 접수된 신규 문의 목록
- **text**: "오늘 신규 문의" 카드를 클릭하면 모달이 열려 오늘 자정 이후 접수된 모든 신규 문의(최대 20건)를 시간 역순으로 표시합니다.
- **highlight**: ".cursor-pointer.onclick*todayNew"
- **trigger**: click
- **tips**:
  - 자정 기준: 00:00:00 KST
  - 표시 항목: 이름, 연락처, 유입 채널, 상태, 접수 시간
  - 각 항목 클릭 시: 상세 페이지로 이동

### Step 3: 통계 카드 - 답변 대기중 클릭
- **title**: 미처리 문의 목록
- **text**: "답변 대기중" 카드를 클릭하면 현재 처리 상태가 'new' 또는 'pending'인 모든 미응답 문의(최대 20건)를 시간 역순으로 표시합니다.
- **highlight**: ".cursor-pointer.onclick*pending"
- **trigger**: click
- **tips**:
  - 대기중 포함: 신규 상태 + 상담 중단 상태
  - 우선순위: 가장 오래된 문의부터 처리 권장
  - 상태 변경: 상세 페이지에서 처리 가능

### Step 4: 통계 카드 - 오늘 처리 완료 클릭
- **title**: 오늘 처리된 문의 목록
- **text**: "오늘 처리 완료" 카드를 클릭하면 오늘 종료된 문의(closed_at >= 자정 KST)를 시간 역순으로 표시합니다(최대 20건).
- **highlight**: ".cursor-pointer.onclick*processed"
- **trigger**: click
- **tips**:
  - 종료 상태: closed, hold, cancelled, 예약, 결제 완료 등
  - 완료 기준: closed_at 필드 기준
  - 추적: 하루 처리량 KPI 확인 용도

### Step 5: 뷰 토글 - 카드보기
- **title**: 칸반 보드 방식 시각화
- **text**: 상단 토글에서 "카드보기"를 선택하면 문의를 3개 열(문의 발생 → 처리 중 → 완료/보류)로 구분하여 표시하는 칸반 보드 형식으로 전환됩니다. 카드는 드래그앤드롭으로 상태를 변경할 수 있습니다.
- **highlight**: "a:contains('카드보기')"
- **trigger**: click
- **tips**:
  - 기본 뷰: 카드보기 (보드 형식)
  - 각 열: 색상으로 구분 (빨강/파랑/초록)
  - 카드 수: 각 열의 헤더에 표시됩니다

### Step 6: 뷰 토글 - 리스트
- **title**: 테이블 형식 목록 보기
- **text**: "리스트"를 선택하면 문의 데이터를 테이블 형식으로 표시합니다. 열 헤더를 통해 이름, 연락처, 유입 경로, 구분, 상태 등을 한눈에 비교할 수 있습니다.
- **highlight**: "a:contains('리스트')"
- **trigger**: click
- **tips**:
  - 테이블 열: 이름, 연락처, 유입 경로, 구분, 상태, 접수 시간
  - 행 클릭: 상세 페이지로 이동
  - 필터 유지: 뷰 전환 시에도 필터 상태 유지됩니다

### Step 7: 문의 등록 버튼
- **title**: 수동 문의 추가
- **text**: "문의 등록" 파란색 버튼을 클릭하면 새 문의를 수동으로 생성할 수 있는 모달이 열립니다. 고객 정보, 유입 채널, 상담 내용 등을 입력할 수 있습니다.
- **highlight**: "#openCreateModalBtn"
- **trigger**: click
- **tips**:
  - 필수 입력: 이름, 연락처
  - 선택 입력: 채널, 상담 내용, 태그 등
  - 저장 시: 자동으로 새 문의로 추가되고 테이블/보드 새로고침

### Step 8: 검색 필드
- **title**: 이름 또는 연락처 검색
- **text**: 검색창에 고객명이나 전화번호를 입력하고 "필터 적용" 버튼을 클릭하면 매칭되는 문의만 조회됩니다.
- **highlight**: "input[name='q']"
- **trigger**: input
- **tips**:
  - 부분 검색 지원: "홍" 입력 시 "홍길동" 검색됨
  - 연락처: 번호의 일부만 입력 가능
  - 필터 적용: "필터 적용" 버튼 또는 엔터 키로 검색

### Step 9: 상태 필터
- **title**: 문의 상태별 조회
- **text**: "전체 상태", "대기/신규", "상담중", "완료", "보류" 드롭다운에서 선택하여 특정 상태의 문의만 필터링합니다.
- **highlight**: "select[name='status']"
- **trigger**: click
- **tips**:
  - 보드 뷰: 상태별 열이 자동으로 구성됨
  - 리스트 뷰: 상태 필터 적용 시 매칭 행만 표시
  - 필터 적용: 자동 또는 버튼 클릭 시 반영

### Step 10: 경로 필터 (유입 채널)
- **title**: 문의 유입 경로 필터링
- **text**: "전체 경로", "홈페이지", "온라인/채팅", "전화", "워크인" 드롭다운에서 선택하여 특정 채널의 문의만 조회합니다.
- **highlight**: "select[name='channel']"
- **trigger**: click
- **tips**:
  - 홈페이지: intake_form 채널
  - 온라인/채팅: chatgpt_app, naver_talk, web_widget 등
  - 전화: 전화 상담 문의
  - 워크인: 직접 방문 고객

### Step 11: 구분 필터 (신규/기존고객)
- **title**: 고객 분류별 조회
- **text**: "전체 구분", "신규", "기존고객" 드롭다운에서 선택하여 신규 고객 또는 기존 고객의 문의만 필터링합니다.
- **highlight**: "select[name='patient_type']"
- **trigger**: click
- **tips**:
  - 신규: 처음 문의하는 고객
  - 기존고객: 과거 내원 또는 예약 이력이 있는 고객
  - 자동 판단: 환자 DB와 연동하여 자동 분류

### Step 12: 태그 필터
- **title**: 태그로 문의 분류
- **text**: "전체 태그" 드롭다운에서 등록된 태그를 선택하면 해당 태그가 붙은 문의만 조회됩니다. 여러 태그를 관리하여 상담 유형을 분류할 수 있습니다.
- **highlight**: "select[name='tag']"
- **trigger**: click
- **tips**:
  - 태그 생성: 환자 정보 페이지에서 생성
  - 다중 태그: 하나의 문의에 여러 태그 가능
  - 필터: 하나의 태그로만 필터링 (다중 선택 불가)

### Step 13: 필터 적용 버튼
- **title**: 필터 조건 반영
- **text**: 각 필터를 선택한 후 "필터 적용" 버튼을 클릭하면 모든 필터 조건을 합쳐서 결과를 갱신합니다.
- **highlight**: "button:contains('필터 적용')"
- **trigger**: click
- **tips**:
  - 복합 필터: 검색 + 상태 + 경로 + 태그 동시 적용 가능
  - 즉시 반영: 버튼 클릭 시 테이블/보드 갱신

### Step 14: 필터 초기화 링크
- **title**: 모든 필터 제거
- **text**: 필터가 적용 중일 때 "필터 초기화" 링크를 클릭하면 모든 필터가 해제되고 전체 문의를 다시 조회합니다.
- **highlight**: "a:contains('필터 초기화')"
- **trigger**: click
- **tips**:
  - 조건부 표시: 필터가 적용되었을 때만 표시됨
  - 전체 초기화: 모든 필터 동시 제거
  - 상태 초기화: "전체 상태" 등으로 수동 초기화도 가능

### Step 15: 칸반 보드 - 문의 발생 열
- **title**: 신규/미처리 문의
- **text**: 보드 뷰의 첫 번째 빨간색 열 "문의 발생"에는 상태가 'new' 또는 'pending'인 모든 문의가 표시됩니다. 각 문의는 하나의 카드로 표시됩니다.
- **highlight**: ".kanban-column:first-child"
- **trigger**: confirm
- **tips**:
  - 색상: 빨강색 헤더
  - 카드 수: 열 헤더의 오른쪽 배지에 표시
  - 우선 처리: 이 열의 문의부터 상담 시작

### Step 16: 칸반 보드 - 처리 중 열
- **title**: 상담 진행 중 문의
- **text**: 보드 뷰의 두 번째 파란색 열 "처리 중"에는 상태가 'consulting'인 문의가 표시됩니다. 상담이 진행 중이거나 정보 수집 중인 문의가 모입니다.
- **highlight**: ".kanban-column:nth-child(2)"
- **trigger**: confirm
- **tips**:
  - 색상: 파란색 헤더
  - 상담 중: 직원과의 상담이 진행 중인 상태
  - 예약 대기: 진료 예약 또는 결제 대기 중

### Step 17: 칸반 보드 - 완료/보류 열
- **title**: 종료된 문의 및 보류 중인 문의
- **text**: 보드 뷰의 세 번째 초록색 열 "완료/보류"에는 상태가 'closed', 'hold', 'cancelled', 또는 각종 예약/결제 상태인 문의가 표시됩니다. 오늘 종료된 문의만 표시됩니다 (closed_at >= 자정).
- **highlight**: ".kanban-column:nth-child(3)"
- **trigger**: confirm
- **tips**:
  - 색상: 초록색 헤더
  - 완료: 예약 완료, 결제 완료, 취소 등
  - 보류: 고객 결정 대기 중
  - 날짜 기준: 오늘 자정 이후 종료된 문의만

### Step 18: 칸반 카드 - 기본 정보
- **title**: 문의 카드 요약 정보
- **text**: 각 칸반 카드는 고객명, 유입 채널(배지), 고객 유형(신규/기존), 상담 유형(내원/비대면), 연락처, 상담 내용 요약을 표시합니다.
- **highlight**: ".draggable-card"
- **trigger**: confirm
- **tips**:
  - 채널 배지: 색상으로 유입 경로 표시
  - 고객 유형: 파란색(신규) / 초록색(기존고객)
  - 상담 유형: 빨강(내원) / 파랑(비대면)
  - 보류 상태: 왼쪽 상단에 "⏸ 보류" 배지 표시

### Step 19: 칸반 카드 - 보류 사유 표시
- **title**: 보류 사유 및 메모 확인
- **text**: 문의가 보류 상태('first_visit_hold' 또는 'remote_hold')일 때, 카드에는 보류 사유가 주황색 배경으로 표시됩니다. 사유는 "비용 고민", "일정 조율", "가족 상의", "타병원 비교", "고민 중", "기타" 중 하나입니다.
- **highlight**: ".bg-amber-50.p-2.5.rounded-lg"
- **trigger**: confirm
- **tips**:
  - 사유 아이콘: 💰(비용), 📅(일정), 👨‍👩‍👧(가족), 🔍(비교), 🤔(고민), 📝(기타)
  - 추가 메모: 사유 옆에 "-" 뒤에 표시됨
  - 처리 시: 상세 페이지에서 사유 수정 가능

### Step 20: 칸반 카드 - 태그 표시
- **title**: 문의에 붙은 태그 확인
- **text**: 카드 하단에 해당 문의에 붙은 태그가 파란색 배지로 최대 3개까지 표시됩니다. 4개 이상일 때는 "+N" 표시로 추가 태그 수를 표시합니다.
- **highlight**: ".flex.flex-wrap.gap-1.mb-2"
- **trigger**: confirm
- **tips**:
  - 최대 3개 표시: 공간 제약으로 인해 처음 3개만 표시
  - "+N" 표시: 추가 태그 수 (예: "+2" = 2개 더 있음)
  - 클릭 불가: 태그 필터는 필터 영역에서만 가능

### Step 21: 칸반 카드 - 드래그앤드롭
- **title**: 카드를 드래그하여 상태 변경
- **text**: 칸반 카드를 마우스로 클릭하여 드래그하고 다른 열에 드롭하면 문의의 상태가 자동으로 변경됩니다. 예: "문의 발생" → "처리 중"으로 이동하면 상태가 'consulting'으로 변경됩니다.
- **highlight**: ".draggable-card[draggable='true']"
- **trigger**: click
- **tips**:
  - 마우스 커서: 드래그 시 손가락 모양으로 변함 (cursor-grabbing)
  - 애니메이션: 드롭 시 스냅 애니메이션
  - 실시간 저장: 드롭 완료 시 API 호출로 즉시 저장됨

### Step 22: 칸반 카드 - 클릭하여 상세 보기
- **title**: 문의 상세 페이지로 이동
- **text**: 칸반 카드를 클릭하면 해당 문의의 상세 페이지로 이동합니다. 상세 페이지에서는 고객 정보, 접수 내용, 상담 이력, 상태 변경 등을 관리할 수 있습니다.
- **highlight**: ".draggable-card"
- **trigger**: click
- **tips**:
  - 네비게이션: data-nav-url 속성으로 이동 경로 설정됨
  - 메타 정보: 모든 카드 데이터는 data-* 속성으로 인코딩됨
  - 보안: demo-blur 클래스로 미등록 시 흐림 처리

### Step 23: 리스트 뷰 - 테이블 헤더
- **title**: 테이블 열 설명
- **text**: 리스트 뷰의 테이블 헤더에는 "이름", "연락처", "유입 경로", "구분", "상태", "접수일시", "관리" 7개 열이 있습니다. 각 열은 좌측 정렬 또는 오른쪽 정렬되어 있습니다.
- **highlight**: "thead th"
- **trigger**: confirm
- **tips**:
  - 열 너비: 각 열이 비례적으로 조정됨
  - 헤더 색상: 회색 배경 (bg-slate-50)
  - 고정: 가로 스크롤 시에도 헤더 유지

### Step 24: 리스트 뷰 - 행 클릭하여 이동
- **title**: 리스트 행을 클릭하여 상세 페이지 이동
- **text**: 테이블의 모든 행을 클릭하면 해당 문의의 상세 페이지로 이동합니다. 행의 어느 위치든 클릭 가능하며 호버 시 배경색이 변합니다.
- **highlight**: "tbody tr"
- **trigger**: click
- **tips**:
  - 호버 효과: 행 배경색이 옅은 회색으로 변함
  - 클릭 영역: 행 전체가 클릭 가능
  - 커서: 마우스 오버 시 손가락 모양으로 변함

### Step 25: 채널 배지 - 유입 경로 시각화
- **title**: 문의 유입 경로를 색상으로 표시
- **text**: 각 카드/행의 채널 배지는 문의가 어느 경로로 들어왔는지를 색상과 텍스트로 표시합니다. 예: 초록색 "네이버 예약", 파란색 "AI 상담", 주황색 "전화 문의" 등.
- **highlight**: ".inline-flex.items-center.px-2.py-0.5.rounded"
- **trigger**: confirm
- **tips**:
  - 색상 코드: 채널별로 고정된 색상 사용
  - 주요 채널: 홈페이지, 채팅, AI, 전화, 카카오, 네이버 등
  - 필터링: 필터 영역에서 채널 선택 시 같은 색상 배지 표시

### Step 26: 상태 변경 모달 - 문의 상세에서
- **title**: 문의 상태 업데이트
- **text**: 문의 상세 페이지에서 "상태 변경" 버튼(또는 상태 드롭다운)을 클릭하면 상태 선택 모달이 열립니다. 예약, 보류, 상담 완료 등 다양한 상태 옵션을 선택할 수 있습니다.
- **highlight**: "button:contains('상태 변경')"
- **trigger**: click
- **tips**:
  - 옵션: 예약(내원/비대면), 보류(내원/비대면), 재방문 예약, 종료 등
  - 보류 선택 시: 추가로 "보류 사유" 선택 모달이 열림
  - 저장: 확인 버튼으로 상태 저장

### Step 27: 보류 사유 모달
- **title**: 보류 이유 및 메모 기록
- **text**: 문의 상태를 "보류"로 변경할 때 열리는 추가 모달에서 보류 사유와 메모를 입력합니다. 사유는 "비용 고민", "일정 조율", "가족 상의", "타병원 비교", "고민 중" 등에서 선택할 수 있습니다.
- **highlight**: "#holdReasonModal"
- **trigger**: confirm
- **tips**:
  - 필수 입력: 보류 사유 (선택 옵션)
  - 선택 입력: 메모 (추가 정보)
  - 저장 후: 카드에 "⏸ 보류" 배지 + 사유 표시

### Step 28: 문의 생성/수정 모달
- **title**: 문의 정보 입력 및 수정
- **text**: "문의 등록" 버튼 클릭 시 또는 상세 페이지의 "수정" 버튼 클릭 시 열리는 모달에서 고객 이름, 연락처, 채널, 상담 내용, 태그 등을 입력/수정할 수 있습니다.
- **highlight**: "#createInquiryModal"
- **trigger**: confirm
- **tips**:
  - 필수 입력: 이름, 연락처, 채널
  - 선택 입력: 메모, 태그, 상담 유형(내원/비대면)
  - 저장: "문의 등록" 버튼으로 저장 (자동 새로고침)
