# EMR 데이터 변환기 (/admin/data/converter) — 투어 시나리오

## 페이지 목적
한의사랑, 오케이차트, 네오보감, 한의맥 EMR 시스템의 환자 데이터를 CSV/Excel 파일로부터 읽어 변환하여 시스템에 일괄 등록하는 페이지

## 시나리오 (8 스텝)

### Step 1: 데이터 변환기 페이지 진입
- **title**: EMR 데이터 변환기 개요
- **text**: 이 페이지는 기존 EMR 시스템의 환자 데이터를 Clinic-OS로 마이그레이션하는 도구입니다. 상단에는 페이지 제목과 설명이 있으며, 아래로 3단계 프로세스가 순차적으로 배치되어 있습니다: 1)EMR 선택, 2)파일 업로드, 3)데이터 미리보기 및 등록.
- **highlight**: `.max-w-7xl.mx-auto`
- **trigger**: confirm
- **tips**:
  - 4가지 EMR 시스템 지원: 한의사랑, 오케이차트, 네오보감, 한의맥
  - 각 시스템별 파일 형식이 다름
  - 프로세스는 순차적으로 진행됨

### Step 2: EMR 시스템 선택
- **title**: EMR 시스템 선택하기
- **text**: '1. EMR 시스템 선택' 섹션에서 4개의 EMR 시스템 버튼(한의사랑, 오케이차트, 네오보감, 한의맥)을 볼 수 있습니다. 각 버튼에는 시스템 이름, 아이콘, 지원 파일 형식(CSV/XLSX)이 표시되어 있습니다. 사용하는 EMR 시스템의 버튼을 클릭하여 선택합니다.
- **highlight**: `.grid.grid-cols-1.md\\:grid-cols-3.gap-4`
- **trigger**: click
- **buttons**:
  - selector: `.emr-select-btn[data-emr="hanisarang"]`
  - label: "🏥 한의사랑"
  - desc: 한의사랑 CSV/XLSX 파일 선택
  - selector: `.emr-select-btn[data-emr="okchart"]`
  - label: "📊 오케이차트"
  - desc: 오케이차트 CSV 파일 선택
  - selector: `.emr-select-btn[data-emr="neobogam"]`
  - label: "📋 네오보감"
  - desc: 네오보감 XLS/XLSX 파일 선택
  - selector: `.emr-select-btn[data-emr="hanimek"]`
  - label: "🏥 한의맥"
  - desc: 한의맥 XLSX 파일 선택
- **tips**:
  - 선택된 EMR은 버튼 배경색이 파란색으로 변함
  - EMR 선택 후에만 파일 업로드 가능
  - 각 EMR마다 데이터 포맷과 필드명이 다름

### Step 3: 파일 업로드 영역
- **title**: 파일 업로드하기
- **text**: '2. 파일 업로드' 섹션의 점선 테두리 박스(#fileUploadArea)에 EMR 내보내기 파일을 드래그-드롭하거나, 박스를 클릭하여 파일을 선택할 수 있습니다. 호버 시 박스 배경이 파란색으로 변하며 업로드 준비 상태를 나타냅니다. 지원 파일 형식은 EMR 선택 후 자동으로 업데이트됩니다.
- **highlight**: `#fileUploadArea`
- **trigger**: click
- **tips**:
  - 드래그-드롭 지원
  - 파일 크기 제한 있음
  - EMR 선택 전에는 파일 업로드 불가
  - 파일 선택 후 자동으로 파싱 시작

### Step 4: 파일 처리 및 파싱
- **title**: 파일 파싱 진행
- **text**: 파일 선택 후 자동으로 파일 처리가 시작되며, 로딩 표시('파일 처리 중...')가 나타납니다. 시스템이 Excel 라이브러리를 동적으로 로드하고(CDN 또는 로컬 npm), 파일의 첫 번째 시트에서 환자 데이터를 추출합니다. 처리 중에 인터넷 연결이 필요할 수 있습니다.
- **highlight**: `.flex.items-center.justify-center`
- **trigger**: confirm
- **tips**:
  - Excel 파일의 경우 CDN에서 라이브러리 로드 (실패 시 로컬 npm 사용)
  - CSV는 fetch → text() → CSV 파싱
  - 파싱 실패 시 오류 메시지 표시
  - 네트워크 오류 시 VPN 확인 메시지 표시

### Step 5: 데이터 미리보기 섹션
- **title**: 파싱 결과 미리보기
- **text**: 파일 처리 완료 후 '3. 데이터 미리보기' 섹션이 표시됩니다. 이 섹션에는 유효한 환자 데이터 개수와 변환 오류 개수가 상단에 표시됩니다. 오류가 있으면 빨간색 박스에 오류 목록이 표시되며(최대 20개, 그 이상은 생략), 테이블에는 유효한 데이터가 최대 50개까지 표시됩니다.
- **highlight**: `#previewSection`
- **trigger**: confirm
- **tips**:
  - 유효한 데이터: 필수 필드가 모두 있는 레코드
  - 오류: 필수 필드 누락, 데이터 형식 오류 등
  - 테이블은 스크롤 가능 (많은 데이터는 생략)
  - 미리보기는 최대 50개 행 표시

### Step 6: 데이터 미리보기 테이블
- **title**: 변환된 환자 데이터 확인
- **text**: 테이블에는 번호, 차트번호, 이름, 연락처, 생년월일, 초진일, 최종진료일, 진료횟수, 성별, 주소, 메모 등 환자의 주요 정보가 칼럼별로 표시됩니다. 각 행은 한 명의 환자를 나타내며, 필드값이 없으면 '-'으로 표시됩니다. 호버 시 행 배경이 변합니다.
- **highlight**: `.overflow-x-auto.rounded-xl.border`
- **trigger**: confirm
- **tips**:
  - 테이블 데이터는 파싱된 원본 데이터와 동일
  - 칼럼명은 한글로 표시됨
  - 주소는 긴 텍스트이므로 오버플로우 처리
  - 각 EMR별로 칼럼 순서가 다를 수 있음

### Step 7: 오류 목록 확인
- **title**: 변환 오류 세부 확인
- **text**: '변환 오류' 섹션(있는 경우)에는 파싱 중 발생한 오류의 상세 내용이 리스트 형태로 표시됩니다. 오류 메시지는 어느 행에서 어떤 필드가 문제인지 명확하게 표시합니다. 예: "Row 5: 생년월일 형식이 유효하지 않음 (YYYY-MM-DD)". 오류가 있어도 등록은 가능하지만, 오류 레코드는 건너뜁니다.
- **highlight**: `#errorList`
- **trigger**: confirm
- **tips**:
  - 오류는 최대 20개까지 표시되며, 초과 시 '... 외 N건' 표시
  - 각 오류에는 행 번호와 필드명 포함
  - 오류 레코드는 데이터베이스에 저장되지 않음
  - 오류가 0개면 섹션 자체가 숨겨짐

### Step 8: 등록 버튼 및 최종 확인
- **title**: 데이터 등록하기
- **text**: 테이블 하단 우측의 '등록하기' 버튼(#importBtn)을 클릭하여 파싱된 환자 데이터를 시스템에 등록합니다. 버튼에는 등록할 환자 개수가 표시됩니다. 클릭 시 확인 대화창('N건의 환자 데이터를 등록하시겠습니까?')이 나타나며, 확인하면 등록이 시작됩니다. 등록 후 결과 요약(성공/중복/오류)이 표시됩니다.
- **highlight**: `#importBtn`
- **trigger**: click
- **buttons**:
  - selector: `#importBtn`
  - label: "등록하기 (N건)"
  - desc: 파싱된 환자 데이터를 시스템에 등록
- **tips**:
  - 등록 전 확인 대화창이 반드시 표시됨
  - 등록 중 로딩 표시('데이터 등록 중...') 나타남
  - 등록 완료 후 결과 알림 표시
  - 성공: 신규 등록 건수, 중복: 이미 존재하는 연락처/차트번호, 오류: 유효성 검사 실패
  - 등록 후 파일 입력값이 초기화됨
