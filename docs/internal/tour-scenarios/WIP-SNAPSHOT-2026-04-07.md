## 주요 학습

### 설계 원칙
1. **AC-first**: 품질 기준 먼저 정의 → 검증 스크립트 작성 → 반복 검증
2. **병렬 처리**: 각 배치를 독립 에이전트 동시 진행 (5배 속도 향상)
3. **품질 게이트**: 미달 시나리오 즉시 기각 → 재작성 (배송 예시)
4. **한글 매칭 이슈**: AC 스크립트의 영어 grep vs "모달" 한글 텍스트 불일치 → 수동 검증

### 페이지 특성

```
단순 (3~5 스텝)  →  CRUD만 필요 (회원, 공지, 문서)
중간 (10~15 스텝) →  멀티탭 + 모달 (환자, 블로그)
복잡 (17~37 스텝) →  다중 모달 + 중첩 폼 (직원, 설정, 배송)
지표 (11~13 스텝) →  필터 + 차트 (분석, 대시보드)
```

### 시나리오 템플릿

```
1. 페이지 소개 (전체 레이아웃)
2. 주요 필터 (날짜/담당의/상태)
3. 액션 버튼 (추가/수정/삭제)
4. 모달 A 진입 (버튼 클릭 → 열림)
5. 모달 A 필드 (input/select → 입력)
6. 모달 A 저장 (submit → 닫힘)
7. [모달 B가 있으면 4~6 반복]
8. 상태 변경 (행별 액션)
9. 데이터 내보내기/인쇄 (선택)
10. 도움말/학습 링크 (선택)
```
