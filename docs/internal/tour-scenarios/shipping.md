# 배송/해피콜 관리 (/admin/shipping) — 투어 시나리오

## 페이지 목적
한약 배송 일정 관리, 배송 처리(택배사/송장), 해피콜 대상자 추적, 배송 알림 문자 발송을 하는 페이지.

## 시나리오 (10 스텝)

### Step 1: 페이지 소개
- **title**: 배송/해피콜 관리
- **text**: 한약 배송 일정과 해피콜 대상자를 관리하는 페이지입니다. 상단 5개 카드로 현황을 파악하고, 하단 테이블에서 배송을 처리합니다.
- **highlight**: 없음
- **trigger**: confirm
- **tips**:
  - 해피콜: 마지막 배송 후 21일이 경과하면 자동으로 해피콜 대상이 됩니다
  - 지연/임박 배송은 대시보드에서도 확인됩니다
  - 환불된 건은 잔여 수량에서 자동 차감됩니다

### Step 2: 현황 카드
- **title**: 배송/해피콜 — 현황 대시보드
- **text**: 5개 카드로 현재 배송 현황을 한눈에 파악합니다. 각 카드를 클릭하면 해당 상태의 항목만 필터링됩니다.
- **highlight**: 없음
- **trigger**: confirm
- **buttons**:
  - selector: `a[href="?status=happy_call"]` / label: 해피콜 대상 / desc: 마지막 배송 후 21일 경과한 환자 (주황색)
  - selector: `a[href="?status=overdue"]` / label: 배송 지연 / desc: 다음 배송일이 지난 항목 (빨강)
  - selector: `a[href="?status=due_soon"]` / label: 배송 임박 / desc: 3일 이내 배송 예정 (노랑)
  - selector: `a[href="?status=all_active"]` / label: 전체 활성 / desc: 잔여 수량이 남은 모든 항목
  - selector: `a[href="?status=today_shipped"]` / label: 오늘 출고 / desc: 오늘 출고 처리된 건 + 문자 미발송 건수

### Step 3: 검색/필터
- **title**: 배송/해피콜 — 검색 및 필터
- **text**: 환자 이름이나 전화번호로 검색하고, 상품별로 필터링할 수 있습니다.
- **highlight**: `input[name="q"]`
- **trigger**: confirm
- **buttons**:
  - selector: `input[name="q"]` / label: 검색 / desc: 환자 이름 또는 전화번호로 검색합니다
  - selector: `select[name="product"]` / label: 상품 필터 / desc: 특정 상품의 배송만 표시합니다

### Step 4: 배송 목록 테이블
- **title**: 배송/해피콜 — 배송 목록
- **text**: 각 행에 환자명, 상품, 잔여 횟수, 다음 배송일, 최근 배송일, 상태 뱃지가 표시됩니다. 환자 이름을 클릭하면 차트로 이동합니다.
- **highlight**: 없음
- **trigger**: confirm
- **tips**:
  - 빨강 뱃지 "지연": 배송일이 지났습니다 — 즉시 출고 필요
  - 주황 뱃지 "해피콜": 3주 이상 연락 없음 — 전화 필요
  - 노랑 뱃지 "임박": 3일 이내 배송 예정
  - 잔여 수량이 0이면 (환불 포함) 목록에서 사라집니다

### Step 5: 출고 처리 버튼
- **title**: 배송/해피콜 — 출고 처리
- **text**: 각 행의 "출고" 버튼을 클릭하면 택배사 선택 모달이 열립니다. 출고 처리 시 잔여 수량이 1 감소하고 배송 기록이 생성됩니다.
- **highlight**: 없음
- **trigger**: confirm

### Step 6: 택배사 선택 모달
- **title**: 배송/해피콜 — 택배사 선택 모달
- **text**: 출고 버튼 클릭 시 열리는 모달입니다. 택배사를 선택하고, 송장번호를 입력하고, 배송지 주소를 확인/수정합니다. "출고 완료" 버튼을 누르면 배송이 등록됩니다.
- **highlight**: `#courier-modal`
- **trigger**: confirm
- **tips**:
  - 택배사: CJ대한통운, 한진, 롯데, 로젠, 우체국 등에서 선택
  - 송장번호를 입력하면 자동으로 조회 URL이 생성됩니다
  - 주소가 비어있으면 "주소 검색" 버튼으로 다음 우편번호 API를 통해 입력 가능
  - 출고 후 자동으로 "오늘 출고" 탭에 표시됩니다

### Step 7: SMS 발송 모달
- **title**: 배송/해피콜 — 배송 알림 문자 발송 모달
- **text**: 출고 완료 후 "문자 발송" 버튼을 클릭하면 SMS 템플릿 선택 모달이 열립니다. 미리 등록된 알림톡 템플릿 중 하나를 선택하여 환자에게 배송 안내 문자를 보냅니다.
- **highlight**: `#sms-modal`
- **trigger**: confirm
- **tips**:
  - 알림톡 템플릿은 "캠페인 > 메시지 템플릿" 메뉴에서 미리 등록해야 합니다
  - 송장번호가 입력된 경우 자동으로 택배 조회 URL이 문자에 포함됩니다
  - 문자 발송 후 해당 건의 "문자 미발송" 뱃지가 사라집니다
  - 알림톡이 실패하면 자동으로 LMS로 대체 발송됩니다

### Step 8: 해피콜 처리
- **title**: 배송/해피콜 — 해피콜 완료 기록
- **text**: 해피콜 대상 환자에게 전화 후, "해피콜 완료" 버튼으로 통화 결과를 기록합니다. 복용 상태와 다음 내원 일정을 확인하세요.
- **highlight**: 없음
- **trigger**: confirm
- **tips**:
  - 해피콜 완료 시 해피콜 대상 카운트가 감소합니다
  - 통화 내용은 환자 차트의 이벤트 기록에 자동 저장됩니다

### Step 9: 오늘 출고 탭
- **title**: 배송/해피콜 — 오늘 출고 현황
- **text**: "오늘 출고" 카드를 클릭하면 오늘 출고 처리된 건만 표시됩니다. 문자 미발송 건수가 카드에 함께 표시되어, 아직 알림을 보내지 않은 건을 바로 확인할 수 있습니다.
- **highlight**: `a[href="?status=today_shipped"]`
- **trigger**: confirm
- **tips**:
  - "문자 미발송" 뱃지가 있는 건은 환자에게 아직 알림이 안 간 상태입니다
  - 각 건의 "문자 발송" 버튼을 클릭하여 개별 발송합니다

### Step 10: 완료 건 보기
- **title**: 배송/해피콜 — 완료 건 조회
- **text**: 상단 탭에서 "완료" 탭을 선택하면 배송이 모두 끝난(잔여 수량 0) 건을 확인할 수 있습니다. 하단 페이지네이션으로 이전/다음 페이지를 이동합니다.
- **highlight**: 없음
- **trigger**: confirm
