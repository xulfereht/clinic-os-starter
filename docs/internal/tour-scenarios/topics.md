# AEO 토픽(Pillar) 관리 (/admin/topics) — 투어 시나리오

## 페이지 목적
검색 엔진 최적화(AEO)를 위한 핵심 주제(Pillar Topics)와 FAQ 클러스터를 생성/관리하는 페이지

## 시나리오 (6 스텝)

### Step 1: 토픽 관리 페이지 진입
- **title**: AEO 토픽 관리 페이지 개요
- **text**: 이 페이지는 AI 검색 최적화를 위한 핵심 주제(Pillar)들을 관리하는 곳입니다. 상단에는 페이지 제목과 설명, '+ 새 토픽 만들기' 버튼이 있습니다. 아래로는 생성된 토픽들이 카드 형태로 표시되며, 각 토픽마다 썸네일, 제목, 설명, 관련 프로그램 정보, 감수자, 관리 링크 등이 표시됩니다.
- **highlight**: `.max-w-6xl.mx-auto`
- **trigger**: confirm
- **tips**:
  - 토픽은 회사/병원의 주요 서비스 영역을 나타냄
  - 각 토픽 아래에는 FAQ 클러스터가 구성됨
  - 토픽은 SEO를 위해 매우 중요한 구조

### Step 2: 새 토픽 만들기 버튼
- **title**: 토픽 생성 시작
- **text**: 우측 상단의 '+ 새 토픽 만들기' 버튼을 클릭하면 토픽 생성 페이지로 이동합니다. 여기서 토픽의 제목, 슬러그, 요약, 썸네일 이미지, 관련 프로그램 연결 등의 정보를 입력하여 새 토픽을 생성할 수 있습니다.
- **highlight**: `a[href="/admin/topics/new"]`
- **trigger**: click
- **buttons**:
  - selector: `a[href="/admin/topics/new"]`
  - label: "+ 새 토픽 만들기"
  - desc: 토픽 생성 페이지로 이동
- **tips**:
  - 토픽 생성은 별도 상세 페이지에서 수행됨
  - 제목, 슬러그, 요약은 필수 항목

### Step 3: 토픽 카드 - 기본 구성
- **title**: 토픽 카드 정보 확인
- **text**: 각 토픽 카드는 좌측에 썸네일(또는 기본 아이콘 📝), 중앙에 토픽 정보(제목, 슬러그, 요약), 우측에 액션 버튼들로 구성됩니다. 카드는 모바일 환경에서는 세로 레이아웃으로 변경됩니다. 제목은 링크로 표시되어 클릭하면 토픽 상세 편집 페이지로 이동합니다.
- **highlight**: `.SectionCard`
- **trigger**: confirm
- **tips**:
  - 썸네일은 토픽을 대표하는 이미지
  - 썸네일이 없으면 기본 아이콘(📝) 표시
  - 제목 옆에는 슬러그가 회색 글씨로 표시됨
  - 요약은 최대 2줄까지 표시

### Step 4: 토픽 제목 및 슬러그
- **title**: 토픽 제목과 URL 경로 확인
- **text**: 각 토픽 카드의 제목(파란색 링크)은 토픽의 편집 페이지로 연결됩니다. 제목 옆의 회색 Slug(예: /back-pain-treatment)는 토픽의 공개 페이지 URL 경로를 나타냅니다. 슬러그는 시스템에서 고유하며, URL 친화적인 형식(소문자, 하이픈)으로 자동 변환됩니다.
- **highlight**: `h3 a[href*="/admin/topics/"]`
- **trigger**: click
- **tips**:
  - 토픽 제목을 클릭하면 편집 페이지로 이동
  - 슬러그는 변경 가능하지만 변경 시 기존 링크가 깨질 수 있음
  - 슬러그는 SEO 최적화에 중요한 역할

### Step 5: 토픽 메타 정보 (프로그램 연결, 감수자)
- **title**: 토픽의 관련 정보 확인
- **text**: 각 토픽 카드의 하단에는 메타 정보가 배지로 표시됩니다: 1)🔗 프로그램 연결 (선택적) - 토픽과 연결된 프로그램, 2)👨‍⚕️ 감수자 - 해당 토픽을 감수/관리하는 의료진. 프로그램이 연결되면 보라색 배지로 표시되며, 감수자가 없으면 '미지정'으로 표시됩니다.
- **highlight**: `.flex.flex-wrap.gap-2.text-xs`
- **trigger**: confirm
- **tips**:
  - 프로그램 연결은 선택사항 (없을 수도 있음)
  - 감수자는 직원 정보에서 선택됨
  - 감수자는 해당 토픽의 편집 권한을 가질 수 있음

### Step 6: 토픽 액션 버튼
- **title**: 토픽 관리 및 미리보기
- **text**: 각 토픽 카드의 우측에는 두 가지 액션 버튼이 있습니다: 1)'관리 / FAQ' 버튼은 토픽의 상세 편집 페이지로 이동하여 토픽 정보와 관련 FAQ들을 관리할 수 있습니다. 2)'미리보기 ↗' 버튼은 새 탭에서 토픽의 공개 페이지를 미리볼 수 있습니다. 미리보기 페이지에는 토픽 제목, 요약, FAQ 목록 등이 표시됩니다.
- **highlight**: `a[href*="/admin/topics/"], a[href*="/topics/"]`
- **trigger**: click
- **buttons**:
  - selector: `a:has-text("관리 / FAQ")`
  - label: "관리 / FAQ"
  - desc: 토픽 상세 편집 및 FAQ 관리 페이지로 이동
  - selector: `a:has-text("미리보기")`
  - label: "미리보기 ↗"
  - desc: 토픽 공개 페이지 미리보기 (새 탭에서 열림)
- **tips**:
  - 관리 버튼을 클릭하면 토픽 편집 페이지로 이동
  - 미리보기 버튼은 새 탭에서 열림
  - FAQ 관리는 토픽 편집 페이지에서 수행

### Step 7: 빈 토픽 상태
- **title**: 토픽이 없을 때
- **text**: 아직 생성된 토픽이 없으면 페이지 중앙에 "등록된 토픽이 없습니다." 메시지와 함께 '첫 번째 토픽(Pillar) 생성하기' 링크가 표시됩니다. 이 링크를 클릭하면 토픽 생성 페이지로 이동합니다.
- **highlight**: `.text-center.py-20.bg-slate-50`
- **trigger**: click
- **tips**:
  - 초기 상태에서는 토픽이 없을 수 있음
  - 첫 토픽 생성은 SEO 전략 수립의 첫 단계
  - 토픽 생성 후 FAQ를 추가하여 구조를 완성함
