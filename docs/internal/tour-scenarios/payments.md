# 결제 관리 (/admin/payments) — 투어 시나리오

## 페이지 목적
결제 내역을 조회, 등록, 환불 처리하고 매출 통계를 한눈에 파악합니다.

## 시나리오 (22 스텝)

### Step 1: 페이지 진입 및 통계 카드
- **title**: 매출 현황 대시보드
- **text**: 이 페이지는 결제 관리 시스템입니다. 상단의 4개 카드는 선택한 기간의 매출 데이터를 보여줍니다. 왼쪽부터 '총 매출'(수입금), '환불액'(반출금), '순매출'(실제 이익), '건수'(거래 횟수)입니다. 각 카드는 시간대별 통계에 사용됩니다.
- **highlight**: .grid-cols-1.md:grid-cols-4
- **trigger**: confirm
- **tips**:
  - 순매출 = 총 매출 - 환불액입니다
  - 환불액은 음수로 표시됩니다 (-₩)
  - 건수는 완료 결제 + 환불 건수를 분리 표시합니다

### Step 2: 기간 필터 - 오늘
- **title**: 기간별 필터링
- **text**: "오늘", "최근 7일", "최근 30일", "전체" 토글을 클릭하여 조회 기간을 변경합니다. 통계와 테이블이 선택한 기간의 데이터로 자동 갱신됩니다.
- **highlight**: .bg-slate-100.p-1.rounded-lg.flex
- **trigger**: click
- **tips**:
  - 오늘: 자정(KST)부터 현재까지
  - 전체: 시스템 운영 이후 전체 결제 내역
  - 필터 변경 시 페이지 새로고침 없이 반영됩니다

### Step 3: 상태 필터
- **title**: 결제 상태별 조회
- **text**: 드롭다운에서 "전체 상태", "완료", "환불", "취소"를 선택하여 특정 상태의 결제만 필터링합니다. 선택 시 자동으로 페이지가 새로고침됩니다.
- **highlight**: select[name="status"]
- **trigger**: click
- **tips**:
  - 완료: 정상 처리된 결제
  - 환불: 환불 처리 완료 건
  - 취소: 취소된 거래
  - 복합 필터: 기간 + 상태를 함께 사용 가능합니다

### Step 4: 검색 필드
- **title**: 환자명 또는 연락처 검색
- **text**: 텍스트 입력창에 환자 이름이나 전화번호를 입력하고 "검색" 버튼을 클릭하면 해당 환자의 결제 기록만 조회됩니다.
- **highlight**: input[name="q"]
- **trigger**: input
- **tips**:
  - 부분 검색 지원: "홍"만 입력해도 "홍길동" 검색 가능
  - 연락처: 번호의 일부만 입력해도 됩니다
  - 실시간 필터링 아님 → "검색" 버튼 클릭 필수

### Step 5: 결제 등록 버튼
- **title**: 새 결제 기록 추가
- **text**: "결제 등록" 파란색 버튼을 클릭하면 환자 검색 모달이 열립니다. 결제를 기록할 환자를 선택한 후 결제 정보를 입력합니다.
- **highlight**: "#openPaymentModalBtn"
- **trigger**: click
- **buttons**:
  - "결제 등록" / 파란 배경 / 결제 등록 모달 열기
  - "환불 등록" / 빨강 배경 / 환불 등록 모달 열기

### Step 6: 환불 등록 버튼
- **title**: 환불 기록 추가
- **text**: "환불 등록" 빨간색 버튼을 클릭하면 환자 검색 모달이 열립니다. 환불을 처리할 환자를 선택한 후 원거래와 환불 금액을 지정합니다.
- **highlight**: "#openRefundRegisterBtn"
- **trigger**: click
- **tips**:
  - 환불은 기존 결제 기록이 있어야만 등록 가능합니다
  - 부분 환불도 지원합니다

### Step 7: 결제 테이블 - 구분 열
- **title**: 결제/환불 구분 표시
- **text**: 테이블의 첫 번째 열 "구분"은 초록색 "결제" 또는 빨간색 "환불" 배지로 각 거래의 종류를 표시합니다.
- **highlight**: "tbody tr:first-child td:first-child"
- **trigger**: confirm
- **tips**:
  - 초록색 배지: 수입 거래 (결제)
  - 빨강색 배지: 지출 거래 (환불)
  - 배경색도 구분: 결제는 흰색, 환불은 연한 빨강색

### Step 8: 결제 테이블 - 일시 열
- **title**: 거래 발생 날짜 및 시간
- **text**: "일시" 열에는 결제 또는 환불이 이루어진 날짜가 "월/일" 형식으로 표시됩니다. 최신순으로 정렬되어 있습니다.
- **highlight**: "tbody tr:first-child td:nth-child(2)"
- **trigger**: confirm
- **tips**:
  - 시간은 표시되지 않습니다 (월/일만)
  - 시스템 타임존: KST (한국 표준시)
  - 정렬: 최신순 (내림차순)

### Step 9: 결제 테이블 - 환자 정보
- **title**: 환자명 및 연락처 표시
- **text**: "환자" 열에는 거래 대상 환자의 이름과 전화번호가 표시됩니다. 이름이 굵게 표시되고 아래에 작은 글씨로 연락처가 표시됩니다.
- **highlight**: "tbody tr:first-child td:nth-child(3)"
- **trigger**: confirm
- **tips**:
  - 환자 정보는 보안상 흐림 처리될 수 있습니다 (demo-blur)
  - 미등록 환자: "미등록"으로 표시
  - 연락처가 없으면 공백으로 표시됩니다

### Step 10: 결제 테이블 - 내용 열
- **title**: 상품명 또는 결제 메모
- **text**: "내용" 열은 결제와 관련된 상품명을 표시합니다. 상품명이 없으면 메모의 첫 줄을 최대 30자까지 표시합니다.
- **highlight**: "tbody tr:first-child td:nth-child(4)"
- **trigger**: confirm
- **tips**:
  - 상품이 지정되면 상품명 우선 표시
  - 상품 없는 경우: 메모 또는 "환불"/"일반 결제" 텍스트
  - 길이 제한으로 인한 생략: 말줄임 표시 (...)

### Step 11: 결제 테이블 - 금액 열
- **title**: 결제 또는 환불 금액
- **text**: "금액" 열은 오른쪽 정렬로 표시되며, 결제는 양수(₩50,000), 환불은 음수(-₩50,000)로 표시됩니다. 천 단위 구분자(쉼표)가 자동 삽입됩니다.
- **highlight**: "tbody tr:first-child td:nth-child(5)"
- **trigger**: confirm
- **tips**:
  - 결제금액: 양수 (초록색 배경 행)
  - 환불금액: 음수 표시 (빨강색 배경 행)
  - 통화: ₩ (한국 원)
  - 소수점 없음 (정수 단위)

### Step 12: 결제 행 클릭 - 상세 모달 열기
- **title**: 결제 상세 정보 확인
- **text**: 테이블의 결제 행을 클릭하면 우측 슬라이드 형식의 상세 모달이 열립니다. 결제 정보, 환자 정보, 메모 등을 확인할 수 있습니다.
- **highlight**: "tbody tr"
- **trigger**: click
- **tips**:
  - 행의 어느 위치든 클릭 가능
  - 모달은 반투명 검정 배경(backdrop)으로 열립니다
  - 닫기: X 버튼 또는 배경 클릭

### Step 13: 상세 모달 - 환자 정보 카드
- **title**: 클릭 가능한 환자 프로필
- **text**: 모달의 상단에 "환자 정보" 카드가 표시되며, 이름과 연락처가 보입니다. 이 카드를 클릭하면 환자 상세 페이지로 이동합니다.
- **highlight**: "#detailPatientLink"
- **trigger**: click
- **tips**:
  - 파란 테두리로 강조되어 있습니다
  - 호버 시: 테두리가 파란색으로 변합니다
  - "(클릭하여 이동)" 텍스트로 안내됩니다

### Step 14: 상세 모달 - 결제 정보
- **title**: 결제 상세 내역
- **text**: 결제일, 구분(결제/환불), 상품명, 수량, 최종 금액, 할인 금액 등이 격자형으로 표시됩니다. 각 항목은 라벨 + 값 형식입니다.
- **highlight**: ".grid.grid-cols-2.gap-4"
- **trigger**: confirm
- **tips**:
  - 할인이 있는 경우만 "할인" 행이 표시됩니다
  - 수량은 상품이 지정된 경우만 표시됩니다
  - 메모는 별도 섹션에 표시됩니다

### Step 15: 상세 모달 - 환불 버튼
- **title**: 결제 환불 처리
- **text**: 상세 모달의 하단 "환불" 버튼을 클릭하면 이 결제에 대한 환불 모달이 자동으로 열립니다. 환불 금액을 재입력하거나 수정할 수 있습니다.
- **highlight**: "#detailRefundBtn"
- **trigger**: click
- **tips**:
  - 빨간 배경 버튼입니다
  - 환불 가능 여부는 결제 상태에 따라 결정됩니다
  - 부분 환불도 가능합니다 (금액 조정 시)

### Step 16: 상세 모달 - 수정 버튼
- **title**: 결제 정보 수정 모드
- **text**: "수정" 버튼을 클릭하면 상세 모달이 수정 모드로 전환됩니다. 상품, 수량, 금액, 할인, 메모를 수정할 수 있으며, "수정 사유"를 필수로 입력해야 합니다.
- **highlight**: "button:contains('수정')"
- **trigger**: click
- **tips**:
  - 수정 사유: 필수 입력 (타임라인에 기록됨)
  - 결제일이나 카드 변경은 환불 후 신규 결제로 처리 권장
  - 저장 시 수정 이력이 타임라인에 추가됩니다

### Step 17: 상세 모달 - 삭제 버튼
- **title**: 결제 기록 삭제
- **text**: "삭제" 버튼을 클릭하면 확인 모달이 나타나고, "확인"을 누르면 이 결제 기록이 삭제됩니다. 삭제된 기록은 통계에 포함되지 않습니다.
- **highlight**: "button:contains('삭제')"
- **trigger**: click
- **tips**:
  - 소프트 삭제: 데이터는 보존되지만 조회에서 제외됨
  - 삭제 확인 대화: 필수
  - 삭제 후 페이지 새로고침으로 테이블 갱신

### Step 18: 환자 검색 모달 - 검색 입력
- **title**: 결제/환불 대상 환자 검색
- **text**: "결제 등록" 또는 "환불 등록" 버튼 클릭 후 열리는 모달의 검색창에 환자명이나 연락처를 입력합니다. 실시간으로 매칭되는 환자 목록이 아래에 표시됩니다.
- **highlight**: "#paymentPatientSearchInput"
- **trigger**: input
- **tips**:
  - 최소 2자 입력 시 검색 시작
  - 부분 검색 지원: "홍" 입력 시 "홍길동" 검색됨
  - 연락처: 번호의 일부 입력 가능

### Step 19: 환자 검색 결과 - 환자 선택
- **title**: 검색 결과에서 환자 선택
- **text**: 검색 결과 목록에서 원하는 환자를 클릭합니다. 환자가 선택되면 그 환자의 정보를 기반으로 결제/환불 폼이 자동으로 채워집니다.
- **highlight**: "#paymentPatientSearchResults .block"
- **trigger**: click
- **tips**:
  - 환자 카드: 이름, 연락처, 최근 결제 내역 표시
  - 호버 시: 배경색 변화 (파란색)
  - 선택 후: 자동으로 폼 필드 채워짐

### Step 20: 결제 등록 모달 - 상품 선택
- **title**: 결제할 상품 선택
- **text**: "상품 선택" 드롭다운에서 사전에 등록된 상품을 선택합니다. 상품을 선택하면 단가가 자동으로 "최종 금액"에 반영됩니다. "직접 입력"을 선택하면 금액을 수동으로 입력할 수 있습니다.
- **highlight**: "#productSelect"
- **trigger**: click
- **tips**:
  - 상품 옵션 형식: "상품명 (가격)"
  - 직접 입력: 상품 목록에 없는 경우 수동 금액 입력
  - 선택 후: 원금/할인/최종금액 자동 계산

### Step 21: 결제 등록 모달 - 프로모션 선택
- **title**: 할인 프로모션 적용
- **text**: "프로모션" 드롭다운에서 적용 가능한 할인/쿠폰을 선택합니다. 프로모션을 선택하면 할인 금액이 자동으로 계산되어 "최종 금액"이 조정됩니다.
- **highlight**: "#promotionSelect"
- **trigger**: click
- **tips**:
  - 프로모션 없음: 기본값 (할인 0원)
  - 할인 유형: 정률(%) 또는 정액(원) 지원
  - 자동 계산: 원금 × 수량 - 할인 = 최종금액

### Step 22: 결제 등록 모달 - 수량 및 최종 금액
- **title**: 수량 입력 및 금액 확정
- **text**: "수량" 필드에 구매 수량을 입력합니다 (기본값 1). "최종 금액"은 자동 계산되지만 필요시 직접 수정 가능합니다. 입력이 완료되면 "결제 등록" 버튼을 클릭하여 저장합니다.
- **highlight**: "#quantityInput, #amountInput"
- **trigger**: input
- **tips**:
  - 자동 계산: 상품 선택 시 자동 갱신
  - 수동 수정: 최종금액 직접 입력 시 암시적 할인 계산
  - 배송 처리: 체크박스로 "배송 완료 처리" 옵션 지원
