# 환자 관리 (/admin/patients) — 투어 시나리오

## 페이지 목적
등록된 모든 환자를 검색, 필터링, 정렬하여 개별 환자의 상세 정보와 최근 활동을 조회하는 환자 데이터베이스 관리 페이지입니다.

## 시나리오 (12 스텝)

### Step 1: 페이지 진입 및 전체 환자 데이터베이스 확인
- **title**: 환자 관리 페이지 진입
- **text**: 환자 관리 페이지에 진입하면, 가장 먼저 좌상단의 파란 카드가 보입니다. 이 카드에는 "전체 환자 데이터베이스"라고 제목이 나와 있고, 큰 숫자로 현재 등록된 총 환자 수(예: "1,234명")가 표시됩니다. 아래에는 두 개의 배지가 있어, "신규 [숫자]"(파란 배경)와 "활성 [숫자]"(초록 배경)를 표시합니다. 신규는 아직 첫 내원 전 상태인 잠재고객, 활성은 최근 결제 기록이 있는 고객입니다.
- **highlight**: `.bg-blue-600.p-5.rounded-2xl` (전체 환자 카드)
- **trigger**: confirm
- **tips**:
  - 이 숫자들은 페이지 로드 시 계산되며, 새로운 환자가 등록되면 자동으로 업데이트됩니다
  - 활성 고객 비율이 높을수록, 한의원의 고객 유지율이 좋다는 뜻입니다
  - 신규 고객 수가 갑자기 증가하면, 마케팅이 잘 작동하고 있다는 신호입니다

---

### Step 2: 세그먼트 관리 링크 확인
- **title**: 고급 필터 및 세그먼트 저장 링크
- **text**: 통계 카드 옆의 큰 흰 박스에는 "환자 세그먼트"라고 제목이 나와 있고, 아래 설명 텍스트가 있습니다: "고급 필터와 세그먼트 저장은 CRM 페이지에서 관리하세요". 우측에는 "🔬 세그먼트 관리"라는 파란 버튼이 있으며, 이를 클릭하면 /admin/crm/segments 페이지로 이동합니다. 이 페이지에서는 복잡한 필터(예: 나이 범위, 방문 빈도, 결제액 등)를 조합하여 고객 세그먼트를 만들고 저장할 수 있습니다.
- **highlight**: `a[href="/admin/crm/segments"]` (세그먼트 관리 버튼)
- **trigger**: click
- **tips**:
  - 환자 관리 페이지에서는 이름, 전화, 차트번호로만 빠른 검색이 가능합니다
  - 더 정교한 필터(예: 초진 후 30일 이상 미방문 고객)가 필요하면 CRM 페이지에서 세그먼트를 생성하세요
  - 세그먼트를 저장해두면, 추후 마케팅 캠페인이나 대량 연락처 등록 시 재사용할 수 있습니다

---

### Step 3: 검색 입력창 사용 - 이름/전화/차트번호 검색
- **title**: 메인 검색 입력창
- **text**: 페이지 상단 중앙의 검색 영역에는 큰 입력창이 있습니다. 이 창의 왼쪽에는 🔍 아이콘이 있고, 플레이스홀더 텍스트는 "이름, 전화번호, 차트번호 검색..."입니다. 이 입력창에 환자의 이름(예: "김철수"), 전화번호(예: "010-1234-5678"), 또는 차트번호(예: "001")를 입력하면, 입력이 멈춘 후 약 0.3초(디바운스 시간) 후에 자동으로 환자 목록이 필터링됩니다. 검색은 페이지를 새로고침하지 않고 AJAX로 처리되므로, 결과를 즉시 확인할 수 있습니다. 입력창을 비우면 전체 목록으로 돌아갑니다.
- **highlight**: `#mainSearchInput` (검색 입력창)
- **trigger**: input
- **tips**:
  - 검색어는 부분 일치로 작동합니다. 예를 들어 "김"만 입력해도 김으로 시작하는 모든 환자가 검색됩니다
  - 전화번호는 하이픈 없이 숫자만 입력해도 검색됩니다 (예: "01012345678")
  - 차트번호로 검색하면, 매우 구체적인 환자 1명을 빠르게 찾을 수 있습니다
  - 검색 중 다른 정렬 기준을 변경해도, 검색 조건은 유지됩니다

---

### Step 4: 검색 상태 표시 및 초기화
- **title**: 활성 검색 조건 표시와 초기화 버튼
- **text**: 검색어를 입력하면, 검색 입력창 아래에 회색 배경의 상태 바가 나타납니다. 이 바에는 "🔍 검색어: [입력한 단어]"가 표시되고, 우측에는 "초기화"라는 작은 링크가 있습니다. 이 링크를 클릭하면 검색 조건이 제거되고 전체 환자 목록으로 돌아갑니다. 또한 주소창(URL)도 자동으로 업데이트되어, ?q=검색어&page=1 형식의 쿼리 파라미터가 추가됩니다. 이를 통해 다른 사용자에게 특정 검색 결과를 공유할 수 있습니다.
- **highlight**: `.bg-slate-50.px-4.py-2.rounded-lg` (검색 상태 바)
- **trigger**: confirm
- **tips**:
  - 검색 상태 바는 검색어가 있을 때만 표시되므로, 이를 보면 현재 필터링 중임을 인지할 수 있습니다
  - URL에 검색 조건이 저장되므로, 브라우저 뒤로가기를 누르면 이전의 검색 조건으로 돌아갑니다
  - 여러 명의 직원이 같은 검색 조건을 사용하려면, URL을 복사해 공유하면 됩니다

---

### Step 5: 정렬 드롭다운 메뉴 사용
- **title**: 환자 목록 정렬 기준 변경
- **text**: 검색 입력창 우측의 "정렬:" 레이블 옆에는 드롭다운 메뉴가 있습니다. 이 드롭다운을 클릭하면 정렬 옵션이 나타나며, 현재 선택된 옵션은 하이라이트됩니다:
  - **최근 활동 ↓** (기본값): 가장 최근에 활동(결제, 내원, 메모 등)이 있는 환자 순
  - **최근 활동 ↑**: 가장 오래전에 활동한 환자 순
  - **차트번호 ↑**: 차트번호 작은 순서(오래전 등록된 환자)
  - **차트번호 ↓**: 차트번호 큰 순서(최근 등록된 환자)
  - **이름 ㄱ→ㅎ**: 이름 가나다순
  - **이름 ㅎ→ㄱ**: 이름 역순
  - **결제액 ↓**: 누적 결제액 많은 순(VIP 고객)
  - **결제액 ↑**: 누적 결제액 적은 순(저가 치료 고객)
  
  옵션을 선택하면 페이지가 자동으로 새로고침되고, 환자 목록이 재정렬됩니다.
- **highlight**: `select[name="sort"]` (정렬 드롭다운)
- **trigger**: click
- **tips**:
  - "최근 활동 ↓"은 가장 자주 사용하는 정렬입니다. 활동이 많은 고객일수록 가치가 높습니다
  - 매달 말일에 "결제액 ↓"으로 정렬하면, VIP 고객(매출액 상위)을 식별할 수 있습니다
  - 신규 환자 확보 현황을 보려면 "차트번호 ↓"을 선택하세요
  - 정렬 옵션도 URL에 저장되므로, 다른 사용자와 같은 정렬 기준을 공유할 수 있습니다

---

### Step 6: 환자 테이블 헤더 정렬 클릭
- **title**: 테이블 헤더 컬럼 클릭으로 정렬
- **text**: 환자 목록 테이블의 헤더 행에는 여러 컬럼명이 있습니다. 이 중 "이름", "차트번호", "최근 활동"은 클릭 가능하며, 클릭하면 해당 컬럼 기준으로 정렬됩니다. 예를 들어 "이름" 헤더를 클릭하면 가나다순으로 정렬되고, 다시 클릭하면 역순으로 정렬됩니다. 정렬 중인 컬럼의 헤더 옆에는 작은 화살표(↑ 또는 ↓)가 파란색으로 표시되어, 현재 정렬 상태를 시각적으로 알립니다. "전화번호"와 "태그" 컬럼은 정렬 불가능하므로 클릭해도 반응이 없습니다.
- **highlight**: `a` (테이블 헤더 정렬 링크)
- **trigger**: click
- **tips**:
  - 테이블 헤더 정렬은 드롭다운 메뉴보다 빠르므로, 반복적으로 정렬을 변경할 때 효율적입니다
  - 정렬 화살표의 색상이 파란색이므로, 정렬 중인 컬럼을 쉽게 인지할 수 있습니다
  - 페이지에서 검색과 정렬을 함께 사용할 수 있습니다 (예: "김" 검색 후 결제액으로 정렬)

---

### Step 7: 환자 테이블 - 기본 정보 확인
- **title**: 환자 이름, 나이, 성별 및 아바타
- **text**: 환자 테이블의 첫 번째 컬럼에는 환자의 아바타와 기본 정보가 표시됩니다. 아바타는 작은 원형 박스로, 남자는 👨🏻, 여자는 👩🏻, 미지정은 👤 이모지가 표시됩니다. 아바타 옆에는 환자 이름(굵은 텍스트, 개인정보보호를 위해 "demo-blur" 클래스로 약간 흐릿함)이 표시되고, 그 아래에는 작은 텍스트로 나이와 성별 정보가 표시됩니다. 예: "35세 · 여" 또는 "- · 남" (생일 미등록 시). 이 정보들은 환자 등록 시 기입한 데이터를 기반으로 합니다.
- **highlight**: `td:first-child` (환자 이름 및 정보)
- **trigger**: confirm
- **tips**:
  - 나이는 자동 계산되므로, 생일을 정확히 등록하면 매년 자동으로 나이가 증가합니다
  - 성별 정보는 차이나는 증상이나 체질을 고려한 진료를 할 때 참고할 수 있습니다
  - 환자 이름이 흐릿하게 표시되는 이유는, 대시보드를 공유 화면으로 사용할 때 개인정보 보호를 위함입니다

---

### Step 8: 환자 테이블 - 차트번호 확인
- **title**: 차트번호 및 등록 순서 추적
- **text**: 두 번째 컬럼에는 차트번호가 표시됩니다. 차트번호는 "#001", "#123" 같은 형식으로, 환자가 처음 등록될 때 자동으로 부여됩니다. 차트번호가 없으면 "-" 기호가 표시됩니다. 차트번호는 환자의 등록 순서를 나타내므로, 차트번호가 낮을수록 오래전 환자, 높을수록 최근 환자입니다. 이를 이용해 한의원의 환자 누적 수를 대략 파악할 수 있습니다 (예: 최고 차트번호가 "#1000"이면 약 1,000명 누적).
- **highlight**: `td:nth-child(2)` (차트번호)
- **trigger**: confirm
- **tips**:
  - 차트번호로 정렬하면, 등록 순서대로 환자를 볼 수 있습니다
  - 차트번호는 수정할 수 없으므로, 잘못 등록된 경우에도 변경되지 않습니다
  - 차트번호 누락 환자가 많으면, 환자 등록 프로세스에 문제가 있을 수 있습니다

---

### Step 9: 환자 테이블 - 전화번호 확인
- **title**: 환자 연락처 정보
- **text**: 세 번째 컬럼에는 환자의 현재 전화번호가 표시됩니다. 전화번호는 "-" 포맷으로 "010-1234-5678" 형식이며, 개인정보보호를 위해 약간 흐릿하게 표시됩니다("demo-blur"). 전화번호는 환자를 직접 연락하거나, 알림 메시지를 보낼 때 사용됩니다. 번호가 등록되지 않으면 공란입니다.
- **highlight**: `td:nth-child(3)` (전화번호)
- **trigger**: confirm
- **tips**:
  - 전화번호는 환자 관리의 가장 중요한 정보이므로, 새 환자 등록 시 정확히 기입해야 합니다
  - 환자가 전화번호를 변경했다면, 환자 상세 페이지에서 수정할 수 있습니다
  - 알림 문자 발송 전에 전화번호를 확인하는 습관을 들이세요

---

### Step 10: 환자 테이블 - 태그 확인
- **title**: 환자 분류 태그
- **text**: 네 번째 컬럼에는 환자에게 붙은 태그가 표시됩니다. 태그는 파란 배경에 하얀 텍스트로 표시되는 작은 배지들입니다 (예: "VIP", "알레르기", "재진율높음"). 한 환자에게 최대 2개의 태그가 표시되고, 그 이상 있으면 생략됩니다. 태그가 없으면 "-" 기호가 표시됩니다. 태그는 환자 상세 페이지에서 추가, 수정, 삭제할 수 있으며, 환자 분류 및 마케팅 캠페인 대상 선정 시 유용합니다.
- **highlight**: `td:nth-child(4)` (태그)
- **trigger**: confirm
- **tips**:
  - 태그는 자유로운 텍스트이므로, 한의원의 분류 체계에 맞게 설정하세요
  - 자주 사용하는 태그를 정하고 일관되게 사용하면, 추후 세그먼트 분석이 용이합니다
  - 태그로 환자를 필터링하려면, CRM 페이지의 고급 필터를 사용하세요

---

### Step 11: 환자 테이블 - 최근 활동 확인
- **title**: 환자의 가장 최근 활동 유형 및 시간
- **text**: 다섯 번째 컬럼에는 최근 활동이 표시됩니다. 이 컬럼에는 두 부분이 있습니다:
  1. **활동 유형 배지**: 색상이 있는 작은 배지로, 가장 최근 활동의 유형을 표시합니다:
     - 💚 "결제" (초록, bg-emerald-100): 결제가 이루어졌음
     - 🔴 "환불" (빨강, bg-red-100): 환불이 이루어졌음
     - 🔵 "내원" (파랑, bg-blue-100): 내원했음
     - 🟠 "메모" (회색, bg-slate-100): 메모가 추가되었음
     - 🟠 "전화" (주황, bg-orange-100): 전화 연락이 기록되었음
     - 💜 "예약" (보라, bg-purple-100): 예약이 있음
     - 🟨 "문의" (주황, bg-amber-100): 문의 항목이 있음
     - 기타 "활동" (회색, bg-slate-100): 다른 종류의 활동
  
  2. **활동 시간**: 배지 아래에 작은 텍스트로 "7월 15, 14:30" 같은 형식의 날짜와 시간이 표시됩니다. 이는 모두 한국 시간대(KST) 기준입니다.
  
  이 컬럼의 헤더는 클릭 가능하며, 클릭하면 최근 활동순으로 정렬됩니다.
- **highlight**: `td:nth-child(5)` (최근 활동)
- **trigger**: confirm
- **tips**:
  - 최근 활동이 오래전인 환자는, 재내원을 독려하는 연락을 할 수 있습니다
  - "환불" 활동이 많으면, 진료 만족도 문제를 의심하세요
  - 활동 유형별 색상을 외워두면, 한눈에 환자의 상태를 파악할 수 있습니다

### Step 12: 환자 테이블 행 클릭 - 개별 환자 상세 페이지 이동
- **title**: 환자 상세 페이지로 이동
- **text**: 테이블의 각 행(환자 한 명)을 어느 곳이든 클릭하면 그 환자의 상세 페이지(/admin/patients/[id])로 이동합니다. 마우스를 행 위에 올리면 배경이 밝아지고("hover:bg-slate-50"), 우측 끝에 작은 화살표(→) 아이콘이 회색으로 표시되어 클릭 가능함을 알립니다. 상세 페이지에서는 환자의 전체 차트, 내원 기록, 결제 이력, 메모, 진료 계획 등을 모두 확인하고 관리할 수 있습니다.
- **highlight**: `tr` (환자 행)
- **trigger**: click
- **tips**:
  - 환자 이름 부분을 클릭해도, 전화번호 부분을 클릭해도 같은 상세 페이지로 이동합니다
  - 환자 상세 페이지에서 환자 정보, 진료 내용, 처방전 등을 수정할 수 있습니다
  - 새 환자를 등록한 직후에 상세 페이지로 이동해 기본 정보를 다시 확인하세요

### Step 13: 검색 결과 없음 화면
- **title**: 검색 결과 없음 시 안내 및 신규 문의 등록
- **text**: 검색 조건에 맞는 환자가 없으면, 테이블 자리에 큰 안내 화면이 표시됩니다:
  - 중앙에 큰 🔍 아이콘
  - "검색 결과가 없습니다."라는 제목
  - "새로운 필터 조건을 설정해보세요."라는 설명
  
  **중요**: 검색어가 있으면(예: "김철수"로 검색했는데 결과가 없음), 추가로 대파란 버튼이 나타납니다:
  - 버튼 텍스트: "📞 '[검색어]' 번호로 신규 문의 등록"
  - 클릭하면 /admin/leads/new 페이지로 이동하여, 검색한 번호를 기반으로 신규 문의를 바로 등록할 수 있습니다
  
  이 기능은 신규 환자가 전화로 문의했는데 아직 등록되지 않았을 때 매우 유용합니다.
- **highlight**: `div.flex.flex-col.items-center.justify-center` (검색 결과 없음)
- **trigger**: confirm
- **tips**:
  - 검색 결과가 없으면, 고객에게 다른 번호로 등록되었을 가능성을 확인하세요
  - "신규 문의 등록" 버튼으로 즉시 고객을 시스템에 추가할 수 있습니다
  - 자주 검색되지 않는 이름이나 번호가 있으면, 고객이 이미 다른 이름이나 번호로 등록되어 있지 않은지 확인하세요

### Step 14: 페이지네이션 - 이전/다음 버튼
- **title**: 페이지 이동
- **text**: 테이블 하단에는 페이지네이션 컨트롤이 있습니다. 좌측에는 "총 1,234명 중 1 - 10" 같은 현재 범위 표시가 있고, 우측에는 버튼들이 있습니다:
  - **이전** 버튼: 현재 페이지보다 이전 페이지로 이동 (1페이지에서는 표시 안 됨)
  - **페이지 표시**: "3 / 123" 같은 형식으로 현재 페이지와 총 페이지 수를 표시
  - **다음** 버튼: 다음 페이지로 이동 (마지막 페이지에서는 표시 안 됨)
  
  각 버튼을 클릭하면 페이지가 이동하고, URL의 page 파라미터가 업데이트됩니다 (예: ?page=2&q=검색어). 페이지당 10명의 환자가 표시됩니다.
- **highlight**: `#paginationContainer` (페이지네이션)
- **trigger**: click
- **tips**:
  - 한번에 10명씩 표시되므로, 1,000명의 환자를 관리해도 페이지 로딩이 빠릅니다
  - 페이지네이션은 검색 조건과 정렬 조건을 유지합니다
  - 특정 페이지의 환자 목록을 동료와 공유하려면 URL을 복사해 전송하세요

### Step 15: 환자 수 및 상태 통계
- **title**: 환자 상태별 분류 통계
- **text**: 전체 환자 카드 아래에는 "신규 [숫자]"와 "활성 [숫자]"라는 두 개의 배지가 있습니다. 이 통계는 다음과 같이 계산됩니다:
  - **신규**: 상태가 "inquiry"(문의) 또는 아직 내원하지 않은 환자
  - **활성**: 최근에 결제 기록이 있는 고객 (status = 'active')
  
  나머지 환자는 다음 범주에 속합니다:
  - "비활성" 또는 "이탈": 오래전에 방문했던 고객
  - "계정 보유": 환자가 고객 포털에 가입한 경우
  
  이 통계는 한의원의 전반적인 고객 건강도를 나타내는 핵심 KPI입니다.
- **highlight**: `.bg-blue-600` (통계 카드)
- **trigger**: confirm
- **tips**:
  - 신규 고객의 전환율(신규 → 활성)을 추적하면, 초진 상담의 성공도를 알 수 있습니다
  - 활성 고객 수가 감소 추세면, 고객 재내원을 유도하는 마케팅이 필요합니다
  - 이 통계는 일 1회, 자정에 갱신됩니다
