# 캠페인/이벤트 관리 (/admin/events) — 투어 시나리오

## 페이지 목적
마케팅 이벤트 및 캠페인을 생성/관리하고, 신청자를 관리하는 페이지

## 시나리오 (10 스텝)

### Step 1: 이벤트 관리 페이지 진입
- **title**: 캠페인/이벤트 관리 페이지 개요
- **text**: 이 페이지는 마케팅을 위한 이벤트 신청 폼을 관리하는 곳입니다. 상단에는 페이지 제목과 설명이 있으며, 우측에는 '+ 새 이벤트 만들기' 버튼이 있습니다. 아래로는 생성된 이벤트들이 테이블 형태로 표시됩니다.
- **highlight**: `.max-w-7xl.mx-auto`
- **trigger**: confirm
- **tips**:
  - 이벤트는 공개 페이지(/events/[slug])를 자동으로 생성함
  - 각 이벤트는 신청자 관리, 상품 관리 등을 포함
  - 이벤트는 활성/비활성 상태를 가짐

### Step 2: 새 이벤트 만들기 버튼
- **title**: 새 이벤트 생성 시작
- **text**: 우측 상단의 '+ 새 이벤트 만들기' 버튼(#newEventBtn)을 클릭하면 이벤트 생성 모달이 열립니다. 모달의 '캠페인 설정' 탭에서 이벤트의 기본 정보(제목, URL Slug, 상태, 설명)와 혜택/상품을 구성할 수 있습니다.
- **highlight**: `#newEventBtn`
- **trigger**: click
- **buttons**:
  - selector: `#newEventBtn`
  - label: "+ 새 이벤트 만들기"
  - desc: 이벤트 생성 모달 열기
- **tips**:
  - 새 이벤트 생성 시 '캠페인 설정' 탭만 표시됨
  - 신청자 탭은 저장 후 이용 가능
  - 모달은 최대 폭 5xl, 90vh 높이의 스크롤 가능 윈도우

### Step 3: 이벤트 테이블 구성
- **title**: 생성된 이벤트 목록
- **text**: 페이지의 테이블에는 모든 생성된 이벤트의 정보가 표시됩니다. 칼럼은 ID, 캠페인명, URL Slug(/events/slug), 신청자 수, 상태(진행중/종료), 생성일, 관리 버튼을 포함합니다. 각 행을 클릭하거나 '수정' 버튼을 클릭하면 이벤트 상세 모달이 열립니다.
- **highlight**: `table.w-full`
- **trigger**: confirm
- **tips**:
  - 테이블은 생성일 역순(최신순)으로 정렬됨
  - 신청자 수는 campaign_[slug] 채널의 리드 수를 기반으로 계산됨
  - 상태 배지(초록=진행중, 회색=종료)로 시각화됨
  - 각 행을 클릭하거나 '수정' 버튼으로 상세 조회 가능

### Step 4: URL Slug 및 미리보기
- **title**: 이벤트 공개 페이지 확인
- **text**: 테이블의 'URL Slug' 칼럼에는 /events/[slug] 형태의 경로가 표시되며, 그 옆의 화살표(↗) 아이콘을 클릭하면 새 탭에서 이벤트의 공개 페이지를 미리볼 수 있습니다. 이벤트가 활성화되면 이 URL로 신청자들에게 공개됩니다.
- **highlight**: `a[href*="/events/"]`
- **trigger**: click
- **tips**:
  - 미리보기는 새 탭에서 열림
  - 공개 페이지에는 이벤트 제목, 설명, 혜택/상품 정보 표시
  - Slug는 URL 친화적이어야 함 (소문자, 하이픈)

### Step 5: 이벤트 모달 - 캠페인 설정 탭
- **title**: 이벤트 기본 정보 입력
- **text**: 모달의 '📝 캠페인 설정' 탭에서는 이벤트의 기본 정보를 입력/수정할 수 있습니다. 입력 항목은 다음과 같습니다: 1)이벤트 제목(필수), 2)URL Slug(필수), 3)상태(진행중/종료 드롭다운), 4)설명(textarea). 모든 항목을 입력 후 하단의 '저장' 버튼을 클릭합니다.
- **highlight**: `.modal-content`
- **trigger**: confirm
- **tips**:
  - 제목과 Slug는 필수 항목
  - Slug는 특수문자 자동 변환 지원
  - 설명은 이벤트 페이지에 표시됨
  - 상태는 즉시 반영됨

### Step 6: 혜택/상품 추가
- **title**: 이벤트 혜택 및 상품 구성
- **text**: '캠페인 설정' 탭의 '혜택/상품 구성' 섹션에서는 이벤트에 포함될 상품을 추가할 수 있습니다. '+ 항목 추가' 버튼을 클릭하면 새 상품 카드가 추가됩니다. 각 상품 카드에는 상품명, 할인가, 정가, 설명, 내부값(value) 입력란이 있습니다. 상품은 여러 개 추가 가능하며, 각 상품의 우측 상단 X 버튼으로 제거할 수 있습니다.
- **highlight**: `#productsContainer`
- **trigger**: click
- **buttons**:
  - selector: `button:has-text("+ 항목 추가")`
  - label: "+ 항목 추가"
  - desc: 새 상품 카드 추가
- **tips**:
  - 상품 카드는 동적으로 추가/제거됨
  - 할인가와 정가는 숫자만 입력
  - 내부값(value)은 시스템 내부용 고유 식별자
  - 예: 상품명="백록감비정", 설명="체지방 감소", 할인가="89000", 정가="120000", 내부값="gambi"

### Step 7: 이벤트 모달 - 신청자 관리 탭
- **title**: 이벤트 신청자 조회 및 통계
- **text**: '👥 신청자 관리' 탭에는 이벤트의 신청자 정보가 표시됩니다. 상단에는 총 신청자 수, 오늘의 신청자 수, 엑셀 다운로드 버튼이 있는 통계 카드가 표시됩니다. 아래로는 신청자 목록 테이블이 표시되며, 각 신청자의 순번, 이름(기존 환자면 뱃지 표시), 연락처, 상태, 선택한 상품, 신청일이 표시됩니다.
- **highlight**: `[data-tab="applicants"]`
- **trigger**: click
- **buttons**:
  - selector: `[data-tab="applicants"]`
  - label: "👥 신청자 관리"
  - desc: 신청자 관리 탭 전환
- **tips**:
  - 신청자는 실시간으로 업데이트됨
  - 기존 환자는 녹색 '기존' 뱃지로 표시됨
  - 신청자 이름을 클릭하면 상세 리드 페이지로 이동(새 탭)
  - 신청자 없을 시 "아직 신청자가 없습니다." 메시지 표시

### Step 8: 신청자 통계 및 다운로드
- **title**: 신청자 통계 및 엑셀 다운로드
- **text**: 신청자 탭의 상단 카드에서는 세 가지 정보를 확인할 수 있습니다: 1)총 신청 [명], 2)오늘 신청 [명], 3)엑셀 다운로드 버튼. '📥 엑셀 다운로드' 버튼을 클릭하면 신청자 목록이 CSV 형식으로 다운로드됩니다. 파일명은 applicants_[YYYY-MM-DD].csv 형태입니다.
- **highlight**: `.grid.grid-cols-3.gap-4`
- **trigger**: click
- **buttons**:
  - selector: `button:has-text("📥 엑셀 다운로드")`
  - label: "📥 엑셀 다운로드"
  - desc: 신청자 목록을 CSV로 다운로드
- **tips**:
  - 다운로드된 CSV는 한글 인코딩 포함
  - 파일에는 헤더(No, 이름, 연락처, 상태, 상품, 신청일) 포함
  - 특수문자는 공백으로 대체되어 제대로 표시됨

### Step 9: 이벤트 저장 및 삭제
- **title**: 이벤트 저장 및 삭제
- **text**: 모달 하단의 푸터에는 '닫기', '저장' 버튼과 기존 이벤트의 경우 '🗑️ 삭제' 버튼이 있습니다. 정보를 수정한 후 '저장' 버튼을 클릭하면 변경사항이 저장되고 페이지가 새로고침됩니다. '🗑️ 삭제' 버튼을 클릭하면 확인 대화창 후 이벤트가 삭제됩니다.
- **highlight**: `.flex.justify-between.items-center`
- **trigger**: click
- **buttons**:
  - selector: `button:has-text("저장")`
  - label: "저장"
  - desc: 이벤트 변경사항 저장
  - selector: `button:has-text("닫기")`
  - label: "닫기"
  - desc: 모달 닫기
  - selector: `#deleteEventBtn`
  - label: "🗑️ 삭제"
  - desc: 이벤트 삭제 (기존 이벤트만)
- **tips**:
  - 저장 시 필수 항목(제목, Slug) 유효성 검사 수행
  - 저장 실패 시 토스트 알림으로 오류 메시지 표시
  - 삭제는 되돌릴 수 없음
  - 삭제 시 신청자 데이터는 보존됨

### Step 10: 모달 탭 전환 및 종료
- **title**: 모달 탭 전환 및 화면 종료
- **text**: 모달의 상단 탭 버튼('📝 캠페인 설정', '👥 신청자 관리')을 클릭하여 탭 간 이동이 가능합니다. 각 탭 버튼의 하단 경계선 색상으로 활성 탭을 표시합니다. 모달을 종료하려면 우측 상단의 '✕' 버튼, 하단의 '닫기' 버튼, 또는 모달 외부를 클릭하면 됩니다. Escape 키로도 모달을 닫을 수 있습니다.
- **highlight**: `.modal-tab-btn`
- **trigger**: click
- **buttons**:
  - selector: `.modal-tab-btn[data-tab="settings"]`
  - label: "📝 캠페인 설정"
  - desc: 캠페인 설정 탭으로 전환
  - selector: `.modal-tab-btn[data-tab="applicants"]`
  - label: "👥 신청자 관리"
  - desc: 신청자 관리 탭으로 전환
- **tips**:
  - 탭 전환 시 입력 중인 데이터는 임시 저장됨
  - 저장하지 않고 탭 전환 가능
  - 모달 종료 시 저장되지 않은 데이터는 손실됨
  - 우측 상단 ✕ 버튼으로 신속 종료 가능
