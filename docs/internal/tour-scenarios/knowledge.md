# 지식 라이브러리 (/admin/knowledge) — 투어 시나리오

## 페이지 목적
상담 및 응대에 활용할 수 있는 지식 카드와 스크립트를 생성/관리하고 분류하는 페이지

## 시나리오 (7 스텝)

### Step 1: 지식 라이브러리 페이지 진입
- **title**: 지식 라이브러리 개요
- **text**: 이 페이지는 상담사가 환자 상담 시 활용할 수 있는 지식 카드와 스크립트를 관리하는 곳입니다. 상단에는 페이지 제목, 설명, 검색창, '⚙️ 카테고리 관리' 버튼, '+ 새 지식 카드' 생성 버튼이 있습니다. 아래로는 전체 통계 바와 카테고리 필터 버튼들, 그리고 지식 카드 목록이 표시됩니다.
- **highlight**: `.max-w-7xl.mx-auto`
- **trigger**: confirm
- **tips**:
  - 지식 카드는 카테고리별로 분류됨
  - 검색 기능으로 빠르게 필요한 지식 찾기 가능
  - 지식 카드는 여러 유형을 지원 (QNA, Script, Guide, Warning, Explanation)

### Step 2: 검색 입력창
- **title**: 지식 카드 검색하기
- **text**: 상단의 검색창(#searchInput)을 사용하여 지식 카드를 키워드로 검색할 수 있습니다. 지식 제목, 내용, 태그 등을 포함한 전문 검색이 지원됩니다. 검색 결과는 실시간으로 반영되며, 검색 조건과 현재 선택된 카테고리/상태 필터와 함께 작동합니다.
- **highlight**: `#searchInput`
- **trigger**: input
- **tips**:
  - 검색은 실시간 반영
  - 대소문자 구분 없음
  - 부분 일치 검색 지원
  - 다른 필터(카테고리, 상태)와 조합 가능

### Step 3: 카테고리 관리 버튼
- **title**: 지식 카테고리 관리
- **text**: '⚙️ 카테고리 관리' 버튼을 클릭하면 모달이 열려 지식 카드 카테고리를 추가, 수정, 삭제할 수 있습니다. 각 카테고리는 이름, 설명, 활성화 여부(대시보드 노출 설정)를 가질 수 있습니다. 카테고리를 활성화하면 대시보드에 표시되어 상담사들이 자주 사용하는 카테고리를 강조할 수 있습니다.
- **highlight**: `#openCatManageBtn`
- **trigger**: click
- **buttons**:
  - selector: `#openCatManageBtn`
  - label: "⚙️ 카테고리 관리"
  - desc: 지식 카테고리 관리 모달 열기
- **tips**:
  - 카테고리는 계층 구조를 지원할 수 있음
  - 카테고리별 지식 카드 개수 표시
  - 대시보드 노출 설정으로 중요한 카테고리 강조

### Step 4: 통계 요약 바
- **title**: 지식 라이브러리 통계 확인
- **text**: 검색/필터 섹션 바로 아래의 통계 요약 바에는 다음 정보들이 표시됩니다: 1)전체 카드 개수, 2)카테고리별 상위 5개 카테고리 및 각 개수, 3)카드 유형별 분포(QNA, Script, Guide, Warning, Explanation). 이 통계로 지식 라이브러리의 현황을 한눈에 파악할 수 있습니다.
- **highlight**: `.bg-gradient-to-r.from-slate-50.to-indigo-50`
- **trigger**: confirm
- **tips**:
  - 상위 5개 카테고리만 표시, 초과 시 '+N개 더' 표시
  - 유형별 분포는 색상 배지로 구분됨
  - 통계는 현재 필터 조건에 맞게 업데이트됨

### Step 5: 카테고리 필터 버튼
- **title**: 카테고리별 필터링
- **text**: 통계 바 아래의 버튼들은 지식 카드를 카테고리별로 필터링합니다. '전체' 버튼은 모든 카드를 표시하며, 각 카테고리 버튼(예: 진단, 치료, 식이요법 등)을 클릭하면 해당 카테고리의 카드만 표시됩니다. 활성 필터 버튼은 파란색 배경으로 강조됩니다.
- **highlight**: `.cat-filter-btn`
- **trigger**: click
- **tips**:
  - '전체' 버튼은 항상 첫 번째에 위치
  - 활성 필터는 파란색(bg-indigo-600)으로 표시
  - 한 번에 한 가지 카테고리만 필터링 가능
  - 필터 선택 시 검색 조건 유지됨

### Step 6: 상태 필터 및 새 카드 생성
- **title**: 상태별 필터링 및 신규 카드 생성
- **text**: 카테고리 필터 다음에는 상태 필터(⚠️ 수정 필요)가 있습니다. 이 버튼을 클릭하면 수정이 필요한 카드만 표시됩니다. 우측 상단의 '+ 새 지식 카드' 버튼을 클릭하면 새로운 지식 카드 생성 페이지로 이동합니다.
- **highlight**: `#filter-needs-revision, #createBtn`
- **trigger**: click
- **buttons**:
  - selector: `#filter-needs-revision`
  - label: "⚠️ 수정 필요"
  - desc: 수정이 필요한 카드만 표시
  - selector: `#createBtn`
  - label: "+ 새 지식 카드"
  - desc: 지식 카드 생성 페이지로 이동
- **tips**:
  - '수정 필요' 필터는 별도 상태 추적을 기반으로 함
  - 새 카드 생성 시 카테고리, 유형, 내용을 입력해야 함

### Step 7: 지식 카드 목록 및 관리
- **title**: 지식 카드 조회 및 편집
- **text**: 필터 적용 후 지식 카드들이 목록 형태로 표시됩니다(페이지당 30개). 각 카드에는 제목, 카테고리, 유형(QNA/Script/Guide/Warning/Explanation - 색상 배지), 상태, 작성자, 마지막 수정일이 표시됩니다. 카드를 클릭하면 상세 편집 페이지로 이동합니다. 우측의 더보기 메뉴(⋮) 또는 직접 클릭으로 다양한 작업을 수행할 수 있습니다.
- **highlight**: `.grid.gap-4`
- **trigger**: click
- **tips**:
  - 목록은 페이지네이션 지원
  - 각 유형별로 색상 배지로 구분됨
  - 상태(Draft/Published/Archived) 표시
  - 카드 클릭으로 편집 페이지 이동
  - 카드별 메뉴에서 삭제, 복제, 공유 등의 작업 가능

### Step 8: 라디에이터 설정 (Radiator Config)
- **title**: 대시보드 노출 설정
- **text**: '대시보드 노출 설정 (Radiator Config)' 섹션(페이지 하단, 기본 접혀있음)을 클릭하여 펼치면, 각 카테고리의 대시보드 노출 여부를 토글로 설정할 수 있습니다. 활성화된 카테고리(파란색)는 운영 대시보드에 표시되어 상담사들이 자주 접근할 수 있도록 합니다. 비활성 카테고리(회색)는 대시보드에서 숨겨집니다.
- **highlight**: `details`
- **trigger**: click
- **tips**:
  - 섹션은 기본 접혀있어 페이지 로드 성능 최적화
  - 토글은 즉시 저장됨 (API 호출)
  - 중요한 카테고리를 활성화하여 접근성 향상
  - 4개 칼럼 그리드로 구성되어 빠른 전환 가능
