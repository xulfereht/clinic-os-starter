# 직원 관리 (/admin/staff + /admin/staff/new) — 투어 시나리오

## 페이지 목적
의료진, 간호팀, 행정 직원을 등록·관리하고 관리자 권한 및 로그인 계정을 설정합니다.

## 시나리오 (32 스텝)

### Step 1: 직원 관리 페이지 진입 - 직원 목록
- **title**: 직원 통합 관리 대시보드
- **text**: 이 페이지는 한의원의 모든 직원을 관리합니다. 기본적으로 "직원 목록" 탭이 선택되어 있으며 모든 직원(의료진, 간호팀, 행정)이 하나의 테이블로 표시됩니다.
- **highlight**: "a:contains('직원 목록')"
- **trigger**: confirm
- **tips**:
  - 기본 정렬: 직군(의료진 → 간호팀 → 행정), 정렬 순서, 이름순
  - 표시 정보: 이름, 직군, 부서/직급, 연락처, 상태
  - 총 직원 수: 테이블 하단에 표시됨

### Step 2: 직원 목록 탭
- **title**: 직원 정보 테이블
- **text**: "직원 목록" 탭은 현재 재직 중이거나 퇴사한 모든 직원을 표시합니다. 각 직원은 이미지, 이름, 직군(의료진/간호팀/행정), 부서, 연락처, 재직 상태로 정리됩니다.
- **highlight**: "a[href*='view=staff']"
- **trigger**: click
- **tips**:
  - 이미지: 없으면 아이콘 표시 (의료진: 👨‍⚕️, 기타: 👤)
  - 직군 색상: 파란색(의료진), 초록색(간호팀), 회색(행정)
  - 상태 색상: 초록색(재직), 회색(퇴사)

### Step 3: 관리자 계정 탭
- **title**: 로그인 권한이 있는 직원 관리
- **text**: "관리자 계정" 탭을 클릭하면 로그인 권한이 있는 직원 목록이 표시됩니다. 최고관리자 계정과 관리자 접근 권한 직원을 분리하여 보여줍니다.
- **highlight**: "a[href*='view=admin']"
- **trigger**: click
- **tips**:
  - 최고관리자: 별도 섹션 (👑 아이콘)
  - 관리자 접근: 이메일이 있는 직원만 표시
  - 권한 관리: 각 직원의 세부 권한은 별도 설정

### Step 4: 직원 검색
- **title**: 직원명 또는 부서로 검색
- **text**: "이름, 부서 검색..." 입력창에 직원명이나 부서명을 입력하면 매칭되는 직원만 테이블에 표시됩니다. 실시간으로 필터링됩니다.
- **highlight**: "#staffSearch"
- **trigger**: input
- **tips**:
  - 부분 검색: "홍" 입력 시 "홍길동", "홍진호" 모두 검색됨
  - 부서 검색: "진료팀" 입력 시 해당 부서 직원 필터링
  - 실시간: 입력 중 바로 갱신됨

### Step 5: 직군 필터
- **title**: 직급별 직원 필터링
- **text**: "전체", "의료진", "간호/진료팀", "일반직원" 드롭다운에서 선택하면 특정 직군의 직원만 표시됩니다. 각 옵션 옆에 해당 직군의 인원 수가 표시됩니다.
- **highlight**: "#typeFilter"
- **trigger**: click
- **tips**:
  - 의료진: 의사, 한의사 등 면허 보유자
  - 간호/진료팀: 간호사, 침술사, 카이로프랙틱 등
  - 일반직원: 행정, 원무, 경영 담당자
  - 인원 수: 각 옵션에 괄호로 표시

### Step 6: 상태 필터
- **title**: 재직 상태별 직원 조회
- **text**: "전체 상태", "재직", "퇴사" 드롭다운에서 선택하면 현재 재직 중인 직원 또는 퇴사한 직원만 필터링하여 표시합니다.
- **highlight**: "#statusFilter"
- **trigger**: click
- **tips**:
  - 재직: is_active = 1
  - 퇴사: is_active = 0
  - 용도: 현재 팀 구성 파악 또는 퇴사자 관리

### Step 7: 전체 선택 체크박스
- **title**: 모든 직원 일괄 선택
- **text**: 테이블 헤더의 체크박스를 클릭하면 현재 필터링된 모든 직원이 선택됩니다 (보이는 행만). 선택 해제 시 모든 선택이 취소됩니다.
- **highlight**: "#selectAll"
- **trigger**: click
- **tips**:
  - 선택 범위: 필터링된 행만 포함
  - 표시: "N명 선택" 텍스트로 선택 수 표시
  - 일괄 삭제: 선택 후 "일괄 삭제" 버튼으로 한 번에 삭제 가능

### Step 8: 개별 행 체크박스
- **title**: 특정 직원 선택/해제
- **text**: 각 행의 왼쪽 체크박스를 클릭하면 해당 직원만 선택하거나 해제합니다. 선택된 직원 수에 따라 "일괄 삭제" 버튼이 나타나고 숨어집니다.
- **highlight**: ".row-checkbox"
- **trigger**: click
- **tips**:
  - 선택 표시: 체크박스에 체크 표시
  - 자동 계산: 선택 수를 자동 계산하여 "N명 선택" 업데이트
  - 일괄 액션: 삭제 외에 향후 추가 가능

### Step 9: 일괄 삭제 버튼
- **title**: 선택된 여러 직원 일괄 삭제
- **text**: 1명 이상의 직원을 선택한 후 "일괄 삭제" 버튼을 클릭하면 확인 대화가 나타나고, "확인"을 누르면 선택된 모든 직원이 삭제됩니다.
- **highlight**: "#bulkDeleteBtn"
- **trigger**: click
- **tips**:
  - 조건: 1명 이상 선택 필수
  - 확인: 삭제 전 "N명의 직원을 삭제하시겠습니까?" 확인 필수
  - 소프트 삭제: deleted_at 필드 설정으로 논리적 삭제 (물리 삭제 아님)

### Step 10: 직원 이름 클릭 - 수정 페이지
- **title**: 직원 정보 수정
- **text**: 테이블에서 "수정" 링크(오른쪽 끝, 호버 시 표시)를 클릭하면 해당 직원의 상세 정보 수정 페이지로 이동합니다.
- **highlight**: "a:contains('수정')"
- **trigger**: click
- **tips**:
  - 위치: 각 행의 맨 오른쪽
  - 표시: 행 호버 시에만 나타남 (평상시 투명)
  - 이동: /admin/staff/{employee_id} 페이지로 이동

### Step 11: 직원 등록 버튼 - 신규 추가
- **title**: 새 직원 등록 페이지로 이동
- **text**: "+ 직원 등록" 파란색 버튼을 클릭하면 새 직원을 등록하는 페이지(/admin/staff/new)로 이동합니다.
- **highlight**: "a:contains('+ 직원 등록')"
- **trigger**: click
- **tips**:
  - 위치: 페이지 우측 상단
  - 색상: 파란색 배경
  - 이동 대상: /admin/staff/new 페이지

### Step 12: 최고관리자 추가 버튼
- **title**: 최고관리자 계정 신규 생성
- **text**: "관리자 계정" 탭에서만 표시되는 "+ 최고관리자" 보라색 버튼을 클릭하면 최고관리자 추가 모달이 열립니다. 최고관리자는 시스템 전체 관리 권한을 가집니다.
- **highlight**: "#addSuperAdminBtn"
- **trigger**: click
- **tips**:
  - 표시 조건: 최고관리자만 보임 (currentUser.role === '최고관리자')
  - 색상: 보라색 배경 (다른 버튼과 구분)
  - 모달: 이름, 이메일, 비밀번호 3개 입력 필드

### Step 13: 최고관리자 모달 - 이름 입력
- **title**: 최고관리자 이름 입력
- **text**: 최고관리자 추가/수정 모달의 "이름" 필드에 최고관리자의 실명을 입력합니다. 시스템 내에서 관리자를 식별하는 데 사용됩니다.
- **highlight**: "#saName"
- **trigger**: input
- **tips**:
  - 필수 입력: 신규 추가 시만
  - 선택 입력: 비밀번호 변경 시
  - 길이 제한: 특별한 제한 없음

### Step 14: 최고관리자 모달 - 이메일 입력
- **title**: 최고관리자 로그인 ID (이메일)
- **text**: "이메일 (로그인 ID)" 필드에 최고관리자가 사용할 이메일을 입력합니다. 이메일은 로그인 아이디로 사용되며 중복될 수 없습니다.
- **highlight**: "#saEmail"
- **trigger**: input
- **tips**:
  - 형식: 올바른 이메일 형식 필수
  - 중복 불가: 시스템에 이미 등록된 이메일 불가
  - 비밀번호 변경: 이메일은 수정 불가 (비활성화됨)

### Step 15: 최고관리자 모달 - 비밀번호 입력
- **title**: 최고관리자 로그인 비밀번호
- **text**: "비밀번호" 필드에 최고관리자의 로그인 비밀번호를 입력합니다. 신규 추가 및 비밀번호 변경 시 모두 필수입니다.
- **highlight**: "#saPassword"
- **trigger**: input
- **tips**:
  - 필수 입력: 항상 필수 (신규/수정 모두)
  - 보안: 입력 시 마스크 처리 (●●●)
  - 재입력 필드 없음: 정확한 입력 권장

### Step 16: 최고관리자 모달 - 저장
- **title**: 최고관리자 저장
- **text**: "저장" 버튼을 클릭하면 입력한 정보로 최고관리자를 생성하거나 비밀번호를 업데이트합니다. 저장 후 페이지가 새로고침되어 업데이트된 목록이 표시됩니다.
- **highlight**: "#saveSuperAdminBtn"
- **trigger**: click
- **tips**:
  - 유효성 검사: 필수 필드 모두 입력 필수
  - 에러 표시: 모달 하단에 에러 메시지 표시
  - 성공: 페이지 자동 새로고침

### Step 17: 직원 등록 페이지 - 기본 정보
- **title**: 새 직원 기본 정보 입력
- **text**: /admin/staff/new 페이지에서 직원의 기본 정보를 입력합니다: 이름(필수), 직군(의료진/간호팀/행정), 부서, 직급, 상태(재직/퇴사), 정렬 순서.
- **highlight**: "form#createForm"
- **trigger**: confirm
- **tips**:
  - 필수 입력: 이름, 직군
  - 선택 입력: 부서, 직급, 상태, 순서
  - 기본값: 상태=재직, 순서=0, 직군=행정/일반팀

### Step 18: 직원 등록 페이지 - 이름 입력
- **title**: 직원 이름
- **text**: "이름" 필드에 직원의 실명을 입력합니다 (필수).
- **highlight**: "input[name='name']"
- **trigger**: input
- **tips**:
  - 필수 입력: 빈 값 불가
  - 예: "홍길동"
  - 중복 허용: 같은 이름의 여러 직원 등록 가능

### Step 19: 직원 등록 페이지 - 직군 선택
- **title**: 직원 분류 (의료진/간호팀/행정)
- **text**: "직군 (Type)" 드롭다운에서 직원의 직급 구분을 선택합니다: "의료진 (의사)", "간호/진료팀", "일반/행정팀". 선택하면 의료진 전용 필드(약력, 전문분야, 학력, 경력)가 표시됩니다.
- **highlight**: "#typeSelect"
- **trigger**: click
- **tips**:
  - 의료진: 약력, 전문분야, 학력, 경력 필드 추가 표시
  - 간호팀: 기본 정보만 표시
  - 행정팀: 기본 정보만 표시
  - 변경: 선택 후 즉시 필드 표시/숨김

### Step 20: 직원 등록 페이지 - 프로필 이미지 업로드
- **title**: 직원 프로필 사진 업로드
- **text**: "프로필 이미지" ImageUpload 컴포넌트를 클릭하여 직원의 프로필 사진을 업로드합니다. 클릭하거나 드래그앤드롭으로 파일을 선택할 수 있습니다.
- **highlight**: "ImageUpload"
- **trigger**: click
- **tips**:
  - 지원 형식: JPG, PNG, WebP 등
  - 크기 제한: 권장 1-5MB
  - 저장 경로: public/local/images/staff/
  - 기본: 이미지 없으면 아이콘으로 표시

### Step 21: 관리자 접근 권한 - 이메일 (로그인 ID)
- **title**: 직원 로그인 이메일 주소
- **text**: "관리자 접근 권한" 섹션의 "관리자 이메일 (ID)" 필드에 직원이 사용할 관리자 로그인 이메일을 입력합니다 (선택사항). 이메일을 입력하면 이 직원이 관리자 페이지에 접근할 수 있게 됩니다.
- **highlight**: "input[name='email']"
- **trigger**: input
- **tips**:
  - 선택 입력: 관리자 권한 불필요 시 빈 채로 두기
  - 중복 불가: 이미 등록된 이메일 사용 불가
  - 형식: 올바른 이메일 주소

### Step 22: 관리자 접근 권한 - 비밀번호
- **title**: 관리자 로그인 비밀번호
- **text**: "관리자 접근 권한" 섹션의 "비밀번호" 필드에 이 직원의 관리자 로그인 비밀번호를 입력합니다 (선택사항, 이메일 입력 시 권장).
- **highlight**: "input[name='password']"
- **trigger**: input
- **tips**:
  - 선택 입력: 이메일 입력 시 권장, 필수는 아님
  - 보안: 입력 시 마스크 처리
  - 전달: 이메일 입력 시 함께 입력 권장

### Step 23: 권한 프리셋 - 선택 및 적용
- **title**: 직무별 권한 템플릿 선택
- **text**: "권한 프리셋" 섹션의 드롭다운에서 직무에 맞는 권한 프리셋(관리자, 원장, 부원장, 매니저, 데스크, 치료사 등)을 선택합니다. "적용" 버튼을 클릭하면 해당 프리셋의 권한이 세부 권한 체크박스에 자동으로 적용됩니다.
- **highlight**: "#presetSelect"
- **trigger**: click
- **tips**:
  - 기본: "-- 프리셋 선택 --"로 시작
  - 자동 적용: "적용" 버튼 클릭 시 체크박스 자동 체크
  - 설명: 각 프리셋의 설명이 아래 박스에 표시됨

### Step 24: 권한 프리셋 - 설명 확인
- **title**: 선택한 프리셋의 권한 범위 확인
- **text**: 프리셋을 선택하면 프리셋 설명이 파란색 박스에 표시되어, 이 프리셋이 어떤 권한을 포함하는지 미리 확인할 수 있습니다.
- **highlight**: "#presetDescription"
- **trigger**: confirm
- **tips**:
  - 예: "원장 프리셋: 모든 관리 페이지 접근 + 쓰기 권한"
  - 표시 조건: 프리셋 선택 시만 표시
  - 숨김: 프리셋 해제 시 다시 숨음

### Step 25: 세부 권한 - 펼치기/접기
- **title**: 상세 권한 설정 보기/숨기기
- **text**: "세부 권한 설정" 섹션의 "접기" 또는 "펼치기" 링크를 클릭하면 상세 권한 체크박스 목록이 표시되거나 숨겨집니다. 프리셋만으로 부족할 때 개별 권한을 조정할 수 있습니다.
- **highlight**: "#togglePermissions"
- **trigger**: click
- **tips**:
  - 기본: 숨김 (흰색 박스 표시 안 함)
  - 프리셋 적용: 자동으로 펼쳐짐
  - 수동 조정: 그룹별로 "전체 선택/해제" 가능

### Step 26: 세부 권한 - 리소스별 체크박스
- **title**: 리소스별 조회/수정 권한 설정
- **text**: 세부 권한 섹션은 여러 리소스 그룹(환자관리, 결제, 예약, 일정 등)으로 구분되어 있으며, 각 리소스의 "조회"와 "수정" 권한을 개별 체크박스로 설정할 수 있습니다.
- **highlight**: ".grid.grid-cols-1.md-grid-cols-2.gap-2"
- **trigger**: click
- **tips**:
  - 조회: 해당 페이지/기능 읽기 권한
  - 수정: 데이터 추가/변경/삭제 권한
  - 그룹 토글: 각 그룹 우측의 "전체 선택/해제" 버튼

### Step 27: 세부 권한 - 그룹별 전체 선택/해제
- **title**: 권한 그룹 단위로 일괄 설정
- **text**: 각 권한 그룹 헤더의 "전체 선택/해제" 버튼을 클릭하면 그 그룹의 모든 조회/수정 권한이 한 번에 체크되거나 해제됩니다.
- **highlight**: ".toggle-group-btn"
- **trigger**: click
- **tips**:
  - 토글: 클릭할 때마다 선택 ↔ 해제 반복
  - 범위: 해당 그룹의 모든 항목에만 적용
  - 효율: 프리셋 없이 빠르게 권한 설정 가능

### Step 28: 튜토리얼 투어 - 활성화 토글
- **title**: 신규 직원 로그인 시 투어 자동 실행
- **text**: "튜토리얼 투어" 섹션의 "활성화" 체크박스를 체크하면, 이 직원이 처음 로그인할 때 역할에 맞는 가이드 투어가 자동으로 시작됩니다 (기본 활성화).
- **highlight**: "#enableTourCheckbox"
- **trigger**: click
- **tips**:
  - 기본값: 체크됨 (활성화)
  - 자동 선택: 프리셋 선택 시 자동으로 적합한 투어 선택
  - 투어 ID: 프리셋별로 다른 투어 매핑됨 (fullcourse, director, desk 등)

### Step 29: 의료진 상세 정보 - 약력
- **title**: 의료진 경력 및 소개
- **text**: 직군을 "의료진"으로 선택했을 때만 나타나는 "의료진 상세 정보" 섹션의 "약력 (Bio)" 텍스트에어리어에 의사의 전문 경력, 소개 문구 등을 입력합니다.
- **highlight**: "textarea[name='bio']"
- **trigger**: input
- **tips**:
  - 선택 입력: 필수 아님
  - 길이: 제한 없음 (길수록 좋음)
  - 용도: 클리닉 홈페이지 의료진 소개 페이지에 표시

### Step 30: 의료진 상세 정보 - 전문분야
- **title**: 의료진 전공 분야
- **text**: "전문 분야 (쉼표로 구분)" 필드에 의료진의 전문 분야를 쉼표로 구분하여 입력합니다. 예: "레이저, 보톡스, 필러, 영양치료".
- **highlight**: "input[name='specialties']"
- **trigger**: input
- **tips**:
  - 형식: 쉼표로 분리 (예: "항암, 면역, 통증")
  - 자동 처리: 입력값이 JSON 배열로 변환되어 저장됨
  - 표시: 홈페이지의 의료진 카드에 표시됨

### Step 31: 의료진 상세 정보 - 학력 및 경력
- **title**: 의료진 학위 및 경력 정보
- **text**: "학력" 필드에 최고 학위(석사, 박사 등), "경력" 필드에 주요 경력을 입력합니다. 자유 형식의 텍스트 필드로 사용 가능합니다.
- **highlight**: "input[name='education'], input[name='career']"
- **trigger**: input
- **tips**:
  - 학력: "한의학 박사", "침구학 석사" 등
  - 경력: "15년 임상경험", "대학병원 근무" 등
  - 용도: 프로필 페이지 및 증명서 용도

### Step 32: 직원 등록 - 최종 저장
- **title**: 입력한 정보로 직원 등록
- **text**: 모든 필수 정보(이름, 직군)를 입력한 후 페이지 하단의 "등록하기" 버튼을 클릭하면 새 직원이 시스템에 추가됩니다. 저장 후 직원 목록 페이지로 자동 이동합니다.
- **highlight**: "button:contains('등록하기')"
- **trigger**: click
- **tips**:
  - 유효성 검사: 필수 필드 체크 후 저장
  - 성공: 토스트 메시지 "등록되었습니다." 표시
  - 실패: 에러 메시지 표시 + 페이지 유지 (정보 보존)
  - 취소: "취소" 버튼으로 변경사항 폐기 후 목록 페이지로 돌아가기
