# 디자인 설정 (/admin/design.astro) — 투어 시나리오

## 페이지 목적
한의원 웹사이트의 전체 디자인을 3단계로 설정합니다: (1) 스킨(색감/분위기) 선택, (2) 세부 설정(브랜드 컬러, 라운딩, 밀도), (3) 고급 CSS 오버라이드. 모든 설정은 한 화면에서 관리되며, 프리뷰를 통해 실시간 확인합니다.

---

## 시나리오 (22 스텝)

### Step 1: 페이지 입장 및 3단계 구조 이해
- **title**: 디자인 설정 페이지 소개
- **text**: 이 페이지는 좌측(설정)과 우측(프리뷰)의 2열 레이아웃입니다. 좌측에는 3개 섹션이 있습니다: (1) 메인 스타일 선택 (스킨 카드), (2) 세부 설정 (브랜드 컬러 등 라디오 버튼), (3) 고급 CSS 설정 (텍스트 입력). 우측에는 설정 변경에 따른 실시간 프리뷰가 표시됩니다.
- **highlight**: `.mx-auto.max-w-\[1480px\]`
- **trigger**: confirm
- **tips**:
  - 왼쪽부터 차례로 따라가면서 설정하면 됩니다
  - 프리뷰는 실시간으로 업데이트됩니다
  - 모든 설정은 페이지를 떠나지 않고 이곳에서 완료할 수 있습니다

### Step 2: 스킨 스토어 안내 배너
- **title**: 추가 스킨 다운로드
- **text**: 상단의 보라색 배너에는 "HQ curated 스킨을 더 가져오려면" 메시지와 함께 "스킨 스토어 열기" 버튼이 있습니다. 더 많은 스킨 프리셋을 받으려면 스킨 스토어에서 설치하면, 자동으로 이 페이지에 나타납니다.
- **highlight**: `.mb-6.flex.flex-col.gap-3.rounded-3xl.border.border-violet-200`
- **trigger**: click
- **buttons**:
  - selector: `a[href="/admin/skins/store"]`
  - label: "스킨 스토어 열기"
  - desc: "HQ에서 제공하는 추가 스킨 프리셋을 설치합니다"
- **tips**:
  - 스킨 스토어는 별도 페이지입니다
  - 설치 후 이 페이지에 자동으로 새 스킨이 추가됩니다
  - 처음에는 기본 스킨들만 표시됩니다

### Step 3: Section 1 — 메인 스타일 선택 헤더
- **title**: 스킨 선택 섹션 소개
- **text**: "1. 메인 스타일 선택"이라는 제목 아래에 "스킨 이름, 출처, 기본 프리셋을 읽기 쉽게 정리했습니다"라는 설명이 있습니다. 이 섹션에서 스킨(전체 디자인 테마)을 선택합니다.
- **highlight**: `section:first-of-type`

- **tips**:
  - 스킨은 색상, 서체, 분위기를 통합한 디자인 패키지입니다

### Step 4: 스킨 카드 라디오 버튼
- **title**: 스킨 선택 (라디오 버튼)
- **text**: 각 스킨은 라디오 버튼으로 표시됩니다. 카드의 아무 곳이나 클릭하거나, 우측의 작은 원형 버튼(라디오)을 클릭하면 그 스킨이 선택됩니다. 선택되면 카드의 테두리가 파란색으로 바뀌고 체크 표시가 나타납니다.
- **highlight**: `label[data-testid^="skin-card-"]`
- **trigger**: click
- **buttons**:
  - selector: `input[name="skin"]`
  - label: "스킨 선택"
  - desc: "웹사이트의 기본 색감 및 분위기를 결정합니다"
- **tips**:
  - 각 스킨 카드는 해당 스킨의 색상을 미리 보여줍니다
  - 카드 위의 수평 그래디언트 바는 해당 스킨의 액센트 컬러들을 표현합니다
  - 호버 시 카드가 살짝 위로 올라오며 그림자가 커집니다 (상호작용 시각화)

### Step 5: 스킨 카드 내용 읽기
- **title**: 스킨 카드의 각 정보 항목
- **text**: 각 스킨 카드에는 다음 정보들이 있습니다:
  - **상단 배지**: "로컬", "스토어", "코어" (스킨의 출처)
  - **스킨명**: 한글 이름 (예: "클린 화이트")
  - **ID**: 영문 고유명 (예: "clinic-light")
  - **설명**: 스킨의 특징을 간단히 설명
  - **색상 샘플**: 3개 컬러 박스 (액센트, 서포트, 칩 배경)
  - **기본 프리셋 태그**: "Hue · Blue", "Radius · Medium", "Density · Normal" 등
- **highlight**: `label.group.relative.cursor-pointer`
- **trigger**: confirm
- **tips**:
  - 출처 배지로 스킨이 어디서 왔는지 알 수 있습니다
  - 색상 샘플은 스킨을 선택했을 때의 기본 색감입니다
  - 프리셋 태그는 이 스킨의 추천 세부 설정을 나타냅니다

### Step 6: 추가 예정 스킨 목록
- **title**: 곧 출시될 스킨 미리보기
- **text**: 선택 가능한 스킨 아래에 "추가 예정 프리셋" 섹션이 있으면, 개발 중인 스킨들이 표시됩니다. 이들은 아직 선택할 수 없지만 곧 이용 가능할 것입니다.
- **highlight**: `.mt-4.rounded-2xl.border.border-dashed`
- **trigger**: confirm
- **tips**:
  - 추가 예정 스킨의 정확한 출시 일정은 문의하면 안내해드립니다

### Step 7: Section 2 — 세부 설정 헤더
- **title**: 세부 설정 섹션 소개
- **text**: "2. 세부 설정"이라는 제목 아래에 "선택한 skin 위에 추가로 tone 을 미세 조정합니다"라는 설명이 있습니다. 스킨을 선택한 후, 여기서 브랜드 컬러, 라운딩, 밀도를 커스터마이징합니다.
- **highlight**: `section.rounded-\[1.75rem\].border.border-slate-200.bg-white.p-5.shadow-sm` (두 번째)
- **trigger**: confirm
- **tips**:
  - 스킨은 큰 틀, 세부 설정은 미세 조정입니다
  - 세부 설정을 통해 같은 스킨으로도 다양한 룩을 만들 수 있습니다

### Step 8: 브랜드 컬러 라디오 그룹
- **title**: 브랜드 주요 색상 선택
- **text**: "브랜드 컬러"라는 라벨 아래에 5개의 라디오 버튼이 있습니다 (각 컬러별로): 뉴트럴 슬레이트(회색), 메디컬 블루(파랑), 웰니스 그린(초록), 모던 틸(청록), 웜 브라운(갈색). 하나를 선택하면 프리뷰의 전체 색감이 변합니다.
- **highlight**: `div:has([name="brandHue"])`
- **trigger**: click
- **buttons**:
  - selector: `input[name="brandHue"]`
  - label: "브랜드 컬러"
  - desc: "웹사이트의 주요 액센트 색상을 선택합니다"
- **tips**:
  - 각 라디오 옵션은 컬러 점 아이콘과 한글명, 영문명으로 표시됩니다
  - 예: "메디컬 블루 (Blue)" 옆에는 파란 점이 있습니다
  - 선택하면 라디오 버튼이 파란 배경으로 강조됩니다
  - 같은 스킨(예: clinicLight)이라도 브랜드 컬러에 따라 느낌이 크게 달라집니다

### Step 9: 모서리 둥글기 (라운딩)
- **title**: UI 요소의 모서리 둥글기 설정
- **text**: "모서리 둥글기"라는 라벨 아래에 5개의 라디오 버튼이 있습니다: 없음(정사각형), 작게(조금 둥글게), 보통(적당히), 부드럽게(둥글게), 캡슐형(매우 둥굼). 각 옵션은 미리보기 박스로 시각화되어 있습니다.
- **highlight**: `div:has([name="rounding"])`
- **trigger**: click
- **buttons**:
  - selector: `input[name="rounding"]`
  - label: "모서리 둥글기"
  - desc: "버튼, 카드, 입력 필드의 모서리 반경을 조정합니다"
- **tips**:
  - 각 옵션마다 작은 사각형 미리보기가 있어서 어떻게 보일지 알 수 있습니다
  - "캡슐형"은 버튼이 알약처럼 보입니다
  - 둥글기는 전체 UI의 모던함/캐주얼함을 결정합니다

### Step 10: 밀도 (Density)
- **title**: 화면 여백 및 컴팩트함 설정
- **text**: "밀도"라는 라벨 아래에 4개의 라디오 버튼이 있습니다: 촘촘함(여백 최소), 보통(권장), 넓게(여백 증가), 여백 강조(최대 여백). 선택하면 페이지의 간격, 패딩, 마진이 조정되어 더 빡빡하거나 여유로운 느낌이 됩니다.
- **highlight**: `div:has([name="density"])`
- **trigger**: click
- **buttons**:
  - selector: `input[name="density"]`
  - label: "밀도"
  - desc: "요소 간 여백을 조정하여 화면의 여유감을 결정합니다"
- **tips**:
  - 촘촘함 = 모바일 같은 컴팩트 느낌
  - 보통 = 가장 많이 사용되는 표준 설정
  - 여백 강조 = 럭셔리하고 고급스러운 느낌
  - 프리뷰에서 즉시 효과를 확인할 수 있습니다

### Step 11: Section 3 — 고급 CSS 설정 헤더
- **title**: CSS 변수 오버라이드
- **text**: "3. 고급 CSS 설정"이라는 제목 아래에 "세부 스타일을 변수 단위로 미세 조정합니다"라는 설명이 있습니다. UI 스킨과 세부 설정으로도 부족하면, 여기서 CSS 변수를 직접 입력하여 디자인을 더 세밀하게 조정할 수 있습니다.
- **highlight**: `section.rounded-\[1.75rem\].border.border-slate-200.bg-white.p-5.shadow-sm` (세 번째)
- **trigger**: confirm
- **tips**:
  - CSS 변수를 수정할 때는 조심해야 합니다 (잘못하면 디자인이 깨질 수 있습니다)
  - 기본 설정이 충분하면 이 섹션은 건너뛰어도 됩니다

### Step 12: CSS 액센트 컬러 입력
- **title**: 액센트 색상 커스텀 입력
- **text**: "액센트 색상 (`--accent`)" 입력 필드에 16진수 색상값(hex color, 예: #2563eb)을 입력합니다. 이 색상은 버튼, 링크, 강조 요소 등에 사용됩니다.
- **highlight**: `input[name="css_accent"]`
- **trigger**: input
- **buttons**:
  - selector: `input[name="css_accent"]`
  - label: "액센트 색상"
  - desc: "버튼, 링크, 강조 요소의 색상을 16진수(#RRGGBB)로 지정합니다"
- **tips**:
  - 16진수 형식: #RRGGBB (6자리) 또는 #RGB (3자리)
  - 예: #2563eb (파랑), #16a34a (초록), #dc2626 (빨강)
  - 잘못된 형식을 입력하면 무시됩니다

### Step 13: CSS 폰트 헤딩 입력
- **title**: 제목 폰트(Heading Font) 설정
- **text**: "폰트 헤딩 (`--font-heading`)" 입력 필드에 CSS 폰트 패밀리를 입력합니다. 예: 'Noto Serif KR', serif 또는 'Pretendard Variable', sans-serif. 이 폰트는 모든 제목(h1, h2 등)에 적용됩니다.
- **highlight**: `input[name="css_font_heading"]`
- **trigger**: input
- **buttons**:
  - selector: `input[name="css_font_heading"]`
  - label: "폰트 헤딩"
  - desc: "제목에 사용될 폰트를 지정합니다 (CSS 포맷)"
- **tips**:
  - CSS의 font-family 값을 그대로 입력합니다
  - 예: "'Noto Sans KR', sans-serif" 또는 "Georgia, serif"
  - 띄어쓰기와 따옴표를 정확히 입력해야 합니다
  - 시스템에 없는 폰트를 입력하면 기본 폰트로 대체됩니다

### Step 14: 커스텀 CSS 텍스트 영역
- **title**: 추가 CSS 코드 입력
- **text**: "커스텀 CSS" 텍스트 영역에 CSS 코드를 직접 작성할 수 있습니다. 예: `/* 추가 CSS */ .custom-class { color: red; }`. 여기의 CSS는 모든 다른 스타일을 덮어씁니다(높은 우선순위).
- **highlight**: `textarea[name="css_custom"]`
- **trigger**: input
- **buttons**:
  - selector: `textarea[name="css_custom"]`
  - label: "커스텀 CSS"
  - desc: "추가 또는 덮어쓸 CSS 코드를 입력합니다"
- **tips**:
  - 여러 줄로 작성할 수 있습니다
  - 주석도 가능합니다 (/* ... */)
  - 실수로 입력하면 웹사이트 스타일이 깨질 수 있으니 주의하세요
  - "초기화" 버튼으로 원래 상태로 돌릴 수 있습니다

### Step 15: 커스텀 스킨 만들기 안내
- **title**: 커스텀 스킨 개발
- **text**: 위의 스킨 목록에 원하는 것이 없으면, 터미널에서 `/dev-skin` 스킬을 사용하여 브랜드 컬러, 서체, 분위기를 맞춘 커스텀 스킨을 만들 수 있습니다. 만들어진 스킨은 자동으로 이 목록에 나타납니다.
- **highlight**: `.rounded-2xl.border.border-amber-200.bg-amber-50`
- **trigger**: confirm
- **tips**:
  - 스킬 이름: `/dev-skin`
  - 터미널에서 명령어를 실행하면 대화형으로 스킨을 만들 수 있습니다
  - 만들어진 스킨은 "로컬" 출처로 표시됩니다

### Step 16: 초기화 버튼
- **title**: 모든 설정을 기본값으로 리셋

- **buttons**:
  - selector: `#reset-btn`

  - 저장 실패 시 오류 메시지가 토스트로 표시됩니다
  - 저장 후 웹사이트 곳곳에서 변경사항을 확인할 수 있습니다

### Step 18: 우측 프리뷰 영역 소개
- **title**: 실시간 프리뷰
- **text**: 페이지 우측에는 웹사이트의 실시간 프리뷰가 표시됩니다. 좌측에서 설정을 변경하면 우측의 프리뷰도 즉시 업데이트되어 변경사항을 미리 확인할 수 있습니다.
- **highlight**: `iframe`, `.xl\:grid-cols-\[25rem_minmax\(0,1fr\)\]` (오른쪽 부분)
- **trigger**: confirm
- **tips**:
  - 프리뷰는 메인 홈페이지뿐 아니라 여러 페이지 섹션을 보여줍니다
  - 모바일/데스크톱 뷰 전환이 가능할 수 있습니다
  - 프리뷰는 설정만으로 변경되며, 저장하지 않아도 볼 수 있습니다

### Step 19: 프리뷰 페이지 선택 (필요하면)
- **title**: 프리뷰할 페이지 섹션 선택
- **text**: 프리뷰 상단에 드롭다운이나 탭이 있으면 ("전체 페이지", "히어로", "카드 섹션" 등), 어느 부분을 보고 싶은지 선택할 수 있습니다. 각 섹션마다 디자인이 어떻게 반영되는지 확인합니다.
- **highlight**: `iframe` 주변의 선택 요소들
- **trigger**: click
- **tips**:
  - 프리뷰 페이지들은 미리 만들어진 샘플이므로 실제 데이터와는 다를 수 있습니다
  - 여러 섹션을 확인하면서 일관성이 있는지 검토하세요

### Step 20: 모바일 뷰 프리뷰 (필요하면)
- **title**: 모바일 화면 크기로 프리뷰
- **text**: 프리뷰 도구에 모바일 뷰 버튼이 있으면, 클릭하여 모바일 크기(375px, 414px 등)로 프리뷰할 수 있습니다. 반응형 디자인이 제대로 작동하는지 확인합니다.
- **highlight**: `iframe`
- **trigger**: click
- **tips**:
  - 모바일 뷰는 태블릿, 데스크톱과 다르게 보일 수 있습니다
  - 프리뷰에서 반응형을 확인하고 만족하면 저장하세요

### Step 21: 외관 설정 페이지와의 관계
- **title**: 기존 외관 설정 페이지와의 통합
- **text**: 설명 텍스트에 "기존 appearance 페이지의 기능도 여기에서 같이 저장됩니다"라고 명시되어 있습니다. 즉, `/admin/settings/appearance`에서 저장하던 CSS 설정들이 이 페이지의 CSS 필드에 통합되었습니다.
- **highlight**: `p:has(a[href="/admin/settings/appearance"])`
- **trigger**: confirm
- **tips**:
  - 기존에 외관 페이지를 사용했다면, 이제는 여기서만 관리하면 됩니다
  - 두 곳에서 동시에 저장할 필요 없습니다

### Step 22: 저장 후 확인
- **title**: 저장 완료 및 변경사항 확인
- **text**: 저장 버튼을 클릭하면 로딩 표시 후 완료됩니다. 그 후 웹사이트의 모든 페이지(홈페이지, 블로그, 프로그램 등)에서 새로운 디자인이 적용된 것을 확인할 수 있습니다.
- **highlight**: `#save-btn`
- **trigger**: confirm
- **tips**:
  - 변경이 즉시 반영되지 않으면 브라우저 캐시를 지우고 새로고침(Ctrl+Shift+R)하세요
  - 여러 기기(PC, 모바일)에서도 확인하여 일관성을 검토하세요

## 통합 워크플로우 예시

1. **스킨 선택**: 카드 클릭 → 스킨 선택 (자동 프리뷰)
2. **세부 설정**: 브랜드 컬러 선택 → 라운딩 선택 → 밀도 선택
3. **CSS 커스터마이징**: 액센트 컬러 입력 → 폰트 입력 → 커스텀 CSS 추가
4. **저장**: "저장" 클릭 → 완료 후 웹사이트 전체에 반영
5. **프리뷰 확인**: 우측 프리뷰에서 실시간으로 변경 확인

## 자주 묻는 상황

| 상황 | 해결 방법 |
|------|---------|
| 저장 후에도 웹사이트에 변경이 안 보여요 | 브라우저 캐시를 지우고 새로고침(Ctrl+Shift+R)하세요. 여러 기기에서는 캐시가 따로 있을 수 있습니다 |
| 프리뷰와 실제 웹사이트가 달라요 | 프리뷰는 샘플 데이터를 사용하므로 실제 콘텐츠(글, 프로그램)에 따라 보기가 다를 수 있습니다 |
| CSS를 잘못 입력해서 디자인이 깨졌어요 | CSS 필드를 비우고 저장하면 기본값으로 복원됩니다. 또는 "초기화" 버튼을 사용하세요 |
| 여러 스킨을 조합해서 사용하고 싶어요 | 한 번에 하나의 스킨만 선택 가능합니다. 원하는 조합이 없으면 `/dev-skin` 스킬로 커스텀 스킨을 만드세요 |
| 모바일과 데스크톱에서 다르게 보이게 하고 싶어요 | CSS 미디어 쿼리를 커스텀 CSS 필드에 작성하면 됩니다 (예: `@media (max-width: 768px) { ... }`) |
