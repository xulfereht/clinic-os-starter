# 약관 및 정책 관리 (/admin/settings/terms) — 투어 시나리오

## 페이지 목적
한의원에서 필수 정책(개인정보처리방침, 마케팅 동의, 비대면 진료 동의서, 이용약관) 및 커스텀 정책을 생성/관리하고 버전 관리하는 페이지

## 시나리오 (7 스텝)

### Step 1: 페이지 진입 및 주요 섹션 이해
- **title**: 약관 및 정책 관리 페이지 개요
- **text**: 이 페이지는 한의원 운영에 필수적인 법적 정책들을 관리하는 곳입니다. 상단에는 페이지 제목과 새로운 사용자 정의 정책을 추가할 수 있는 버튼이 있습니다. 아래로 내려가면 '주요 운영 정책'과 '기타 정책(Custom)' 두 섹션으로 나뉩니다.
- **highlight**: `.max-w-6xl.mx-auto`
- **trigger**: confirm
- **tips**:
  - 주요 운영 정책: 개인정보처리방침, 마케팅 동의, 비대면 진료 동의서, 이용약관 (4개 필수)
  - 각 정책은 카드 형태로 표시됨
  - 아직 등록되지 않은 정책은 회색 배경, 등록된 정책은 흰색 배경

### Step 2: 사용자 정의 정책 추가 버튼
- **title**: 커스텀 정책 추가하기
- **text**: 우측 상단의 '+ 사용자 정의 정책 추가' 버튼을 클릭하면 새 정책을 생성할 수 있는 모달창이 열립니다. 제목, Slug(고유ID), 설명을 입력하여 새로운 정책을 만들 수 있습니다.
- **highlight**: `button[onclick*="createTermModal"]`
- **trigger**: click
- **buttons**: 
  - selector: `button[onclick*="createTermModal"]`
  - label: "+ 사용자 정의 정책 추가"
  - desc: 새로운 커스텀 정책 생성 모달 열기
- **tips**:
  - Slug는 자동으로 소문자 및 하이픈만 허용하도록 변환됨
  - 예: "privacy-policy", "marketing-consent"

### Step 3: 주요 운영 정책 카드 - 미등록 상태
- **title**: 필수 정책 생성하기
- **text**: '주요 운영 정책' 섹션에서 아직 등록되지 않은 정책 카드(회색 배경)를 보실 수 있습니다. 각 카드에는 정책 이름, 설명, 아이콘이 표시되어 있으며, 하단에 '생성하기' 버튼이 있습니다. 이 버튼을 클릭하면 시스템 기본 템플릿으로 정책이 생성됩니다.
- **highlight**: `.grid.md\\:grid-cols-2.lg\\:grid-cols-4`
- **trigger**: confirm
- **tips**:
  - 각 필수 정책마다 기본 템플릿이 준비되어 있음
  - 템플릿에는 {{clinic_name}}과 같은 변수가 포함될 수 있음
  - 생성 후 '관리하기' 버튼으로 내용 수정 가능

### Step 4: 주요 운영 정책 카드 - 등록 완료 상태
- **title**: 등록된 정책 관리하기
- **text**: 정책이 생성된 후에는 카드 배경이 흰색으로 변경되며, 하단에 현재 버전과 '관리하기' 버튼이 표시됩니다. 버전 정보(예: v1, v2)를 통해 정책의 수정 이력을 추적할 수 있습니다. '관리하기' 버튼을 클릭하면 해당 정책의 상세 관리 페이지로 이동합니다.
- **highlight**: `.grid.md\\:grid-cols-2.lg\\:grid-cols-4 > div:has(a[href*="/admin/settings/terms/"])`
- **trigger**: confirm
- **tips**:
  - 버전은 각 수정마다 자동으로 증가함
  - '미배포' 상태는 아직 내용이 저장되지 않은 상태
  - 각 정책별로 다른 버전을 동시에 운영 가능

### Step 5: 기타 정책(Custom) 섹션
- **title**: 커스텀 정책 관리
- **text**: '기타 정책(Custom)' 섹션에서는 한의원이 직접 추가한 커스텀 정책들이 카드 형태로 표시됩니다. 각 카드에는 정책 제목, Slug, 설명, 최종 수정일, 현재 버전이 표시됩니다. 카드를 클릭하거나 제목 링크를 클릭하면 상세 관리 페이지로 이동합니다.
- **highlight**: `h2:has-text("기타 정책"):next(.grid)`
- **trigger**: confirm
- **tips**:
  - 커스텀 정책도 필수 정책과 동일하게 버전 관리됨
  - Slug는 시스템에서 고유성을 자동으로 검증함
  - 커스텀 정책도 접수 폼에서 사용 가능

### Step 6: 정책 카드 상호작용
- **title**: 정책 조회 및 링크
- **text**: 정책 카드의 제목, Slug, '관리 / FAQ' 링크 등 모든 요소가 클릭 가능합니다. 제목을 클릭하면 정책 상세 편집 페이지로, Slug는 정책 페이지 미리보기로 이동할 수 있습니다. 각 정책의 최종 수정일과 버전을 통해 업데이트 현황을 한눈에 파악할 수 있습니다.
- **highlight**: `.grid.md\\:grid-cols-2.lg\\:grid-cols-3 a[href*="/admin/settings/terms/"]`
- **trigger**: confirm
- **tips**:
  - 모든 정책 카드의 배경색 변화는 호버 상태를 시각화함
  - 정책 목록은 생성 날짜 역순으로 정렬됨
  - 정책 미리보기는 새 탭에서 열림

### Step 7: 모달 폼 작성
- **title**: 정책 생성 모달 양식 입력
- **text**: '새 정책 추가' 모달에서는 제목, Slug, 설명 세 가지 필수 입력 항목이 있습니다. 제목은 자유로운 텍스트 입력, Slug는 소문자와 하이픈만 허용되며(자동 변환), 설명은 선택사항입니다. 모두 입력 후 '추가하기' 버튼을 클릭하면 새 정책이 생성되고 페이지가 새로고침됩니다.
- **highlight**: `#createTermModal form`
- **trigger**: input
- **buttons**:
  - selector: `#createTermModal button[type="submit"]`
  - label: "추가하기"
  - desc: 새 정책 추가 모달 제출
  - selector: `#createTermModal button[type="button"]`
  - label: "취소"
  - desc: 모달 닫기
- **tips**:
  - Slug 입력 시 자동으로 소문자로 변환됨
  - 예: "이용약관" → slug "이용약관" 가능, 특수문자는 하이픈으로 변환
  - 동일한 Slug는 중복 생성 불가
