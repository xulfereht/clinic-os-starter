# 언어 관리 (/admin/settings/languages) — 투어 시나리오

## 페이지 목적
사이트에서 지원할 언어를 추가/활성화/비활성화하고 관리하는 페이지

## 시나리오 (6 스텝)

### Step 1: 언어 관리 페이지 진입
- **title**: 언어 관리 페이지 개요
- **text**: 이 페이지는 다국어 지원을 위한 언어 설정을 관리하는 곳입니다. 상단에는 페이지 제목과 설명이 있으며, 아래로 '현재 지원 언어' 섹션과 '새 언어 추가' 폼이 배치되어 있습니다. 현재 지원하는 모든 언어의 활성화 상태를 한눈에 볼 수 있습니다.
- **highlight**: `.max-w-7xl.mx-auto`
- **trigger**: confirm
- **tips**:
  - 한국어(ko)는 기본 언어로 항상 활성화됨
  - 각 언어는 개별적으로 활성화/비활성화 가능
  - 비활성 언어로도 번역할 수 있지만 사이트에 표시되지 않음

### Step 2: 현재 지원 언어 목록
- **title**: 지원 언어 목록 조회
- **text**: '현재 지원 언어' 섹션에는 시스템에서 지원하는 모든 언어가 카드 형태로 나열됩니다. 각 카드에는 다음 정보가 표시됩니다: 1)국기 이모지, 2)네이티브 이름(예: 한국어, Tiếng Việt), 3)영어 이름과 언어 코드, 4)상태 배지 또는 토글 버튼. 한국어는 '기본 언어' 배지로 고정되어 있습니다.
- **highlight**: `#languages-list`
- **trigger**: confirm
- **tips**:
  - 각 카드는 언어별로 구분됨
  - 국기 이모지로 시각적 식별 용이
  - 언어 코드는 ISO 639-1 또는 BCP 47 형식
  - 네이티브 이름은 해당 언어로 표시됨

### Step 3: 언어 활성화/비활성화 토글
- **title**: 언어 활성화 상태 관리
- **text**: 각 언어 카드의 우측에는 활성화 상태를 제어하는 토글 버튼이 있습니다. 활성화된 언어는 초록색 '✅ 활성화됨' 버튼으로 표시되고, 비활성화된 언어는 회색 '⏸️ 비활성화' 버튼으로 표시됩니다. 한국어는 '기본 언어' 배지로 고정되어 토글 불가능합니다.
- **highlight**: `.toggle-btn`
- **trigger**: click
- **tips**:
  - 토글 클릭 시 즉시 API 호출로 상태 변경
  - 변경 중에는 버튼에 '저장 중...' 표시
  - 토글 완료 후 색상과 텍스트가 업데이트됨
  - 비활성화된 언어도 번역 데이터는 보존됨

### Step 4: 새 언어 추가 폼
- **title**: 새 언어 추가하기
- **text**: '새 언어 추가' 섹션의 폼에서는 새로운 언어를 시스템에 등록할 수 있습니다. 필수 입력 항목은 다음과 같습니다: 1)언어 코드(필수) - 예: vi, th, id (소문자, 하이픈만 가능, 최대 10자), 2)언어 이름 (영어) - 예: Vietnamese, 3)네이티브 이름 - 예: Tiếng Việt, 4)국기 이모지 - 예: 🇻🇳. 모든 항목을 입력 후 '+ 언어 추가' 버튼을 클릭합니다.
- **highlight**: `#add-language-form`
- **trigger**: confirm
- **tips**:
  - 언어 코드는 반드시 소문자 (예: ko, en, ja)
  - 지역별 구분이 필요하면 하이픈 사용 (예: zh-hans, pt-br)
  - 언어 이름은 영어로 작성 (일관성 유지)
  - 네이티브 이름은 해당 언어로 작성
  - 국기 이모지는 1-4자 범위 내

### Step 5: 언어 코드 입력 및 검증
- **title**: 언어 코드 입력
- **text**: '언어 코드' 입력 필드(name="code")에는 ISO 639-1 표준 코드를 입력합니다. 예: vi (베트남어), th (태국어), id (인도네시아어), zh-hans (중국어 간체). 입력 필드는 패턴 검증이 활성화되어 있어 소문자와 하이픈만 허용됩니다. 중복된 언어 코드는 시스템에서 자동으로 거부합니다.
- **highlight**: `input[name="code"]`
- **trigger**: input
- **tips**:
  - 입력 시 자동으로 소문자로 변환
  - 하이픈은 영어-숫자 조합에만 허용
  - 최대 10자 제한
  - 입력 예: ko, en, ja, vi, th, id, zh-hans, pt-br

### Step 6: 폼 제출 및 새 언어 활성화
- **title**: 새 언어 등록 완료
- **text**: 모든 필드를 입력한 후 '+ 언어 추가' 버튼을 클릭하면 새 언어가 시스템에 등록됩니다. 새로 추가된 언어는 기본적으로 비활성화('⏸️ 비활성화') 상태로 추가되며, 필요하면 토글 버튼으로 활성화할 수 있습니다. 하단의 안내 메시지 "새 언어는 비활성 상태로 추가됩니다"에 명시되어 있습니다.
- **highlight**: `button[type="submit"]`
- **trigger**: click
- **buttons**:
  - selector: `button[type="submit"]`
  - label: "+ 언어 추가"
  - desc: 새 언어를 시스템에 등록
- **tips**:
  - 등록 완료 후 페이지가 새로고침됨
  - 새 언어는 언어 목록에 자동으로 추가됨
  - 추가 후 번역 관리 페이지에서 즉시 번역 작업 가능
  - 활성화는 별도 토글로 진행 (순서 중요)

### Step 7: 언어 활성화 순서 및 주의사항
- **title**: 새 언어 활성화 및 배포
- **text**: 새 언어를 추가한 후 즉시 사용하려면 토글 버튼으로 활성화해야 합니다. 활성화 전에는 번역 콘텐츠가 사이트에 표시되지 않으므로 주의하세요. 권장 절차: 1)새 언어 추가, 2)번역 작업 수행 (번역 관리 페이지), 3)번역 검수, 4)언어 토글로 활성화, 5)배포. 이 순서를 따르면 완성되지 않은 번역이 표시되는 것을 방지할 수 있습니다.
- **highlight**: `.toggle-btn`
- **trigger**: confirm
- **tips**:
  - 활성화 전 번역 완성도 확인 권장
  - 번역이 없는 상태로 활성화하면 기본 언어(한국어)로 표시됨
  - 언어 비활성화는 데이터 손실이 없음
  - 번역 관리 페이지에서 진행도를 확인 후 활성화 결정
