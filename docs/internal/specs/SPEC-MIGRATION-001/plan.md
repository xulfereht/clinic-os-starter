## 4. 커스텀 분류 기준

### 4.1 분류 결정 트리

```
파일 분류 결정 트리:

1. clinic-os에 동일 파일이 있는가?
   ├── YES → 내용이 동일한가?
   │         ├── YES → SKIP (이식 불필요)
   │         └── NO  → 차이점 분석 → LOCAL로 분리 또는 코어에 머지
   └── NO  → brd-clinic 전용인가?
             ├── YES → LOCAL로 이식
             └── NO  → 코어에 추가 제안 (승격 후보)
```

### 4.2 경로 매핑 규칙

| 원본 경로 | 대상 경로 | 조건 |
|-----------|-----------|------|
| `src/lib/*.ts` | `src/lib/local/*.ts` | 커스텀 로직 |
| `src/pages/*.astro` | `src/plugins/local/pages/*.astro` | 테넌트 전용 페이지 |
| `src/components/*.tsx` | `src/plugins/local/components/*.tsx` | 테넌트 전용 컴포넌트 |
| `public/*` | `public/local/*` | 브랜드 리소스 |
| `migrations/*.sql` | 무시 | DB 공유로 해결 |

## 5. 배포 전략

### 5.1 배포 방식 비교

| 방식 | 장점 | 단점 | 권장 |
|------|------|------|------|
| A: 기존 프로젝트에 새 코드 배포 | 단순함 | 롤백 시 재배포 필요 | △ |
| B: 별도 프로젝트 + 도메인 전환 | 즉시 롤백 | 설정 복잡 | ◎ |

### 5.2 권장 배포 순서 (방식 B)

```
1. brd-clinic-staging (스테이징)
   └── staging.brd-clinic.com

2. brd-clinic-2 (신규 프로덕션)
   └── 임시: new.brd-clinic.com

3. 도메인 전환
   └── brd-clinic.com → brd-clinic-2

4. 안정화 후
   └── brd-clinic (기존) → archive
   └── brd-clinic-2 → brd-clinic (이름 변경)
```
