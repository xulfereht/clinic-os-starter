## Core Framework (One-Minute Read)

### The Golden Rules

1. **Segment ruthlessly** — Max 2-5 minutes per video. (45-60 sec achieves 74.8% completion)
2. **Hook with pain, not product** — Open with problem in first 5 seconds
3. **Show benefit, not feature** — "Save 2 hours/week" beats "One-click exports"
4. **Conversational tone** — Friend-like narration, not corporate
5. **Minimal text, maximum narration** — Don't duplicate; use arrows/highlights instead
6. **Follow-along + overview** — Inspire AND enable (hybrid approach)
7. **Captions mandatory** — Accessibility + 15-20% retention boost
8. **Clear CTA at end** — Tell viewers exactly what to do next

### Universal Narrative Structure (Proven Across All Companies)

```
HOOK (0-5 sec)       → Open with relatable problem
CONTEXT (5-15 sec)   → Why this matters
SOLUTION (15-25 sec) → Introduce feature as benefit
DEMO (25-80 sec)     → Product speaks visually
BENEFIT (80-90 sec)  → Time/effort saved recap
CTA (90-100 sec)     → Clear next action
```

### Course Hierarchy (Proven Effective)

```
COURSE (e.g., "Clinic Staff Onboarding")
├─ MODULE 1 (5-7 lessons, ~30 min total)
│  ├─ Lesson: Video (2-3 min) → Concept
│  ├─ Interactive (5-10 min) → Hands-on
│  └─ Quiz (2 min) → Validation
├─ MODULE 2 (same structure)
└─ MODULE 3 (same structure, end with capstone)
```

## Implementation Roadmap (For Clinic-OS)

### Phase 1: Foundation (Week 1)
- [ ] Identify 3-5 critical pain points (ask staff, review support tickets)
- [ ] Assign 1 person as "lead producer"
- [ ] Acquire equipment: USB microphone ($50-100), screen recording software
- [ ] Create video style guide (intro/outro graphics, branding)

### Phase 2: Pilot (Week 2-3)
- [ ] Produce first video (1 feature, ~2 min)
- [ ] Follow production checklist: pre-production → recording → editing → review
- [ ] Collect feedback from 5 staff
- [ ] Publish to unlisted YouTube or Vimeo

### Phase 3: Measure (Week 4)
- [ ] Track: completion rate (target 60%+), average view duration (70%+), CTA click-through (15%+)
- [ ] If successful → plan module (5-7 related videos)
- [ ] If not → diagnose (audio quality? pacing? hook strength?) → iterate

### Phase 4: Scale (Month 2)
- [ ] Produce Module 1 (5-7 videos covering staff onboarding)
- [ ] Add interactives (practice tasks, quizzes)
- [ ] Integrate into learning platform (Teachable, Kajabi, or custom Astro)
- [ ] Launch to new clinic staff

### Phase 5: Iterate & Expand (Ongoing)
- [ ] Monitor which topics drive most questions (make videos for these)
- [ ] Refresh videos quarterly (product updates)
- [ ] Expand to other roles (doctors, clinical staff, patients)

## Critical Success Factors

### Technical
- **Audio quality is 80% of perceived quality** — Invest in USB mic + quiet room
- **Screen clarity matters** — Record at 1920x1080 minimum; test on phone
- **Captions are mandatory** — Non-negotiable for accessibility + engagement

### Narrative
- **Problem comes before feature** — Every successful video opens with pain, not product
- **Conversational tone beats corporate** — "You're spending 30 minutes..." beats "The Clinic-OS scheduler allows one to..."
- **Minimal on-screen text** — Rely on narration, use annotations for emphasis only

### Organizational
- **Pick low-stakes first** — Don't start with complex workflow; start with what people get wrong most
- **Get feedback before polishing** — Show rough cut to 5 staff; let them guide edits
- **Track metrics relentlessly** — Completion rate is your signal; if <60%, shorten or strengthen hook

## FAQ

### Q: Can we use Zoom recordings instead of ScreenFlow?
**A:** Yes, but quality is lower. Zoom assumes conferencing (lower bitrate); ScreenFlow/Camtasia are designed for training videos. If budget is tight, Zoom works; if quality matters, invest in ScreenFlow.

### Q: How do we handle product changes?
**A:** Plan for this. Assign someone to review videos quarterly. If product changes significantly, either re-shoot (30 min) or note "This video was made with [version]; some screens may look different."

### Q: What if staff don't complete the videos?
**A:** This is normal. Completion rates 50-70% is industry standard. Use analytics to find where people drop off—if they stop at 1 min, your hook is weak; if at 3 min, you're too long.

### Q: Can we use AI narration?
**A:** Technically yes, but conversational human narration outperforms AI 85% of the time. For first videos, use human. If scaling to 100+ videos, AI might be justified.

### Q: Who should narrate?
**A:** Someone knowledgeable about the feature (not necessarily the creator). Clinic director/experienced staff member = more credible than faceless voice.

### Q: Do we need to show the narrator on camera?
**A:** Not necessary. Screencast-only (Slack, Loom-style) is standard. If you want personal touch, overlay small camera feed in corner (adds production time).

### Q: What if a video flops (low completion)?
**A:** Diagnose:
- <40% completion: Hook is weak or video is too long. Shorten or open with bigger pain.
- 40-60% completion: OK. Acceptable for longer tutorials.
- Audio feedback: "Hard to hear": Boost narration volume. "Boring voice": Re-record with more energy.

### Q: How do we measure ROI?
**A:** Track:
1. **Before videos:** Time to full onboarding (staff reaching productivity)
2. **After videos:** Same metric
3. **Support tickets:** Decrease in "how do I..." questions
4. Example: If videos cut onboarding time 40%, that's 2 hours saved per staff member × 10 new hires/year = 20 hours saved.

### Q: Can we use external video producers?
**A:** Yes. Budget $1,500-5,000 per video module (5-7 videos). Pros: Professional. Cons: Slower feedback loops, less control. For first module, DIY to learn; then consider agencies if scaling.
