## Notion Academy

### Pattern: Problem → Concept → Template

**Course:** "Organize Your Life"  
**Lesson:** "Building a Task Database"

**Structure:**
1. **Hook (30 sec)** — "You have 47 tasks scattered across notes, emails, and your brain. Let's put them in one place."
2. **Concept (90 sec)** — Shows creating a simple database: name, properties, views
3. **Show & Tell (60 sec)** — Demo adding tasks, filtering by status
4. **Template (30 sec)** — "Here's a template you can copy and customize"
5. **CTA (20 sec)** — "Copy this template and fill it with your tasks"

**Key takeaway:** Videos end with *giveaway* (template/base to use immediately)

**Narration style:** Encouraging, friend-like. "Here's how I'd approach it..."

**Video metadata:** 
- Duration: 3-5 min
- Includes: Written guide + downloadable PDF + template link
- Certification: Optional badge on completion

## Figma Learn

### Pattern: Concept → Hands-On Project → Deep Dive

**Course:** "Figma Design for Beginners (2025)"  
**Lesson:** "Understanding Components"

**Structure:**
1. **Why Components Matter (90 sec)** — Overview video, no interaction expected
   - "Design systems are everywhere. Here's how components make them possible."
   - Shows real example: button library updated once, syncs everywhere
2. **Interactive Project (10 min)** — Hands-on in real Figma
   - "Open Figma. Follow this walkthrough. Create 3 components."
   - Copyable Figma file with instructions
3. **Advanced Patterns (5 min)** — Optional deep-dive
   - Variants, responsive components (for intermediate users)

**Key takeaway:** Video teaches concept; students validate by doing it themselves in real tool

**Narration style:** Measured, assumes some familiarity with design

**Platform feature:** YouTube auto-dubbing (videos available in 20+ languages)

**Metadata:**
- Organized as "learning paths" (beginner → intermediate → advanced)
- Includes "bite-sized projects" (quick wins)
- Progress tracking (show which lessons completed)

## HubSpot Academy

### Pattern: Feature → Demo → Quiz → Certification

**Course:** "HubSpot Sales Certification"  
**Lesson:** "Managing Your Sales Pipeline"

**Structure:**
1. **Upfront transparency** — Course page shows:
   - 12 lessons
   - 15 videos
   - 8 quizzes
   - ~2 hours total
2. **Feature video (2 min)** — "Here's what the Sales Pipeline is"
   - Screenshot-based demo
   - Clear narration
3. **How-To video (3 min)** — "How to use it for your deals"
   - Step-by-step demo
   - Real data (actual sales scenario)
4. **Quiz (5 min)** — 5-10 quick questions
   - Validates understanding
   - Shows score immediately
5. **Badge/Certification** — Complete all → get badge

**Key takeaway:** Heavy emphasis on validation and credentials

**Narration style:** Professional, clear. Feature-by-feature breakdown.

**Structure philosophy:** "Short, focused" — each video covers ONE feature

**Metadata:**
- Every course lists video count upfront (sets expectations)
- Certification badges are shareable (LinkedIn)
- Progress is tracked (show % completion)

## Canva Design School

### Pattern: Watch → Activity → Cheat Sheet

**Course:** "Canva Essentials"  
**Lesson:** "Creating Your First Video"

**Structure:**
1. **Video Tutorial (3-5 min)** — "How to create a video in Canva"
   - Conversational narration
   - Fast pacing (show results, not every click)
   - Beginner-friendly (no jargon)
2. **Hands-On Activity (5-10 min)** — "Try it yourself"
   - Worksheet with step-by-step instructions
   - Optional: worksheets with visual callouts
3. **Cheat Sheet** — Download printable PDF
   - Quick reference for future
   - Not video-dependent (text alternative)

**Key takeaway:** Multiple formats for different learning styles

**Narration style:** Upbeat, encouraging. "Let's create a video together!"

**Metadata:**
- ~30 min per course total
- 3-10 min per video
- Includes downloadable templates for practice

## Stripe Developer Docs

### Pattern: Problem → Code Example → Video Supplement

**Documentation:** "Accepting Payments"  
**Section:** "Setting Up Your First Payment"

**Structure:**
1. **Written docs (primary)** — Code examples in Python/Node/Ruby
2. **Quickstart guide** — "Get started in 5 minutes"
3. **Video (5-7 min)** — For complex concepts only
   - "Here's what that code does in practice"
   - Screen recording of running code
   - Explains why (not just what)
4. **References** — API docs with examples

**Key takeaway:** Video is supplemental, not primary (developer audience = prefer reading)

**Narration style:** Technical, precise. Assumes audience is developers.

**Platform:** YouTube (community channel) + embedded in docs

**Metadata:**
- Videos organized by SDK/language
- Linked from relevant code sections
- Transcripts available

## Loom (Teaching Loom Itself)

### Pattern: Context-Rich Screen + Narration

**Tutorial:** "How to Record Your First Loom"

**Structure:**
1. **Setup (30 sec)** — What you need (browser, screen, optionally camera)
2. **Demo (2-3 min)** — Recording a screencast
   - Shows Loom interface
   - Shows narration being recorded
   - Optionally shows narrator in webcam corner (context)
3. **Playback (1 min)** — What you'll see after recording
   - Editing interface
   - Sharing options
4. **CTA (20 sec)** — "Try it yourself. Click 'Record' below."

**Key takeaway:** Shows actual interface users will see (context-rich demo)

**Narration style:** Natural, conversational. "I'm going to show you how to..."

**Production:** Often live-recorded (less polished but more authentic)

**Metadata:**
- Free for educators (discount program)
- Organized by topic (basics, education, business)
- Community-contributed examples

## Airtable Academy

### Pattern: Lesson → Interactive Base → Quiz

**Learning Path:** "Introduction to Airtable"  
**Module:** "Building Your First Base"

**Structure:**
1. **Video (2-3 min)** — "What is a base?"
   - Concept overview
   - Visual examples
   - Shows real Airtable interface
2. **Interactive Tutorial (5-10 min)** — "Create a base"
   - Real Airtable loaded in parallel browser
   - "Click here, add this field, see what happens"
   - Copyable starter base
3. **Optional: Video (3 min)** — "Advanced: Views & Filtering"
   - For intermediate learners
4. **Quiz (3 min)** — 3-5 questions
   - "What's the difference between a record and a field?"
   - Immediate feedback

**Key takeaway:** Video teaches; hands-on base validates; quiz confirms

**Narration style:** Professional but approachable. "Let's build a project management system..."

**Recent update:** Shorter videos (courses divided into small segments for easier digestion)

**Metadata:**
- Learning paths have prerequisites ("Start here")
- Includes downloadable PDF guides as text alternative
- Progress tracked (you're on Step 2 of 5...)

## Slack

### Pattern: Basics First, Then Features, Then Best Practices

**Tutorial Series:** "Getting Started with Slack"  
**Video 1:** "What is Slack?"

**Structure:**
1. **Basics (2 min)** — "Here's how Slack works"
   - Show channels, messages, sidebar
   - Quick demo of sending message
2. **Features (3 min each, series)** — "How to use [feature]"
   - How to use channels
   - How to use threads
   - How to use calls
3. **Best Practices (3-5 min)** — "How productive teams use Slack"
   - Organization tips
   - Using integrations

**Key takeaway:** Start simple, then specific, then aspirational

**Narration style:** Conversational, expert tone. "Here's the basics of Slack..."

**Presentation:** Host navigates UI without appearing on camera (informal but effective)

**Metadata:**
- Quick-start guide (video series overview)
- Feature-based organization
- Mixed video + text tutorials

## Universal Patterns (Across All Companies)

### 1. Problem-First Hook (Every Company Does This)

❌ **Don't start with:**
> "Figma is a design collaboration tool for teams..."

✅ **Do start with:**
> "You're sending design files back and forth via email. Designers and stakeholders are working on different versions. Here's how to fix that."

### 2. Benefit Over Feature (Universal)

❌ **Don't say:**
> "Airtable has a filtering feature."

✅ **Do say:**
> "In 3 seconds, show only the tasks due today. No manual sorting."

### 3. Segmentation (All Companies)

❌ **Don't do:**
> One 15-minute video covering 5 features

✅ **Do this:**
> 5 separate 2-3 minute videos, one per feature. Users watch only what they need.

### 4. Conversational Tone (Strong Preference)

❌ **Don't:**
> "The user is able to configure the system to their specification."

✅ **Do:**
> "You can set this up exactly how you want."

### 5. Show → Don't Tell (Universal)

❌ **Bad:**
> (Narration only) "The button is in the top right. It opens a dialog. The dialog has three options..."

✅ **Good:**
> (Show cursor moving, button clicked, dialog appearing, with minimal narration) "Click here. See the three options. Choose the one you need."

### 6. Interactive After Passive (Pattern)

**Notion:** Video → Template to copy  
**Figma:** Video → Project to complete  
**Airtable:** Video → Base to interact with  
**HubSpot:** Video → Quiz  

All follow: Passive (watch) → Active (do something) → Validate (prove understanding)

### 7. Multiple Formats for Accessibility (Universal)

- **Video** (primary for engagement)
- **Transcript** (for deaf viewers, searchability)
- **Written guide** (for people who prefer reading, reference)
- **Interactive** (hands-on validation)

### 8. Progress Tracking (Increasingly Standard)

- "You're on Step 2 of 5..."
- Badge system (completion visible)
- Quiz score (validation)
- Progress bar

## Production Quality Observations

### What Top Companies DO:

✅ **Clean desktops** — No distracting files, icons, or bookmarks  
✅ **High-quality audio** — Clear narration, no background noise  
✅ **Cursor highlighting** — So viewers follow actions  
✅ **Fast pacing** — Don't linger on loading screens  
✅ **Real data examples** — Not "John Doe", use realistic names  
✅ **Consistent branding** — Intro/outro graphics match  
✅ **Accurate captions** — Not auto-generated alone  

### What They AVOID:

❌ **Showing entire screen** — Zoom in to relevant UI  
❌ **Slow pacing** — Tedious for viewers  
❌ **Heavy text overlays** — Crowds screen, duplicates narration  
❌ **Formal tone** — Feels distant  
❌ **Multiple topics per video** — Causes confusion  
❌ **Auto-generated captions without review** — Accuracy matters  
❌ **No CTA** — Viewers don't know next step
