# Video Production Checklist

**For Clinic-OS Track B-1: Tutorial & Onboarding Videos**

---

## Phase 1: Pre-Production Planning

### Concept Definition
- [ ] **Identify the pain point** — What problem does this solve?
  - Example: "New staff spending 30 min on manual patient intake"
- [ ] **Define the feature/workflow** — What's the solution?
  - Example: "Smart intake form with SMS + auto-parsing"
- [ ] **Identify the viewer** — Who needs this video?
  - Example: "Clinic admin scheduling their first patient"
- [ ] **Determine video type** — Overview, follow-along, or hybrid?
  - [ ] Overview: Show what's possible (1-2 min, fast, inspiring)
  - [ ] Follow-along: Step-by-step mirror-on-screen (2-5 min, slower)
  - [ ] Hybrid: Overview + follow-along combo (3-5 min total)
- [ ] **Set viewer action** — What will they do after watching?
  - Example: "Create their first patient intake form"
- [ ] **Estimate duration** — Based on steps/complexity
  - Single task: 2 min max
  - Multi-step: 3-5 min
  - Complex workflow: split into 2-3 videos

### Script Development
- [ ] **Write hook (0-5 sec)** — Open with the problem
  - [ ] Relatable to target viewer
  - [ ] Sets up emotional tension (time waste, frustration)
  - [ ] No product name yet—just the pain
  
  *Example:*
  > "You're filling out the same intake form by hand, 10 patients a day. That's 2 hours of typing."

- [ ] **Introduce solution (5-10 sec)** — Name the feature simply
  - [ ] Clear, jargon-free name
  - [ ] Frame as benefit, not feature
  
  *Example:*
  > "Smart intake forms cut that in half—patients fill it on their phone in 3 minutes."

- [ ] **Show the change (10-20 sec)** — What's different?
  - [ ] Before/after or side-by-side comparison
  - [ ] One transformation highlighted clearly
  
  *Example:*
  > "Instead of clipboard + typing, they scan QR or click a link. Data flows straight in."

- [ ] **Plan demo flow (20-180 sec)** — List exact steps
  - [ ] 3-5 major steps only (no minor clicks)
  - [ ] Fast-forward boring parts (loading screens, typing)
  - [ ] Highlight outcome at each step
  
  *Example steps:*
  1. Create new intake form
  2. Customize 2-3 fields
  3. Send link via SMS
  4. Show patient filling it out (sped up)
  5. Show data in patient record

- [ ] **Benefit recap (final 10 sec)** — What changed?
  - [ ] Time/effort saved quantified
  - [ ] Key outcome of feature
  
  *Example:*
  > "3-minute form fills, auto-organized notes, zero data entry errors."

- [ ] **Call to action (last 5 sec)** — What's next?
  - [ ] Specific action (not vague "Learn more")
  - [ ] Match viewer stage (onboarding, feature exploration, etc.)
  
  *Examples:*
  > "Ready to try? Click 'Create Intake Form' below."
  > "Next step: customize it for your clinic."
  > "Start your free trial — includes intake setup."

- [ ] **Tone & narration notes**
  - [ ] Conversational language ("you're", "let's", contractions)
  - [ ] No jargon without explanation
  - [ ] Natural pace (aim for ~2.5 min of narration for 3-min video)
  - [ ] Avoid reading on-screen text

### Demo Setup
- [ ] **Prepare demo account/data**
  - [ ] Patient names + realistic data (not "John Doe")
  - [ ] Real clinic details (your address, logo, staff names)
  - [ ] Test data in place (so demo runs smoothly)
- [ ] **Document demo flow** — Write out exact clicks in order
  - Step 1: Click [UI element]. Result: [what appears]
  - Step 2: Fill [field]. Result: [what changes]
  - *Document this so you don't improvise while recording*

## Phase 2: Environment Preparation

### Physical Setup
- [ ] **Desktop cleanup**
  - [ ] Close all browser tabs except ones you need
  - [ ] Hide browser bookmarks bar (visual clutter)
  - [ ] Hide desktop files (move to folder temporarily)
  - [ ] Disable notifications (disable Slack, Mail, Calendar alerts)
  - [ ] Full-screen demo app only (no taskbar/menu bar visible if possible)
- [ ] **Audio setup**
  - [ ] Microphone positioned 6 inches from mouth
  - [ ] Test audio: Record 10 sec, play back (clear? loud enough? noise?)
  - [ ] If using USB mic, ensure input level is consistent
  - [ ] Quiet room (listen for: HVAC, traffic, keyboard clicks)
  - [ ] Optional: light audio compression on narration track
- [ ] **Display settings**
  - [ ] Zoom in UI to 125-150% (make buttons/text readable at small screen sizes)
  - [ ] Use system light theme OR dark theme consistently
  - [ ] Test recording at actual resolution (1080p min, 4K preferred)

### Recording Software Setup
- [ ] **ScreenFlow / Camtasia / QuickTime open**
  - [ ] Set resolution: 1920x1080 or higher
  - [ ] Set frame rate: 30fps (standard)
  - [ ] Audio input: USB microphone
  - [ ] Optional: cursor zoom enabled (makes cursor movement obvious)
  - [ ] Optional: cursor highlighting enabled

---

## Phase 3: Recording

### Take 1-2: Initial Recordings
- [ ] **Do not obsess over perfection**
  - [ ] Typos can be cut
  - [ ] Slightly awkward narration can be re-dubbed
  - [ ] Focus: Get the demo flow correct
- [ ] **Record without stopping** — Capture full demo flow (even with mistakes)
- [ ] **Note timestamps of good segments** — "0:45-2:15 looks good, use that"

### Takes 3+: Polish Takes
- [ ] **Narration separately?** (Optional, cleaner audio)
  - [ ] Record demo video silent (no narration)
  - [ ] Record narration as audio-only track
  - [ ] Sync in edit (gives you flexibility to tweak timing)
- [ ] **Or record together** (faster, more natural pacing)
  - [ ] Talk naturally; don't try to match a script word-for-word
  - [ ] Pause 2-3 seconds before big transitions (give brain time to process)

### Quality Checkpoints
- [ ] **Audio check: Play back raw audio**
  - [ ] Audible? (at least 70db)
  - [ ] No background noise/hum?
  - [ ] Narration clear and engaging?
- [ ] **Video check: Play back raw video**
  - [ ] Focus sharp on important UI?
  - [ ] Cursor visible and trackable?
  - [ ] Fast-forwards are actually fast (or need more speed)?
  - [ ] Demo flow matches your plan?

---

## Phase 4: Editing

### Assembly
- [ ] **Import and sync** audio/video if recorded separately
- [ ] **Trim unused sections**
  - [ ] Remove dead air (start, end, between takes)
  - [ ] Cut mistakes / long pauses
  - [ ] Tighten spacing between steps
- [ ] **Sequence major sections**
  1. Hook (0-5 sec)
  2. Solution intro (5-10 sec)
  3. Demo (10-180 sec)
  4. Benefit recap (170-175 sec)
  5. CTA (175-180 sec)

### Visual Polish
- [ ] **Cursor management**
  - [ ] Enable cursor zoom/highlight (show where action is)
  - [ ] Ensure cursor doesn't cover important text
  - [ ] Consider small cursor highlight circle or color
- [ ] **On-screen annotations** (optional, depends on complexity)
  - [ ] Arrows pointing to next action
  - [ ] Color circles around important elements (50% opacity)
  - [ ] Keyboard key callouts ("Press ENTER")
  - [ ] Counter: Show elapsed time for context
  - **Rule:** Minimal text. Use narration, not annotations.
- [ ] **Transitions**
  - [ ] Cuts between steps (fast, no dissolves needed)
  - [ ] Fade to black for major section changes (optional)
  - [ ] No unnecessary slide/spin transitions (keeps focus on content)
- [ ] **Color grading** (optional)
  - [ ] Adjust levels if video is too bright/dark
  - [ ] Maintain consistent color temperature
  - [ ] Don't go overboard (should look natural)

### Audio Cleanup
- [ ] **Normalize audio levels** — Consistent throughout
- [ ] **Remove background noise** (if recorded in non-ideal room)
- [ ] **Reduce plosives** (soft 'p' sounds with pop filter or software)
- [ ] **Add light compression** (optional, pro feature)

### Intro/Outro Graphics (Optional but Recommended)
- [ ] **Intro** (3-5 sec)
  - [ ] Your clinic logo
  - [ ] Video title
  - [ ] Brief hook text
  - [ ] Music or silence (depends on tone)
- [ ] **Outro** (3-5 sec)
  - [ ] "Next Steps:" text
  - [ ] Logo
  - [ ] CTA button (if embedded)

### Captions/Subtitles (MANDATORY)
- [ ] **Generate captions**
  - [ ] Use Adobe Premiere, YouTube auto-captions (with review), or Rev.com
  - [ ] Review for accuracy (auto-captions are 70-80% accurate)
  - [ ] Fix technical terms, product names, names
- [ ] **Format captions**
  - [ ] Font: Sans-serif (Arial, Helvetica)
  - [ ] Size: Large enough to read on phone
  - [ ] Color: White with black outline (high contrast)
  - [ ] Placement: Not bottom 20% (where captions will sit if toggled on)
  - [ ] Burn in OR keep as separate SRT file
- [ ] **Test readability** — Watch on phone at arm's length

## Phase 5: Review & Feedback (Before Publishing)

### Internal Review (You)
- [ ] **Watch full video with sound off**
  - [ ] Can you follow the story from visuals alone?
  - [ ] Do captions match what's on screen?
  - [ ] Are step transitions clear?
- [ ] **Watch with sound, no captions**
  - [ ] Is narration clear and engaging?
  - [ ] Does pacing feel right (not too fast, not dragging)?
  - [ ] CTA clear and compelling?
- [ ] **Watch on phone** — Text and cursor highlight readable?

### Staff Feedback (5 people minimum)
- [ ] **Give them context:** "You're a clinic admin new to Clinic-OS. Watch this 3-minute video and tell me:"
  - [ ] Did you understand what the feature does?
  - [ ] Could you follow the steps if you tried it yourself?
  - [ ] Is there anything confusing?
  - [ ] What would you do next?
- [ ] **Collect specific feedback**
  - If "I didn't understand X" → Edit to clarify
  - If "Pacing felt slow" → Tighten cuts, speed up non-critical parts
  - If "I didn't know what to do next" → Strengthen CTA
  - If "Audio was hard to hear" → Boost narration level
- [ ] **Iteration:** If 3+ people say same thing, fix it

## Phase 6: Publishing & Integration

### File Export
- [ ] **Video format**
  - [ ] Format: MP4 (H.264, AAC audio)
  - [ ] Resolution: 1920x1080 or higher
  - [ ] Frame rate: 30fps
  - [ ] Bitrate: 5-8 Mbps (quality/file size balance)
- [ ] **Caption export**
  - [ ] Format: SRT (SubRip) or VTT (WebVTT)
  - [ ] Encoding: UTF-8
- [ ] **Transcript export**
  - [ ] Format: TXT or PDF (readable)
  - [ ] Include speaker names and section timestamps

### Upload & Hosting
- [ ] **YouTube (unlisted or private)**
  - [ ] Upload video + SRT subtitle file
  - [ ] Add title (follow naming convention)
  - [ ] Add description with transcript/link
  - [ ] Set visibility: Unlisted (searchable but not recommended)
  - [ ] Enable captions
- [ ] **Vimeo (if privacy is a concern)**
  - [ ] Same process as YouTube
  - [ ] Can restrict to domain/password if needed
- [ ] **Self-host (advanced)**
  - [ ] Upload MP4 to your server
  - [ ] Host via video.js or similar player
  - [ ] Embed SRT captions in player

### Learning Platform Integration
- [ ] **Add to course hierarchy**
  - [ ] Module: [Name]
  - [ ] Lesson: [Name]
  - [ ] Duration: [2 min]
  - [ ] Skill level: Beginner / Intermediate / Advanced
- [ ] **Add metadata**
  - [ ] Title (searchable, clear)
  - [ ] Description (3-4 sentences, what viewer learns)
  - [ ] Learning objectives (bullet points, "After watching, you'll be able to...")
- [ ] **Add transcript link**
  - [ ] Downloadable PDF or viewable text
  - [ ] Searchable (if platform supports it)
- [ ] **Link to next step**
  - [ ] "Next: Try it yourself [interactive]"
  - [ ] "Next video: [title]"
  - [ ] "Practice: [assignment or quiz]"

### Documentation
- [ ] **Add to docs/guides**
  - [ ] Link in relevant page (e.g., "Getting Started" → embedded video)
  - [ ] Note in table of contents
  - [ ] Add context paragraph (when to watch, why it matters)
- [ ] **Create reference in learning center**
  - [ ] By topic (feature-based)
  - [ ] By skill level
  - [ ] By role (admin, clinical staff, doctor)

## Phase 7: Launch & Tracking

### Analytics Setup
- [ ] **YouTube Analytics**
  - [ ] Views
  - [ ] Engagement (likes, comments)
  - [ ] Completion rate (see where viewers drop off)
  - [ ] Average view duration
- [ ] **Custom tracking (if using your player)**
  - [ ] Video started: yes/no
  - [ ] Completion rate: % watched
  - [ ] Interactive clicked: yes/no
  - [ ] CTA clicked: yes/no
- [ ] **Learning platform analytics**
  - [ ] Viewer watched: yes/no
  - [ ] Time spent: [minutes]
  - [ ] Quiz taken: yes/no
  - [ ] Quiz score: [%]

### Measurement Targets
- [ ] **Completion rate:** Aim for 60%+
  - If <60%: Video is too long or weak hook
- [ ] **Average view duration:** Aim for 70%+
  - If <70%: Pacing issue or audio quality
- [ ] **Interactive engagement:** Aim for 50%+
  - If <50%: Interactive is too hard or unclear
- [ ] **CTA click-through:** Aim for 15%+
  - If <15%: CTA is unclear or not compelling

### Iteration
- [ ] **Week 1:** Collect initial feedback (50+ viewers)
- [ ] **Week 2:** Fix critical issues (audio, unclear sections)
- [ ] **Week 4:** Review analytics
  - If completion >60% and >50% do CTA: Success! Move to next video
  - If not: Identify issue and re-edit
- [ ] **Monthly:** Review which videos are rewatched
  - High rewatch = pain point, important to get right
  - Low rewatch = less critical, OK to be brief

## Phase 8: Series Planning

### After First Video Succeeds
- [ ] **Identify next pain point** (ask staff, support tickets)
- [ ] **Plan series structure**
  - [ ] Module goal (e.g., "Getting Started")
  - [ ] Number of lessons (5-7 per module)
  - [ ] Total duration (30 min per module)
  - [ ] Sequencing (prerequisites first)
- [ ] **Add interactives between videos**
  - [ ] "Try it yourself" 5-10 min hands-on
  - [ ] Quiz to validate understanding
  - [ ] Practice project
- [ ] **Create companion guides**
  - [ ] Downloadable PDF (video + text)
  - [ ] Checklist for setup
  - [ ] FAQ for common issues

### Long-Term Maintenance
- [ ] **Review after major updates**
  - [ ] Product changed? Video might be outdated
  - [ ] Update or create v2
- [ ] **Monitor support requests**
  - [ ] Still getting questions about X? Make a video
  - [ ] Fewer questions about Y? Series is working
- [ ] **Refresh every 6-12 months**
  - [ ] Update screenshots/UI
  - [ ] Add new features to existing tutorials
  - [ ] Retire videos for deprecated features

## Quick Reference: Timeline

| Phase | Time | Key Deliverables |
|-------|------|---|
| **Pre-Production** | 2-3 hours | Script, demo plan, storyboard |
| **Recording** | 1-2 hours | Raw video + audio (3-5 takes) |
| **Editing** | 2-3 hours | Polished video, captions, transcript |
| **Review & Feedback** | 1-2 hours | Feedback from 5 staff, notes for edits |
| **Revisions** | 30 min - 1 hour | Apply feedback, re-export |
| **Publishing** | 30 min | Upload, integrate, test links |
| **Launch** | Ongoing | Monitor analytics, iterate |

**Total time per video: 7-12 hours (first video slower; subsequent videos faster once you have templates/process)**

## Common Issues & Fixes

| Issue | Fix |
|-------|-----|
| **Audio too quiet** | Record again with mic closer + boost in edit |
| **Cursor hard to see** | Enable cursor zoom/highlight; use light background |
| **Narration sounds formal** | Re-record with conversational language; think "friend explaining" |
| **Pacing feels slow** | Tighten cuts between steps; remove pauses; speed up non-critical parts |
| **Viewers don't know what to do next** | Strengthen CTA; be specific ("Click 'Create Intake Form'" not "Learn more") |
| **Low completion rate** | Shorten or split into 2 videos; strengthen hook (open with bigger pain) |
| **Captions misread product names** | Manually correct before publishing; save as template for future |
| **Text on-screen too small** | Re-record with zoom at 150%; test on phone |
| **Demo goes wrong mid-recording** | Edit out the mistake or re-take just that segment |

## Naming Conventions

**Video file:** `clinic-os-tutorial-[feature]-[version].mp4`
- Example: `clinic-os-tutorial-patient-intake-v1.mp4`

**Course structure:** `[Role]-[Context]-[Number].md`
- Example: `admin-onboarding-1-dashboard.md` → "Dashboard Basics for Admins"

**Learning platform title:** `[Action] Your [Thing]`
- Example: "Create Your First Patient Intake" (start with verb, specific)

**Transcript file:** `clinic-os-tutorial-[feature]-transcript-[version].txt`
- Example: `clinic-os-tutorial-patient-intake-transcript-v1.txt`
