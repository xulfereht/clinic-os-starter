# 병원 기본 정보 설정 가이드

본 가이드는 클리닉 OS의 핵심 정보인 병원명, 연락처, 진료 시간 및 기본 브랜딩 설정을 관리하는 방법을 설명합니다. 이 정보들은 홈페이지 하단(Footer), 예약 메시지, 지도 위젯 등에 공통으로 사용됩니다.

---

## 1. 설정 메뉴 접속 방법

1.  관리자 페이지 상단 또는 모든 기능(Hub)에서 **⚙️ 시스템 설정** 그룹을 찾습니다.
2.  **🛠️ 기본 설정** (또는 병원 설정) 메뉴를 클릭합니다.

---

## 2. 주요 설정 항목

### 📋 기본 정보
병원 운영에 필요한 공적 정보를 입력합니다.
- **병원명**: 홈페이지 및 모든 문서에 표시될 공식 명칭입니다.
- **대표자명**: 사업자등록증상의 대표자 성함을 입력합니다.
- **사업자등록번호**: 정확한 번호를 입력하면 필요 시 시스템에서 연동하여 사용합니다.
- **한줄 소개**: 검색 결과나 메인 하단에 표시될 병원의 핵심 슬로건을 입력합니다.

### 🎨 브랜딩 (로고 및 파비콘)
병원의 아이덴티티를 설정합니다.
- **로고 이미지**: 배경이 투명한 PNG 파일을 권장하며, 비워둘 경우 시스템에서 병원명을 텍스트로 자동 표시합니다.
- **파비콘**: 브라우저 탭에 표시되는 작은 아이콘입니다. (ICO 또는 PNG 파일)

### 📞 연락처 및 위치
고객 응대 및 지도 표시를 위한 정보입니다.
- **전화번호**: 하이픈(-)을 포함하여 입력합니다.
- **주소**: 지번 또는 도로명 주소를 입력합니다.
- **네이버 지도 URL**: 네이버 지도에서 '공유하기' 버튼을 눌러 얻은 URL을 넣으면 오시는 길 섹션과 연동됩니다.
- **지도 검색 키워드**: 홈페이지 내 지도 위젯에 표시될 정확한 상호명/지점명을 입력합니다.

### 🗓️ 진료 시간
홈페이지와 채팅 위젯 등에 노출될 업무 시간입니다.
- **평일, 토요일, 점심시간, 정기휴무**: 자유로운 텍스트 형식으로 입력할 수 있습니다. (예: 09:30 ~ 18:30)

---

## 3. 기능 토글 (On/Off)

필요에 따라 시스템의 주요 기능을 켜거나 끌 수 있습니다.
- **비대면 진료 기능**: 활성화 시 홈페이지와 관리자 모드에 비대면 전용 메뉴가 나타납니다.
- **다국어 기능**: 외국인 환자를 위한 언어 선택 및 자동 번역 기능을 활성화합니다.
- **샘플 데이터 모드**: 실제 운영 전에 시스템 UI를 확인하기 위해 가상의 데이터를 채워 넣는 기능입니다. 실제 운영 시에는 반드시 **OFF**로 설정해야 합니다.

---

## 4. 관리자 계정 보안

시드 데이터에 포함된 기본 관리자 비밀번호(`admin123`)를 반드시 변경해야 합니다.

1.  **[비밀번호 변경]** 페이지(`/admin/change-password`)에 접속합니다.
2.  **새 비밀번호**: 최소 8자, 영문+숫자+특수문자 조합을 권장합니다.
3.  **관리자 이메일**: 비밀번호 분실 시 복구용 이메일을 등록합니다.

> [!WARNING]
> 기본 비밀번호를 변경하지 않으면 누구나 관리자 페이지에 접근할 수 있습니다.

---

## 5. 약관 및 정책 문서

법적으로 필요한 문서(개인정보처리방침, 이용약관, 환불 규정)를 관리합니다.

1.  **[약관 관리]** 메뉴(`/admin/settings/terms`)에 접속합니다.
2.  시드 템플릿에 포함된 `{{병원명}}`, `{{대표자명}}`, `{{연락처}}` 등의 플레이스홀더를 실제 병원 정보로 교체합니다.
3.  약관 저장 시 자동으로 버전이 생성되며, 환자의 동의 기록이 버전별로 연결됩니다.

### 필수 문서
- **개인정보처리방침**: 법적 의무사항. 반드시 병원 실정에 맞게 수정.
- **이용약관**: 서비스 이용 조건 명시.
- **환불 규정**: 선택 사항이나 비대면 진료 시 권장.

---

## 6. 환경변수 설정 (배포 전)

Cloudflare 배포 시 사용되는 환경변수를 설정합니다.

- **ADMIN_PASSWORD**: `wrangler.toml`의 `[vars]` 섹션에서 설정. DB 비밀번호와 별개로 Cloudflare 배포 환경의 관리자 인증에 사용됩니다.
- 기본값에서 반드시 보안 비밀번호로 변경해야 합니다.

> [!NOTE]
> 이 설정은 관리자 페이지가 아닌 `wrangler.toml` 파일에서 직접 수정합니다.

---

## 7. 샘플 데이터 정리

실제 운영 전에 시드로 생성된 더미 데이터를 정리합니다.

1.  **기능 토글**: `설정 > 기능 토글`에서 '샘플 데이터 모드'를 **OFF**로 변경합니다.
2.  **데이터 삭제**: `/admin/posts`에서 샘플 게시글을 삭제하거나, 샘플 데이터 일괄 삭제 API를 사용합니다.
3.  대상: 샘플 블로그 글, 리뷰, 테스트 환자 데이터 등.

> [!WARNING]
> 삭제된 데이터는 복구할 수 없으므로 실제 운영 데이터가 아닌지 반드시 확인하세요.

---

## 💡 팁
- 모든 설정 변경 후에는 하단의 **[저장]** 버튼을 반드시 눌러야 반영됩니다.
- 다국어 기능을 사용할 경우, 하단의 '다국어 설정' 섹션에서 영어, 일본어, 중국어 명칭과 주소도 함께 입력해 주세요.
