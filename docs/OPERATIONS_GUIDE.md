# Clinic-OS 운용 핵심 개념

> 비기술자를 위한 워크샵 자료
> **모든 기술적 작업은 Gemini CLI에게 요청합니다**

---

## 🎯 핵심 원칙

```
┌─────────────────────────────────────────────────┐
│  당신은 "무엇을 원하는지"만 말합니다           │
│  Gemini CLI가 "어떻게 하는지"를 처리합니다     │
└─────────────────────────────────────────────────┘
```

---

## 🌍 개념 1: 두 개의 세계

### 로컬 (내 컴퓨터)
- 나만 볼 수 있는 **연습장** 같은 곳
- 잘못해도 아무 영향 없음
- 새 기능 테스트하는 곳

### 프로덕션 (인터넷)
- 실제 환자들이 보는 **공식 웹사이트**
- 변경하면 바로 모두에게 적용
- 항상 **로컬에서 먼저 테스트** 후 배포

### 💡 Gemini CLI에게 이렇게 요청:
| 원하는 것 | 요청 예시 |
|-----------|-----------|
| 로컬에서 테스트 | "로컬에서 사이트 실행해줘" |
| 프로덕션에 배포 | "지금 변경사항 배포해줘" |

---

## 🗄️ 개념 2: 두 개의 데이터베이스

### 로컬 DB
- 내 컴퓨터에만 있는 테스트용 데이터
- 마음대로 수정/삭제 가능
- 프로덕션에 영향 없음

### 프로덕션 DB
- 실제 환자 데이터가 저장된 곳
- 신중하게 다뤄야 함
- 삭제하면 복구 어려움

### 💡 Gemini CLI에게 이렇게 요청:
| 원하는 것 | 요청 예시 |
|-----------|-----------|
| 로컬 DB 초기화 | "로컬 데이터베이스 초기화해줘" |
| 프로덕션 DB 수정 | "환자 테이블에 필드 추가해줘 (프로덕션)" |

---

## 📦 개념 3: 업데이트 종류

### 코어 업데이트
- 기본 시스템이 새 버전으로 변경
- `npm run core:pull` 명령으로 자동 다운로드
- 내 커스터마이징(`local/` 폴더)은 **보호됨**

### 💡 Gemini CLI에게 이렇게 요청:
| 상황 | 요청 예시 |
|------|-----------|
| 최신 버전 확인 | "새 업데이트 있는지 확인해줘" |
| 업데이트 적용 | "core:pull 실행해줘" |

---

## ⏰ 개념 4: Git = 타임머신

### Git이란?
- 코드의 **변경 이력을 기록**하는 시스템
- 언제든 **과거로 돌아갈 수 있음**
- 안전하게 변경을 관리

### 왜 중요한가?
```
오늘 수정 → 문제 발생 → 어제로 되돌리기 ✅
```

### 💡 Gemini CLI에게 이렇게 요청:
| 원하는 것 | 요청 예시 |
|-----------|-----------|
| 현재 상태 저장 | "지금 상태로 저장해줘" |
| 이전으로 되돌리기 | "어제 상태로 되돌려줘" |
| 변경사항 확인 | "뭐가 바뀌었는지 보여줘" |

---

## 🔄 개념 5: 시스템 업그레이드 흐름

```
[현재 내 시스템 v1.0]
        │
        ▼
[새 버전 v1.1 다운로드]  ←  "npm run core:pull"
        │
        ▼
[자동 병합]
   - 새 기능 추가
   - 내 커스터마이징(local/) 유지
   - 충돌 시 안내
        │
        ▼
[테스트 후 배포]
```

---

## ⚠️ 반드시 기억할 것

### ✅ 해도 되는 것
- 로컬에서 마음껏 테스트
- Gemini CLI에게 모르는 것 물어보기
- 업데이트 적용 전 내용 확인 요청

### ❌ 하면 안 되는 것
- 테스트 없이 바로 프로덕션 배포
- 프로덕션 DB에서 삭제 명령
- Git 저장 없이 큰 변경

---

## 🆘 문제 상황별 요청

| 상황 | Gemini CLI에게 요청 |
|------|---------------------|
| 처음 설치 | "npm run setup 실행해줘" |
| 사이트가 안 열려요 | "로컬 서버 상태 확인해줘" |
| 오류 메시지 나왔어요 | "[오류 메시지] 이런 오류 났어" |
| 수정한 게 안 보여요 | "캐시 문제인지 확인해줘" |
| 이전 상태로 돌아가고 싶어요 | "git으로 이전 상태 복구해줘" |
| 업데이트 적용하고 싶어요 | "npm run core:pull 실행해줘" |

## 📋 워크샵 체크리스트

### 이해했는지 확인:
- [ ] 로컬과 프로덕션의 차이를 설명할 수 있다
- [ ] 왜 로컬에서 먼저 테스트해야 하는지 안다
- [ ] 두 개의 DB가 따로 있다는 것을 안다
- [ ] Git이 "되돌리기" 기능이라는 것을 안다
- [ ] 모든 기술적 요청은 Gemini CLI에게 한다

---

## 🚀 Gemini CLI 시작하기

터미널에서 프로젝트 폴더로 이동 후:

```bash
gemini
```

Gemini가 프로젝트를 이해하고 도움을 줄 준비가 됩니다.

> 자세한 설치 방법: [Gemini CLI 설치 가이드](./GEMINI_CLI_SETUP.md)
