# /collect-references — Reference Site Collection & Analysis

> **Role**: Reference Analyst
> **Cognitive mode**: Fetch and analyze external websites to enrich the content pipeline with competitive context, existing site data, and design inspiration.

Optional skill that enriches /analyze-content, /discover-edge, and /plan-content with external reference data. Can be called standalone or invoked as part of the content bootstrap pipeline.

## When to Use

- Client has an existing website being migrated
- Competitor clinics should be analyzed for positioning
- Design inspiration from other healthcare sites
- Data from blog/place extraction is thin and needs supplementation

## Procedure

### Step 0 — Gather URLs

Ask the user (or read from intake data):

```
레퍼런스 사이트가 있으면 알려주세요. 없으면 넘어갑니다.

1. 기존 홈페이지 — 이전 중인 사이트 URL (있다면)
2. 경쟁 한의원 — 벤치마크할 한의원 사이트 (1-3개)
3. 디자인 참고 — 디자인이 마음에 드는 사이트 (의료/비의료 무관)
```

If no URLs provided, skip this skill entirely.

### Step 1 — Fetch and analyze each URL

For each URL, use WebFetch with targeted extraction prompts.

**For existing site:**

WebFetch prompt: "Extract the following from this Korean medicine clinic website: 1) All program/service names 2) Doctor credentials and education 3) Key marketing copy (hero text, taglines) 4) Site structure (main sections) 5) Color palette and design style 6) Trust signals (certifications, publications, review counts). Return in structured format."

**For competitor:**

WebFetch prompt: "Analyze this Korean medicine clinic website as a competitor: 1) Clinic name and positioning 2) Main programs/services 3) Key strengths emphasized 4) Design style and tone 5) Trust signals 6) Unique features. Return in structured format."

**For design inspiration:**

WebFetch prompt: "Analyze this website's design: 1) Hero section style (fullscreen/split/minimal) 2) Color palette 3) Typography feel 4) Section layout pattern 5) Image usage style 6) Overall mood. Return in structured format."

Handle WebFetch failures gracefully:
- If fetch returns thin content: "이 사이트는 자동 분석이 어렵습니다 (JS 렌더링). 주요 특징을 직접 알려주세요."
- If fetch fails entirely: log warning, skip that URL, continue with others

### Step 2 — Synthesize into references.yaml

Save to `.agent/references.yaml`:

```yaml
# .agent/references.yaml
# Generated by /collect-references
generated_at: "{ISO date}"

existing_site:
  url: ""
  structure: []          # [hero, about, programs, map, contact, ...]
  tone: ""               # "formal" | "warm" | "clinical" | "friendly"
  programs: []           # extracted program/service names
  credentials: []        # doctor credentials found
  key_copy: []           # notable marketing phrases
  design:
    palette: {}          # {primary, accent, background}
    layout: ""           # "classic-medical" | "editorial" | "minimal"
  migration_notes: ""    # what to preserve vs change

competitors:
  - url: ""
    name: ""
    strengths: []        # what they emphasize
    programs: []         # their service offerings
    positioning: ""      # "premium" | "neighborhood" | "specialist" | "academic"
    trust_signals: []    # certifications, papers, review count
    design_notes: ""     # notable design choices
    gap_vs_us: ""        # what they have that we might lack

design_inspirations:
  - url: ""
    hero_style: ""       # "fullscreen-image" | "split-text-image" | "video-bg" | "minimal"
    section_pattern: ""  # "alternating" | "card-grid" | "scroll-story"
    mood: ""             # "minimal-warm" | "clinical-trust" | "editorial-premium"
    notable_elements: [] # specific design ideas worth considering
```

### Step 3 — Report

```
📋 레퍼런스 분석 완료

기존 사이트: {url or "없음"}
  프로그램: {programs}
  톤: {tone}

경쟁사: {N}개 분석
  {name1}: {positioning} — {strengths}
  {name2}: {positioning} — {strengths}

디자인 참고: {N}개
  {mood} 계열 — {notable elements}

이 레퍼런스를 /analyze-content, /discover-edge에서 활용합니다.
```
