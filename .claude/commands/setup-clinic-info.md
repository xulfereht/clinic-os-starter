# /setup-clinic-info — Clinic Basic Information Setup

> **Role**: Clinic Setup Manager
> **Cognitive mode**: Extract-first, ask-second. Use every available data source to pre-fill, then confirm with the user.

Sets up clinic name, contact, address, business hours, and other basic information.
This is a core step in Onboarding Tier 1, callable from `/onboarding` or standalone.

## Data Sources (priority order)

1. `.agent/clinic-profile.json` — Place/blog profile generated by `/extract-content` (highest priority)
2. `GET /api/clinic-info.ts` — Current DB settings
3. `GET /api/settings.ts` — Full site settings
4. User direct input — Only for items missing from above sources

## Procedure

### Step 1 — Collect data sources (automatic)

```bash
# Priority 1: clinic-profile.json (extract-content result)
cat .agent/clinic-profile.json 2>/dev/null

# Priority 2: Current DB settings
npx wrangler d1 execute DB --local --command \
  "SELECT category, key, value FROM site_settings WHERE category IN ('general', 'contact', 'info') ORDER BY category, key;"
```

### Step 2 — Auto-mapping

If clinic-profile.json exists, auto-map the following fields:

| clinic-profile Field | → DB Location | Notes |
|---------------------|-----------|------|
| `clinic.name_ko` | `general.site_name` | 한의원 이름 |
| `clinic.representative` | `general.representative` | 대표원장 |
| `clinic.phone` | `contact.phone` | 전화번호 |
| `clinic.address` | `contact.address` | 주소 |
| `clinic.business_hours` | `info.business_hours` | 영업시간 (JSON) |
| `clinic.categories` | (reference only) | 진료과목 힌트 |
| `clinic.naver_place_id` | `contact.naver_place` | 플레이스 링크 |
| `blog.author_name` | `general.representative` | 블로그 작성자 (fallback) |

**Mapping principles:**
- If value exists in clinic-profile → auto-fill
- If DB already has a different value → show conflict, ask user to choose
- If neither has value → ask user

### Step 3 — User confirmation (all at once)

Show mapping results in a single screen and get confirmation:

```
📋 병원 기본 정보 (플레이스에서 자동 추출)

   한의원 이름: 예시한의원              ← 플레이스
   대표원장:   원장 김예시              ← 블로그
   전화번호:   02-555-1234             ← 플레이스
   주소:       서울시 서초구 예시로 1   ← 플레이스
   영업시간:   월-금 09:00-18:30       ← 플레이스
               토 09:00-13:00
               일·공휴일 휴진

   ⚠️ 사업자번호: (없음 — 직접 입력 필요)
   ⚠️ 이메일: (없음 — 선택 사항)

수정할 항목이 있으면 말씀해주세요.
없으면 "확인"이라고 해주시면 바로 적용합니다.
```

**Handling edit requests:**
- "이름 틀렸어, XX한의원이야" → update that field only
- "사업자번호 123-45-67890" → add
- "확인" / "좋아" / "네" → proceed to Step 4

### Step 4 — Apply to DB (batch)

Apply all confirmed information at once:

```bash
npx wrangler d1 execute DB --local --command \
  "INSERT OR REPLACE INTO site_settings (category, key, value) VALUES
   ('general', 'site_name', '{name}'),
   ('general', 'representative', '{representative}'),
   ('contact', 'phone', '{phone}'),
   ('contact', 'address', '{address}'),
   ('info', 'business_hours', '{hours_json}');"
```

**Caution**: When writing to production DB, always get user confirmation before running with `--remote`.

## No Extract Data? (fallback)

If clinic-profile.json does not exist (extract-content not run):

1. Suggest to user first: "네이버 플레이스나 블로그 주소를 알려주시면 정보를 자동으로 가져올 수 있어요. 직접 입력하시겠어요?"
2. If URL provided → recommend running `/extract-content` first
3. If direct input → ask item by item (legacy approach)
